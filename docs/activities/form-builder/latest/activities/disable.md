# Disable

RCA.Activities.FormBuilder.Disable

## **Description**
Disables a specific user interface component (e.g. text box, dropdown list, button) on an active form. When disabled, the component becomes grayed-out and the end-user cannot enter text, select options, or click on it. This is useful for building dynamic form behaviors where elements are only editable when certain criteria are met.

![disable](/static/img/disable.png)

(\*For mandatory)

## **In the body of the activity**
* **Key Name** - The unique key name of the form element you want to disable. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"submit"`).

## **Step-by-Step Usage**
The **Disable** activity must run inside the **Do** block of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) because it operates on a live form instance.

For example, to disable the submit button (`"submitButton"`) when a checkbox named `"agreeCheckbox"` is unticked:
1. **Configure components in Form**: In the Form Builder, add a **Checkbox** component (set Label to `I Agree`, Property Name under the Field Key tab to `agreeCheckbox`, and check **Execute DO on change** under the Display tab). Add a **Button** component (set Action to `Submit` and Property Name under the Field Key tab to `submitButton`).
2. **Assign Form Id**: On the [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) activity, go to the Properties panel, under **Output** -> **Form Id**, press `Ctrl + K` to create a variable named `formId` (Variable type: `Int32`).
3. **Get Checkbox Status**: Inside the **Do** block of the `Display Form` activity, drag a [Get Element Value](/docs/activities/form-builder/latest/activities/get-element-value.md) activity:
   * **Form Id**: `formId`
   * **Key Name**: `"agreeCheckbox"`
   * **Output Data**: `isAgreed` (Create a variable of type `Object`)
4. **Check Condition**: Below that, drag an **If** activity:
   * **Condition**: `Convert.ToBoolean(isAgreed) = False`
5. **Disable Button**: Inside the **Then** branch (checkbox is unticked), drag a **Disable** activity:
   * **Form Id**: `formId`
   * **Key Name**: `"submitButton"`
6. **Enable Button**: Inside the **Else** branch (checkbox is ticked), drag an [Enable](/docs/activities/form-builder/latest/activities/enable.md) activity:
   * **Form Id**: `formId`
   * **Key Name**: `"submitButton"`

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a warning that `Key Name` or `Form Id` is required.
  * **Cause**: One of these properties is empty or the `Key Name` string lacks quotes.
  * **Solution**: Ensure `Key Name` is formatted as a string (e.g. `"submit"`) and `Form Id` is mapped to your `Form Id` variable.

### **Run-Time Errors**
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found` at runtime.
  * **Cause**: The key passed in **Key Name** (e.g. `"xyz"`) does not match any Field Key (Property Name) inside the JSON schema file.
  * **Solution**: Open the JSON schema file in Form Builder, verify the component's **Property Name** under the **Field Key** tab, and update the **Key Name** to match it exactly (case-sensitive).
* **Symptom**: `ArgumentException: Form ID does not exist`.
  * **Cause**: The form is already closed.
  * **Solution**: Ensure this activity is executed within the boundaries of the **Do** block execution loop.
