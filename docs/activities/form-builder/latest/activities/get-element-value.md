# Get Element Value

RCA.Activities.FormBuilder.GetElementValue

## **Description**
Retrieves the current value of a specific component (e.g., a text input, checkbox status, selected dropdown item) from an active form while the form remains open. This is primarily used inside the **Do** block of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) to dynamically inspect user entries and trigger conditional validation rules.

![get-element-value](/static/img/get-element-value.png)

---

## **In the body of the activity**
* **Key Name** - The unique key name of the form element from which you want to retrieve the value. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"age"`).

## **Step-by-Step Usage**
The **Get Element Value** activity operates on a live form instance, meaning it must be executed inside the **Do** block container of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md).

For example, to read a text input field and log its value dynamically:
1. **Configure components in Form**: In the Form Builder, add a **Text Field** component (set Label to `Full Name`, Property Name under the Field Key tab to `fullName`, and check **Execute DO on change** under the Display tab).
2. **Assign Form Id**: On the [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) activity, go to the Properties panel, under **Output** -> **Form Id**, press `Ctrl + K` to create a variable named `formId` (Variable type: `Int32`).
3. **Get Input Value**: Inside the **Do** block of the `Display Form` activity, drag a **Get Element Value** activity:
   * **Form Id**: `formId`
   * **Key Name**: `"fullName"`
   * **Output Data**: `fullNameObj` (Create a variable of type `Object` to store the output value)
4. **Log the Value**: Drag a **Log Message** activity below it:
   * **Message**: `"User entered name: " + Convert.ToString(fullNameObj)`

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Validation warning showing that `Output Data` is required.
  * **Cause**: You did not assign a variable to store the retrieved value.
  * **Solution**: Create a variable of type `System.Object` in your Variables panel and bind it to the **Output Data** field.

### **Run-Time Errors**
* **Symptom**: `InvalidCastException` when using the retrieved value in the workflow.
  * **Cause**: You attempted to directly assign or cast the `Object` output into an incompatible variable type (e.g., casting a text field's empty string directly into a boolean or integer variable).
  * **Solution**: Always convert or cast safely. Use `Convert.ToString(myValue)` for text, `Convert.ToBoolean(myValue)` for checkboxes, and `Convert.ToInt32(myValue)` for numbers, while checking for null values beforehand: `If(myValue IsNot Nothing, Convert.ToString(myValue), "")`.
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found`.
  * **Cause**: The key is misspelled or does not match any component's Field Key (Property Name) in the JSON schema.
  * **Solution**: Verify the key in the Form Builder UI designer.
