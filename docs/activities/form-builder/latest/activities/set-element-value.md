# Set Element Value

RCA.Activities.FormBuilder.SetElementValue

## **Description**
Programmatically updates or populates the value of a specific component (e.g., text box, checkbox, dropdown list, date picker) on an active form while it is open. This is useful for auto-filling form elements based on other user inputs (e.g., auto-populating City and State when a Zip Code is typed) or loading default data dynamically.

![set-element-value](/static/img/set-element-value.png)

---

## **In the body of the activity**
* **Key Name** - The unique key name of the form element you want to update. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"city"`).
* **Value** - The data you want to write into the form component. Supported value types are: `String`, `Int32`, `Double`, `Decimal`, `Float`, `Boolean`, `JObject`, `JArray`, `Object`.

## **Step-by-Step Usage**
The **Set Element Value** activity operates on a live form instance, so it must be executed inside the **Do** block container of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md).

For example, to automatically fill a City field (`"city"`) when the user enters a specific Zip Code (`"90210"`):
1. **Configure components in Form**: In the Form Builder, add a **Text Field** component for Zip Code (set Label to `Zip Code`, Property Name under the Field Key tab to `zipCode`, and check **Execute DO on change** under the Display tab). Add another **Text Field** component for City (set Label to `City` and Property Name under the Field Key tab to `city`).
2. **Assign Form Id**: On the [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) activity, go to the Properties panel, under **Output** -> **Form Id**, press `Ctrl + K` to create a variable named `formId` (Variable type: `Int32`).
3. **Read Zip Code**: Inside the **Do** block of the `Display Form` activity, drag a [Get Element Value](/docs/activities/form-builder/latest/activities/get-element-value.md) activity:
   * **Form Id**: `formId`
   * **Key Name**: `"zipCode"`
   * **Output Data**: `zipCodeObj` (Create a variable of type `Object`)
4. **Check Zip Code Value**: Drag an **If** activity below it:
   * **Condition**: `Convert.ToString(zipCodeObj) = "90210"`
5. **Autofill City**: Inside the **Then** branch, drag a **Set Element Value** activity:
   * **Form Id**: `formId`
   * **Key Name**: `"city"`
   * **Value**: `"Beverly Hills"`

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a warning that `Value` or `Key Name` is missing.
  * **Cause**: One of these required properties has not been assigned a variable or literal expression.
  * **Solution**: Fill in all mandatory fields. Ensure the `Key Name` is enclosed in double quotes.

### **Run-Time Errors**
* **Symptom**: `InvalidCastException` or the form component displays unexpected formatting.
  * **Cause**: The data type passed in the **Value** property does not match the type expected by the form component in the JSON schema.
    * *Example:* Passing a string `"true"` instead of a Boolean `True` to a checkbox component, or passing a non-numeric string to a number-only input.
  * **Solution**: Ensure correct type mapping:
    * Checkbox -> Boolean variable (`True`/`False`)
    * Number Field -> `Int32` or `Double` variable
    * Text Field -> `String` variable
    * Date Picker -> `DateTime` or string in the specific format required by the schema.
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found`.
  * **Cause**: The key is misspelled or does not exist in the schema.
  * **Solution**: Check the component's Field Key configuration in the Form Builder layout designer.
