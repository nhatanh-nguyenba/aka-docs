# Set Focus

RCA.Activities.FormBuilder.SetFocus

## **Description**
Sets visual focus (places the active input cursor) onto a specific input component on an active form. This is used to guide the end-user's attention to the next required field, or to highlight a field that failed validation check during dynamic form processing.

![set-focus](/static/img/set-focus.png)

---

## **In the body of the activity**
* **Key Name** - The unique key name of the form element on which you want to place focus. This value must match the component's Property Name (Field Key) from the Form Builder layout, and must be enclosed in double quotes (e.g., `"age"`).

## **Step-by-Step Usage**
The **Set Focus** activity operates on a live form instance, so it must be executed inside the **Do** block container of [Display Form](/docs/activities/form-builder/latest/activities/display-form.md).

For example, to direct the user's cursor focus back to the email field (`"email"`) if they check a checkbox to confirm verification:
1. **Configure components in Form**: In the Form Builder, add a **Checkbox** component (set Label to `Check Email`, Property Name under the Field Key tab to `validateEmail`, and check **Execute DO on change** under the Display tab). Add a **Text Field** component for Email (set Label to `Email` and Property Name under the Field Key tab to `email`).
2. **Assign Form Id**: On the [Display Form](/docs/activities/form-builder/latest/activities/display-form.md) activity, go to the Properties panel, under **Output** -> **Form Id**, press `Ctrl + K` to create a variable named `formId` (Variable type: `Int32`).
3. **Read Checkbox State**: Inside the **Do** block of the `Display Form` activity, drag a [Get Element Value](/docs/activities/form-builder/latest/activities/get-element-value.md) activity:
   * **Form Id**: `formId`
   * **Key Name**: `"validateEmail"`
   * **Output Data**: `validateEmail` (Create a variable of type `Object`)
4. **Check Condition**: Below that, drag an **If** activity:
   * **Condition**: `Convert.ToBoolean(validateEmail) = True`
5. **Set Focus**: Inside the **Then** branch, drag a **Set Focus** activity to focus on the email input field:
   * **Form Id**: `formId`
   * **Key Name**: `"email"`

## **Self-Service Troubleshooting**

### **Design-Time Validation**
* **Symptom**: Studio displays a warning that `Key Name` or `Form Id` is required.
  * **Cause**: One of these properties is empty or the `Key Name` string lacks quotes.
  * **Solution**: Ensure `Key Name` is formatted as a string (e.g. `"firstName"`) and `Form Id` is mapped to your `Form Id` variable.

### **Run-Time Errors**
* **Symptom**: The cursor does not appear on the element, but no error is thrown.
  * **Cause**:
    1. The target component is currently hidden or disabled using the [Disable](/docs/activities/form-builder/latest/activities/disable.md) activity. Hidden or disabled components cannot receive focus.
    2. Another UI window has captured the global screen focus.
  * **Solution**: Ensure the component is active (call [Enable](/docs/activities/form-builder/latest/activities/enable.md) first if it was previously disabled) before calling **Set Focus**.
* **Symptom**: `KeyNotFoundException: Component key 'xyz' not found`.
  * **Cause**: The key is misspelled or does not exist in the schema.
  * **Solution**: Check the component's Field Key configuration in the Form Builder layout designer.
