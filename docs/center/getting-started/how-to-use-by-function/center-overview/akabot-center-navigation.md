Understanding the navigation structure of akaBot Center is essential for working efficiently within the platform. The left sidebar is the primary means of moving between sections, and it is organized into logical groups that reflect the different responsibilities of RPA team members — from business users monitoring results, to operations staff managing workflows, to administrators controlling access and configuration.

The sidebar is divided into major sections: **Home**, **Monitoring** (Productivity, Report), **Automation** (Task, Workflow, Schedule, Package, Trigger), **Resources** (Agent, Agent Group, Agent Pool), **Assets**, **Queue**, and **Administration** (User Management, Role Management, Organization Unit, Webhook, License, Token). Each section can be expanded or collapsed independently, keeping the interface uncluttered for users who work primarily within one functional area.

Breadcrumb navigation at the top of the main content area always shows where you are within the application hierarchy, such as `Automation > Workflow > Edit Workflow`. Clicking any segment of the breadcrumb takes you directly to that level. Global search in the top header allows you to quickly find any resource — workflow, agent, package, or user — by keyword without having to navigate through menus.

## Navigation Structure

| Section | Sub-items | Primary Users |
|---|---|---|
| Home | — | All users |
| Monitoring | Productivity, Report | Operations, Business users |
| Automation | Task, Workflow, Schedule, Package, Trigger | Operations, IT-Dev |
| Resources | Agent, Agent Group, Agent Pool | IT-Operations |
| Assets | — | Operations, Developers |
| Queue | — | Operations, Developers |
| Administration | User Mgmt, Role Mgmt, Org Unit, Webhook, License, Token | Administrators |
