# 要素が存在するのを待つ

RCA.Activities.Browser.WaitElementExist

## **説明**

Wait Element Exist アクティビティは、選択した要素が Web ページに表示されるのを待ち、指定した時間が経過しても目的の要素が表示されない場合は、(必要に応じて) エラーをスローします。

![image-20220505135459-1.png](/static/img/2069cf_image-20220505135459-1.png)

（\*必須）

## **アクティビティ本体内**

* **ターゲット要素を選択\*** - 表示されるまで待機するフィールドを選択します。このアクティビティは、そのフィールドの場所を指定する文字列変数 (セレクター) を生成します。

## **プロパティ**

**共通**

* **エラー時に続行 (ブール値)** - ブール変数には、True または False の 2 つの値があります。
True - アクティビティ内でエラーが発生した場合でも、プロセスの残りの部分が実行を継続できるようにします。
False (デフォルト) - プロセスの実行の継続をブロックします。
* **タイムアウト MS (Int32) -** エラーがスローされるまでにアクティビティの完了を待機する最大時間 (ミリ秒単位)。タイムアウトが経過すると、アクティビティは終了します。デフォルト値: 30000 (ミリ秒)。
例: 30000

**入力**

* **セレクター (文字列)** - 指定されたフィールドを識別する文字列。
* **表示を待機 (チェックボックス)** - アクティビティを実行する前にターゲットが表示されるまでオートメーションを待機させる場合は、このボックスをオンにします。これはデフォルトでチェックされています

**その他**

* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: [2343154764] 待機要素が存在します
* **公開 (チェックボックス)** - 公開する場合にチェックを入れます。使用する前に、データ セキュリティ要件を必ず考慮してください。

**出力**

* **見つかった要素 (IWebElement)**\* - 結果として得られる UI 要素。このフィールドは、UiElement 変数のみをサポートします。

**スクロール**: アクションを実行する前にシステムが画面をスクロールしてターゲット要素を表示する方法を制御します*。*

* **水平スクロール:** 数値（ピクセル）を入力して水平にスクロールします。
  + 正の値 → 右にスクロール
  + 負の値 → 左にスクロール
  + 例: 200
* **要素をスクロールして表示 (チェックボックス):** システムは、ターゲット要素が表示されるまで自動的にスクロールします。デフォルト設定はチェックが入っていません。
* **垂直スクロール:** 数値 (ピクセル) を入力して垂直スクロールします。
  + 正の値 → 下にスクロール
  + 負の値 → 上にスクロール
  + 例: 300
