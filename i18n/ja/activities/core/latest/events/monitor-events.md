# イベントの監視

RCA.Activitites.Core.MonitorEvents

## **説明**

このアクティビティを使用すると、一連のアクティビティ (トリガーとも呼ばれます) を監視でき、トリガーされると、イベント ハンドラー コンテナーで指定されたアクティビティを実行できます。

![image-20220506112234-1.png](/static/img/02df6e_image-20220506112234-1.png)

（\*必須）

**アクティビティ本体内**

* **トリガー イベント**\* - 観察したいアクティビティまたはアクティビティのシーケンス
* **イベント ハンドラー**\* - イベントがトリガーされたときに実行するアクティビティ。

## **プロパティ**

* **Continue On Error (Boolean)** - アクティビティがエラーをスローした場合でもオートメーションを続行するかどうかを指定します。可能な値は True または False の 2 つだけです。 True - アクティビティ内でエラーが発生した場合でも、プロセスの残りの部分が実行を継続できるようにします。 False (デフォルト) - プロセスの実行の継続をブロックします。

**その他**

* **パブリック (チェックボックス)** - チェックすると、このアクティビティのデータがログに表示されます。使用する前にデータのセキュリティを考慮してください。
* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: これを監視してください

**オプション**

* **Repeat Forever (Boolean)** – このオプションは True を許可します。トリガーがアクティブになるたびにブロックが実行されます。これがデフォルト値です。 False - アクティビティは 1 回だけ実行されます。
