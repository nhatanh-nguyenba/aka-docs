# キュー項目の追加

RCA.Activities.Core.AddQueueItem

## **説明**

このアクティビティを使用すると、新しい項目をキューに追加できます。商品の状態は新品になります。

![image-20220506113229-1.png](/static/img/257b51_image-20220506113229-1.png)

（\*必須）

## **プロパティ**

**共通**

* **エラー時に続行 (ブール値)** - ブール変数には、True または False の 2 つの値があります。 True: アクティビティ内でエラーが発生した場合でも、プロセスの残りの部分が実行を継続できるようにします。 False (デフォルト): プロセスの実行の継続をブロックします。

例: 本当です。

* **TimeOut MS (数値)** - エラーがスローされるまでにアクティビティの完了を待機する最大時間 (ミリ秒単位)。タイムアウトが経過すると、アクティビティは終了します。デフォルト値: 30000 (ミリ秒)

**入力**

* **期限 (日時)** - キュー アイテムが処理される直前の日付。
* **アイテム情報** - 追加する新しいキューに関する追加情報。情報はアイテムに保存され、トランザクション処理中に使用されます。
* **アイテム情報コレクション (`Dictionary<String, Object>`)** - これにより、キュー アイテムの情報のディクショナリ全体をインポートできます。文字列のみがサポートされます。
* **延期 (日付時刻)** - キュー項目が処理される可能性がある後の日付。
* **優先度 (ドロップダウンリスト)**\* - キューの優先度レベル。低、標準、高レベルを選択できます。デフォルト値は「通常」です。
* **QueueName (String)**\* - 新しいキュー アイテムが追加されるキュー。文字列のみがサポートされています。
* **参照 (文字列)** - この機能は、自動化プロジェクト内で使用される他のアプリケーションにトランザクションをリンクするために使用されます。 255 文字に制限します。

**その他**

* **公開 (チェックボックス)** - 公開したい場合にチェックを入れます。使用する前に、データのセキュリティ要件を必ず考慮してください。
* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: キュー項目を追加します。
