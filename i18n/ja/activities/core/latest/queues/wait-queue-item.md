# 待機キュー項目

RCA.Activities.Core.WaitQueueItem

## **説明**

このアクティビティを使用すると、指定したキューからキュー アイテムを取得し、それを QueueItem 変数に保存できます。実行時にキューが空の場合、センターはキュー項目を受信する準備ができたことを開始するメッセージを Center に送信し、新しい項目が追加されるのを待ちます。

![image-20220506113715-1.png](/static/img/75b8ab_image-20220506113715-1.png)

（\*必須）

## **プロパティ**

**共通**

* **エラー時に続行 (ブール値)** - ブール変数には、True または False の 2 つの値があります。 True - アクティビティ内でエラーが発生した場合でも、プロセスの残りの部分が実行を継続できるようにします。 False (デフォルト) - プロセスの実行の継続をブロックします。
例: 本当
* **タイムアウト MS (Int32)** - エラーがスローされるまでにアクティビティの完了を待機する最大時間 (ミリ秒単位)。タイムアウトが経過すると、アクティビティは終了します。デフォルト値: 30000 (ミリ秒)。

**入力**

* **QueueName (String)** - 新しいキューが追加されるキュー。文字列のみがサポートされています。
例: 「キュー akaBot 1」
* **PollTimeMS (int)** - アクティビティが新しいアイテムを再度検索するまでの待機時間 (ミリ秒単位)。

**その他**

* **公開 (チェックボックス)** - 公開したい場合にチェックを入れます。使用する前に、データのセキュリティ要件を必ず考慮してください。
* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: 待機キュー項目。

**出力**

* **結果 (QueueItem)** - 取得されたキュー アイテムは、QueueItem 変数に保存されます。

**参考**

* **フィルター戦略** - リファレンスに従ってデータをフィルターできます。次の 2 つのオプションがあります。
・開始: 参照フィールドに示された値で始まる参照を持つ項目を探します。
・等しい：参照フィールドに示された値と同じ参照を持つ項目を検索します。
デフォルト値は次のとおりです。
* **参照 (文字列)** - 参照に基づいて項目をフィルターします。
