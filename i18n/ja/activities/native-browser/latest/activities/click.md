# クリック(注)

RCA.Activities.NativeBrowser.Click

## **説明**

Click アクティビティは、クリック アクションを Web ページに送信するために使用されます。

![image-20220505145431-1.png](/static/img/deaa58_image-20220505145431-1.png)

（\*必須）

## **アクティビティ本体内**

* **ターゲット要素を選択**\* - 存在を確認する要素を選択します。このアクティビティは、その要素の場所を指定する文字列変数 (セレクター) を生成します。
* **クリック タイプ** - クリック イベントをシミュレートするときに使用されるマウス クリックのタイプを指定します。デフォルトでは、シングルクリックが選択されています。
**CLICK\_SINGLE** - 1 回のクリック
**CLICK\_DOUBLE** - 2 回続けてクリックします。
* **マウス ボタン** - マウス ボタン (左、右) はクリック操作に使用されます。デフォルトでは、マウスの左ボタンが選択されています。
**MOUSE\_LEFT** - マウスの左ボタンを使用してクリックするアクションを設定します。
**MOUSE\_RIGHT** - マウスの右ボタンを使用してクリックするアクションを設定します。

## **プロパティ**

**共通**

* **Continue On Error (Boolean)** - このプロパティは、エラーが発生した場合にオートメーションを継続するタイミングを指定します。可能な値は True または False の 2 つだけです。 True - アクティビティ内でエラーが発生した場合でも、プロセスの残りの部分が実行を継続できるようにします。 False (デフォルト) - プロセスの実行の継続をブロックします。
* **Delay After (Int32)** - アクティビティ実行後の遅延時間 (ミリ秒単位)。デフォルトの時間は 300 ミリ秒です。
例: 300
* **Delay Before (Int32)** - アクティビティが操作の実行を開始するまでの遅延時間 (ミリ秒単位)。デフォルトの時間は 300 ミリ秒です。
例: 300
* **タイムアウト MS (Int32)** - アクティビティの実行を待機する時間 (ミリ秒単位)。タイムアウトが経過すると、ページはそれ以上読み込まれなくなります。デフォルトの時間は 30000 ミリ秒です。
例: 30000

**入力**

* **クリック タイプ (ドロップダウン リスト)** - クリック イベントをシミュレートするときに使用するマウス クリックのタイプを指定します。デフォルトでは、シングルクリックが選択されています。
* **マウス ボタン (ドロップダウン リスト)** - マウス ボタン (左、右) はクリック操作に使用されます。デフォルトでは、マウスの左ボタンが選択されています。
* **セレクター (文字列)**\* - アクティビティの実行時に特定の UI 要素を検索するために使用されるテキスト プロパティ。これは実際には、探している GUI 要素とその親の一部の属性を指定する XML フラグメントです。
* **表示を待機 (チェックボックス)** - アクティビティを実行する前にターゲットが表示されるまでオートメーションを待機させる場合は、このボックスをオンにします。これはデフォルトでチェックされています

**その他**

* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: [5656456] をクリック
* **公開 (チェックボックス)** - 公開する場合にチェックを入れます。使用する前に、データ セキュリティ要件を必ず考慮してください。デフォルトではチェックが入っていません。

**オプション**

* **同期 (チェックボックス)** – ブール変数には、True または False の 2 つの値が考えられます。 True - アクティビティの実行を続行する前に、クリック ハンドラーが完了することを許可します。 False (デフォルト) - プロセスの実行の継続をブロックします。
* **キー修飾子 (KeyModifiers)** - キー修飾子を追加できます。次のオプションが使用可能です: Alt、Ctrl、Shift、Win。
* **クリックをシミュレートする (チェックボックス)** - 選択すると、ターゲット アプリケーションのテクノロジーを使用してクリックをシミュレートします。この入力方法は 3 つの中で最も高速で、バックグラウンドで動作します。選択されていない場合、クリックはハードウェア ドライバーを使用して実行されます。この方法は最も遅く、バックグラウンドで動作できませんが、すべてのデスクトップ アプリと互換性があります。デフォルトでは、このチェックボックスはオンになっています。
