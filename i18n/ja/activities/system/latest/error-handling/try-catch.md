# トライキャッチ

System.Activities.Statements.TryCatch

## **説明**

Try Catch アクティビティを使用すると、アクティビティ内で特定の種類の例外をキャッチし、エラー通知を表示するか、エラー通知を無視してプログラムを続行できます。

![image-20220506112458-1.png](/static/img/1da425_image-20220506112458-1.png)

（\*必須）

**アクティビティ本体内**

* **試してください**\* - 例外を見つけたいアクティビティまたは一連のアクティビティをこのセクションに配置する必要があります。
* **キャッチ**\* - 探している例外の種類を指定し、必要に応じて、見つかった例外についてユーザーに通知するアクティビティを実行します。
* **最後に** - (オプション) 例外が発生したかどうかに関係なく、このブロック内のアクティビティ (利用可能な場合) は常に実行されます。

**その他**

* **パブリック (チェックボックス)** - チェックすると、このアクティビティのデータがログに表示されます。使用する前にデータのセキュリティを考慮してください。
* **表示名 (文字列)** - このアクティビティの名前。アクティビティの名前を編集して、コードをより適切に整理および構造化できます。
例: Catch ブロックを試してください。
