# はじめに

**Windows アクティビティ** パッケージには、**ウィンドウ アプリケーション** 自動化プロジェクトの作成に使用されるすべての**基本アクティビティ**が含まれています。

これらのアクティビティにより、ロボットは次のことが可能になります。

* 基本的な Windows アプリケーションの自動化のために、マウスやキーボードのコマンドの実行、テキストの入力や抽出などの人間の操作を **シミュレート** します。
* アプリケーションの動作に基づいて **トリガー** を作成すると、マシン上で特定のイベントが発生したときにロボットが特定のアクションを実行できるようになります。
* **ウィンドウ**の操作を実行します。

**ターゲット要素を選択**

UI 要素に対してアクションを実行するアクティビティは、デザイナーのアクティビティの本文にある [**ターゲット要素を選択**] ボタンを使用してデザイン時に構成できます。

[**ターゲット要素を選択**] ボタンをクリックすると、**セレクター エディター** ウィザードが開きます。

![image-20220509101433-1.png](/static/img/58b598_image-20220509101433-1.png)

**ターゲット要素を選択** ボタンをクリックすると、**セレクター** ウィザードが開きます。

![image-20220509101946-3.png](/static/img/00bf8d_image-20220509101946-3.png)

**セレクター エディター** ウィザードの最後にある**選択** ボタンをクリックすると、**指定** フィールドで現在指定している内容が指定されます。ウィザードを初めて開くときは、**ターゲット**を指定する必要があります。アンカーが使用可能な場合、ウィザードは考えられるターゲットごとにアンカーを自動的に選択します。
