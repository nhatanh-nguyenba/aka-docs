# デスクトップトリガーについて

**デスクトップ トリガー**を使用すると、新しいウィンドウが開いたり、特定のファイルが作成されたりするなど、コンピューター上で発生するイベントに応じてタスクを自動的に実行できます。

akaBot デスクトップ トリガーは、次のタイプのトリガー イベント (アクションとも呼ばれます) をサポートしています。

窓
: ウィンドウ トリガーは、アプリケーション ウィンドウのアクションが発生したときに自動タスクを起動します。 デスクトップ トリガーは、開いているアプリケーションのリストを提供し、いつでもリストを更新できるようにします。トリガー イベントとして次のオプションから選択します。

* アプリケーションウィンドウが開いたとき。
* アプリケーションウィンドウが閉じたとき。

ファイル
: ファイル トリガーは、ファイル アクションが発生したときに自動タスクを起動します。トリガー イベントとして次のオプションのいずれかを選択します。

* 新しいファイルが作成されたとき。
* 既存のファイルが削除された場合。
* ファイルの名前が変更されたとき。
* ファイルが変更されたとき。

フォルダー
: フォルダー トリガーは、イベントがコンピューター上のフォルダーに影響を与えると自動タスクを起動します。トリガー イベントとして次のオプションのいずれかを選択します。

* フォルダー内に新しいファイルが作成されたとき。
* フォルダー内の既存のファイルが削除された場合。
* フォルダー内のファイルの名前が変更されたとき。
* 新しいフォルダーが作成されたとき。
* 既存のフォルダーを削除した場合。
* フォルダーの名前が変更されたとき。
* フォルダーが変更されたとき。

**ホットキー**

ホットキー トリガーは、キーボードの事前定義されたホットキーを押すと自動タスクを起動します。

サービス
: サービス トリガーは、システム サービスがアクションを実行するときに自動タスクを起動します。トリガー イベントとして次のオプションのいずれかを選択します。

* サービスの実行が開始されたとき。
* サービスの実行が停止したとき。
* サービスが実行を再開したとき。
* サービスが一時停止したとき。

電子メール
: 電子メール トリガーは、指定した電子メール アカウントで電子メールが受信されると自動タスクを起動します。メール サーバー上で新しい受信電子メール メッセージのトリガーを指定するには、次のように指定します。

* ホスト名
* ポートの詳細
* ユーザー名
* パスワード
* 新しいメッセージをチェックする間隔
* メール アカウントで SSL を使用している場合は、[サーバーが安全な接続 (SSL) を使用する] をオンにします。

プロセス
: プロセス トリガーは、システム プロセスの開始または停止時に自動タスクを起動します。トリガー イベントとして次のオプションのいずれかを選択します。

* プロセスの実行が開始されたとき。
* プロセスの実行が停止したとき。

この機能は現在ベータ版であり、まだ完全には安定していません。ぜひこの機能を体験していただき、ご質問や問題が発生した場合はサポート チームにフィードバックをお寄せください。

**インターフェース**

インターフェイス トリガーは、ボタンのクリック、ドロップダウン メニューからの項目の選択、フォームへの記入などのユーザー インターフェイス イベントに基づいてタスクを自動化するために使用されます。

この機能は現在ベータ版であり、まだ完全には安定していません。ぜひこの機能を体験していただき、ご質問や問題が発生した場合はサポート チームにフィードバックをお寄せください。
