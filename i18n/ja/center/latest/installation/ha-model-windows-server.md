# Windows Server での高可用性モデルのインストール ガイド

TODO: このページは最新ではありません

このガイドでは、Windows Server 上の高可用性 (HA) 構成で akaBot Center を展開する方法について説明します。 HA モデルは、共有外部データベースを備えたロード バランサーの背後で複数のアプリケーション ノードを使用し、1 つのノードに障害が発生した場合でも自動化が中断されることなく継続されることを保証します。

## 前提条件

- 2 つ以上の Windows Server 2019 または 2022 (64 ビット) ノード (それぞれ少なくとも 16 GB RAM と 100 GB SSD を搭載)
- `JAVA_HOME`が構成されたすべてのノードに Java JDK 17 がインストールされていること
- 専用サーバー上の共有外部データベース (MySQL 8.0、MSSQL 2019、または PostgreSQL 14)
- Windows ネットワーク負荷分散 (NLB) または外部ロード バランサー (F5、Windows 用 Nginx など) が構成されている
- akaBot Center 4.x インストール パッケージ
- すべてのノードとデータベースサーバーへの管理者アクセス
- ActiveMQ 共有ストアのすべてのノードからアクセスできる共有ネットワーク パス (UNC パスまたはマップされたドライブ)

## インストール手順

1. **すべての Windows ノードを準備します**
   - 各ノードに Java JDK 17 をインストールし、システム環境変数に`JAVA_HOME`を設定します。
   - すべてのノードが相互に通信できること、および必要なポートでデータベース サーバーと通信できることを確認します。
   - Windows タイム サービス (`w32tm`) を介してシステム クロックを同期します。

2. **共有外部データベースのセットアップ**
   - (akaBot ノードではなく) 専用サーバーにデータベースをインストールして構成します。
   - それぞれのデータベース ガイドの説明に従って、akaBot データベースとユーザーを作成します。

3. **ノード 1 に akaBot Center をインストールします**
   - akaBot Center 4.x パッケージをノード 1 の`C:\akabot\center`に抽出します。
   - `conf\application.yml`を共有データベース URL、資格情報、`server.address`の NLB 仮想 IP またはホスト名で編集します。

4. **データベースの初期化 (ノード 1 のみ)**
   - 管理者特権でコマンド プロンプトを開き、`C:\akabot\center\bin\db-init.bat`を実行します。
   - これを後続のノードでは実行しないでください。

5. **ActiveMQ 共有ストアの構成**
   - すべてのノードに共有ネットワーク パス (`\\fileserver\akabot-activemq`など) をマウントします。
   - `conf\activemq.xml`を編集して、`kahaDB directory`をすべてのノードの共有ネットワーク パスに設定します。

6. **ノード 2 (および追加のノード) のインストールと構成**
   - akaBot Center パッケージをノード 2 の`C:\akabot\center`に抽出します。
   - 同じ`application.yml`と`activemq.xml`をノード 1 からコピーします。
   - ノード 2 では`db-init.bat`を実行しないでください。

7. **すべてのノードで Windows サービスとして登録します**
   - 各ノードで`install-service.bat`を管理者として実行します。

8. **ロードバランサーの構成**
   - ポート 8080 および 8443 のトラフィックをすべての akaBot Center ノードに転送するように NLB またはロード バランサーを設定します。
   - セッション管理構成で必要な場合は、スティッキー セッション (セッション アフィニティ) を構成します。

9. **すべてのノードでサービスを開始します**
   - Services Manager を介して各ノードで akaBot Center サービスを開始します。
   - `C:\akabot\center\logs\`のアプリケーション ログをチェックして、起動エラーがないことを確認します。

10. **テストフェイルオーバー**
    - ノード 1 で akaBot Center サービスを停止します。
    - ロード バランサがトラフィックをノード 2 にルーティングし、ポータルがアクセス可能な状態であることを確認します。

## 注意事項と警告

> **注意:** デプロイメントでステートフル セッション管理を使用している場合は、ロード バランサーでセッションの永続性 (スティッキー セッション) を構成します。これを行わないと、ロード バランサーがユーザーを別のノードにルーティングするときにユーザーが予期せずログアウトされる可能性があります。

> **警告:** ActiveMQ の共有ネットワーク パスは高可用性である必要があります。この共有がオフラインになると、すべてのノード上のメッセージ ブローカーが失敗し、すべてのロボット通信が中断されます。

> **注意:** 各ノードの Windows ファイアウォールが、他のノードおよびロボット マシンからのポート 8080、8443、および 61616 での受信接続を許可していることを確認してください。
