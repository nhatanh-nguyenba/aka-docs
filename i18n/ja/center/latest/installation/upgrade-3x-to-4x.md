# akaBot Center をバージョン 3x からバージョン 4x にアップグレードする

> このガイドでは、データベース エンジンとして **My SQL** を使用して、**akaBot Center の既存のインスタンスをバージョン 3.x からバージョン 4.x** に **アップグレード**する手順を説明します。このアップグレード手順の対象となるオペレーティング システム (OS) は **Microsoft Windows** です。

**全体的に、4 つの主なステップがあります:**

1. データベース、構成ファイル、ライセンス、ワークフロー ファイルのバックアップ

2. Tomcat(8)、Java(11)をアンインストールする

3. akaBot 4x をインストールする

4. 設定ファイル、ライセンス、ワークフローファイルを復元します

## **0。アカボット 3x**

続行する前に、標準の手順に従って akaBot 3.x がすでにインストールされていることを確認する必要があります。 **[View details](https://docs.akabot.com/bin/view/akaBot%20Center/Center%20Installation/Installation/Basic%20installation%20guide%20for%20Akabot%20Center%20and%20MySQL/#edit)**

# **1. MySQL データベース、構成ファイル、ライセンス、ワークフロー ファイルをバックアップします**

**1.1.データベースのバックアップ**

データベースは、次の 2 つの方法のいずれかを使用してバックアップできます。

***1.1.1 MySQL Workbench を使用したバックアップ***

* **ステップ 1:** AkaBot Center のコンピュータで MySql Workbench を開き、**「ローカル インスタンス MySQL80」** を選択します。

![1774253010662-602.png](/static/img/5b5a99_1774253010662-602.png)

* **ステップ 2:** パスワードを入力します (例: sis@12345)。パスワードをボールトに保存している場合は、この手順をスキップしてください。

![1774253064566-426.png](/static/img/cf013d_1774253064566-426.png)

* **ステップ 3:**「管理」** タブに移動し、**管理** セクションの **「データ エクスポート**」をクリックします

![1774253185643-359.png](/static/img/3f9099_1774253185643-359.png)

* **ステップ 4**: バックアップ パラメータを設定します。
  + (1) aka\_orchestrator スキーマを確認する
  + (2) ダンプ構造とデータを選択します
  + (3)「エクスポートするオブジェクト」を全てチェックします。
  + (4) 自己完結型ファイルにエクスポートにチェックを入れる
  + (5) ファイルを保存する場所を選択します
  + (6) チェック 単一トランザクションでダンプを作成する

![1774253408374-361.png](/static/img/26bcc1_1774253408374-361.png)

* **ステップ 5:**スキーマ (別名 \_orchestrator) をクリックして、右側のテーブル/ビューをオンまたはオフにして、どのテーブル、ビューをエクスポートするかを決定できます。すべてエクスポートしたい場合は、この手順をスキップしてください。

![1774253573828-858.png](/static/img/5536e8_1774253573828-858.png)

* **ステップ 6**: [エクスポート] をクリックします。

![1774253676179-539.png](/static/img/31ff6e_1774253676179-539.png)

エクスポートの進行状況ウィンドウが表示されます。エラーがなければ、選択したディレクトリにバックアップ ファイル (.sql) が生成されます。

![1774253710722-938.png](/static/img/972fd3_1774253710722-938.png)

![1774253893945-268.png](/static/img/fa903b_1774253893945-268.png)

***1.1.2.コマンドラインを使用してバックアップします***

* **ステップ 1:** コマンド ラインを開き、次のように入力してディレクトリを MySql ベース ディレクトリ (通常は C:\Program Files\MySQL\MySQL Server 8.0\bin) に変更します: cd +baseDirectory

![1774254026988-129.png](/static/img/3725d0_1774254026988-129.png)'

* **ステップ 2:** 次のコマンドを使用してデータベースをエクスポートします: **mysqldump u root p --databases aka\_orchestrator > file.sql**
  + ` ここで、file.sql はファイル名とファイルを保存する場所です。例: mysqldump u root p --databases aka\_orchestrator > d:\aka\_orchestrator.sql

![1774254400000-797.png](/static/img/29c688_1774254400000-797.png)

* **ステップ 3:**cmd により、root ユーザーのパスワードの入力が求められます。 (例: sis@12345) を入力し、Enter キーを押してしばらくお待ちください。選択したディレクトリに aka\_orchestrator.sql ファイルが表示されます。データベースから特定のテーブルのみをダンプするには、コマンド ラインでデータベース名に続いて名前を付けます。**mysqldump u root p --databases aka\_orchestrator act\_id\_info act\_ru\_jobasset > d:\aka\_orchestrator.sql**

**1.2. C****設定ファイル、ライセンス、ワークフロー ファイル**

***1.2.1. Tomcat 8*** から設定ファイルをコピーします。

- [プログラム ファイル] > [Apache Software Foundation] > [Tomcat 8.5] > [webapps] > [ROOT] > [WEB-INF] > [クラス] > [構成] を開きます。

- 画像内のこれらの構成ファイルをすべてコピーし、特定のディレクトリに貼り付けます。

![1774260123196-735.png](/static/img/6a3db0_1774260123196-735.png)

***1.2.2. Tomcat 8*** からライセンスをコピーします

- [プログラム ファイル] > [Apache Software Foundation] > [Tomcat 8.5] > [webapps] > [ROOT] > [WEB-INF] > [クラス] を開きます。

- **「ライセンス」**をコピーして、特定のディレクトリに貼り付けます

![1774260028768-432.png](/static/img/241964_1774260028768-432.png)

***1.2.3. Tomcat 8*** からワークフロー ファイルをコピーします。

- [プログラム ファイル] > [Apache Software Foundation] > [Tomcat 8.5] > [Filestorage] > [Nugets] を開きます。

- フォルダー **「Nugets**」をコピーし、特定のディレクトリに貼り付けます。

![1774259611766-788.png](/static/img/b936d3_1774259611766-788.png)

# **2. Tomcat(8)、Java(11) をアンインストールします**

**2.1. Tomcat をアンインストールする (8)**

(1) **設定**を開き、**アプリ**を選択します

(2) **Tomcat (8)** を選択し、オプション **「アンインストール」** を選択します

![1774256004210-859.png](/static/img/eff4c6_1774256004210-859.png)

**2.2. Java のアンインストール (11)**

(1) **設定**を開き、**アプリ**を選択します

(2) **Java (8)** を選択し、オプション **「アンインストール」** を選択します

![1774256552008-273.png](/static/img/694e2c_1774256552008-273.png)

**3. akabot 4x** をインストールする

ダウンロード（[Install akaBot 4x](https://docs.akabot.com/bin/view/akaBot%20Center/Center%20Installation/Installation/Install%20Akabot%20Center%204x%20and%20MySQL%20on%20Microsoft%20Windows/)の指示に従ってください）

# **4.構成ファイル、ライセンス、ワークフロー ファイルを復元します**

**ステップ 1:**前のステップで設定、ライセンス、およびワークフロー ファイル (Tomcat 8 の) を保存したディレクトリにアクセスします。

**ステップ 2:**これらのファイルをすべてコピーします

**ステップ 3**: これらのファイルを **Tomcat 10** ディレクトリ内のそれぞれの場所に貼り付けます。
