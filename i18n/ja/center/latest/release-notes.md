# Akabot Center — リリースノート

## v4.0.0.14

**更新されました**

* **スケジュール**
  - スケジュール作成時のエージェント グループ／エージェント プールの選択ロジックおよびUIを見直しました。いずれか一方のみ選択可能となり、2つのタブを1つに統合しました。

* **エージェント プール**
  - エージェント プール編集時にワークフローを変更できないよう無効化しました。

**バグ修正**

* **パッケージ**
  - ファイルをアップロードする際のMIMEタイプの判別に関するブラウザ互換性を向上させ、パッケージアップロードのバグを修正しました。

* **カレンダー**
  - カレンダーにエージェントのスケジュールが一部表示されないバグを修正しました。
  - 休日設定がある場合に、カレンダーが正しく表示されない問題を修正しました。

* **スケジュール**
  - スケジュール詳細の表示情報（最終実行時刻・次回実行時刻）が誤って表示されるバグを修正しました。

* **エージェント プール**
  - 選択されたワークフローのエージェント グループに属さないエージェントが誤って選択されるバグを修正しました。

## v4.0.0.12

**新機能**

**トリガー**
- エージェント グループ条件を追加しました。

**更新されました**

**トリガー**
- 「実行中」の状態が選択された場合に「タスクの削除」アクティビティを非表示にしました。

**エクスポート ログ**
- ログのエクスポート時の 65,000 行の制限を削除しました。

**バグ修正**

**トリガー**
- トリガー ソースにキュー コンポーネントがある場合のトリガー履歴表示を修正しました。
- スタート タスク コンポーネントを含むトリガーを編集する際のワークフロー表示を修正しました。

**スケジュール**
- 毎月のトリガーを選択する際のタブ切り替えのバグを修正しました。
- クリーン データが設定された場合のスケジュール クエリを修正しました（データを bigint ではなく text にキャスト）。

## v3.2.0

**新機能**
- **オートパイロット オーケストレーション** — 自然言語でトリガー条件と目標を説明します。 Center はスケジュールとキューの構成を自動的に構築します。
- **ロボットの健全性ダッシュボード** — 接続されているすべてのロボットの CPU、メモリ、セッション ステータスをリアルタイムで表示する新しい専用ビュー。
- **環境プロモーション ウィザード** — 承認ゲートを使用して、開発 → UAT → 製品のプロセス パッケージをプロモーションするためのガイド付きワークフロー。

**改善点**
- よりきめ細かいディスパッチ制御のために、キュー項目の優先順位レベルが 3 から 10 に増加しました。
- 実行ログ検索で、正規表現パターンと日付範囲フィルターがサポートされるようになりました。
- `/processes`および`/jobs`エンドポイントの API 応答時間は 35% 改善されました。

**バグ修正**
- 2 つのスケジュールが同じ 1 秒以内に起動された場合にジョブの重複ディスパッチを引き起こす競合状態を修正しました。
- GMT+7 タイムゾーンにおけるロボットの「最後に表示された」タイムスタンプが正しくない問題を解決しました。
- ユーザーごとにフィルタリングする際の監査ログの壊れたページネーションを修正しました。

## v3.1.0

**新機能**
- **マルチテナント** — 単一の Center インストールで、個別のユーザー、ロボット、プロセスを持つ複数の分離テナントをホストできるようになりました。
- **Webhook トリガー** - 外部システムからの受信 HTTP POST を介してプロセスを開始します。
- **SLA 追跡** — キューごとに予想される完了時間を定義します。アイテムが SLA に違反した場合にアラートを送信します。

**改善点**
- 権限グループ テンプレート (オペレーター、開発者、管理者) を使用して再設計された役割管理 UI。
- パッケージのアップロード サイズ制限が 50 MB から 200 MB に増加しました。

**バグ修正**
- スケジュールを無効にしても、次の保留中のトリガーがすぐに停止されない問題を修正しました。
- アイテムが放棄されて再試行されたときのキューのスループット メトリックを修正しました。

## v3.0.0

**新機能**
- **Center 3.0 ポータル** — アクセシビリティが向上し、React 上に構築された完全に再設計された Web UI (WCAG 2.1 AA)。
- **高可用性モード** — Raft コンセンサスを使用した自動リーダー選出によるアクティブ/パッシブ フェイルオーバー。
- **きめ細かい監査ログ** - すべての API 呼び出し、ログイン、構成変更が IP とユーザー エージェントとともに記録されるようになりました。

**重大な変更**
- v2 REST API (`/api/v2/`) は非推奨になりました。 v4.0 リリースの前に`/api/v3/`に移行します。
- v2.x のロボット マシン キーは、アップグレード後に再登録する必要があります。

**バグ修正**
- 同時実行性の高いキュー処理におけるデータベースのデッドロックが解決されました。
- ディレクトリ サーバーが部分的な結果を返したときに、LDAP グループの同期がサイレントに失敗する問題を修正しました。

## v3.0.1.2

**新機能**
1. [タスク] デスクトップ トリガーによってタスクが実行されると、タスク リストのそのタスクのソース フィールドに「デスクトップ トリガー」と表示されます。

2. [タスク] 複数のエージェントを選択して新しいタスクを作成するときのエラー メッセージ

3. [プロセス設計] 長時間実行されるプロセス、特に Bot タスクを設計する場合、設計者は、組織単位 (OU) による制限を受けることなく、そのタスクを任意のエージェントに柔軟に割り当てることができます。

4. 【コラボレーションセンター】「プロセスの開始」画面をより使いやすく改善しました。

5. [休日設定] よりユーザーフレンドリーなアプローチで休日設定を強化します。ユーザーは、「タスクを遅延する」オプションを選択するときに、特定の開始時間を選択できます。

6. [コラボレーション センター] ユーザーがアクションを実行しやすいように、コラボレーション センター内の特定のコンポーネントの位置を変更します。

7. [コラボレーション センター] コラボレーション センターのフォームの下にパディングを追加して、ユーザー インターフェイスを改善します。

8. [ライセンス] Linuxにインストールする場合、Aegisに返送するアクティベーションキーの生成時にマシンコードがキャプチャされない場合があります。ただし、ユーザーがメッセージを受信し、それに応じて調整できるようにするための改良が加えられています。

9. [タスク] 実行中のタスクを削除する際のメッセージ内容を改善しました。

**利用可能な機能**
1. [アセット] 新しいアセット タイプ: オブジェクト。この新しいアセット タイプにより、ユーザーはパラメーターを追加するか、JSON を使用して値を作成できます。

2. [ログイン] ユーザー名とパスワードを使用した基本認証により、別の REST API サーバーのアカウントでセンター ログインをサポートします。

3. [担当者プール] 担当者プールの作成のサポートは、実行が必要なヒューマン タスクを同じ責任を持つ個人のグループに割り当て、タスクを確実に完了できるようにするために提供されます。

4. [プロセス設計] リクエスタへの割り当て: プロセスの設計中の設定のサポートには、ヒューマン タスクの設計時に「リクエスタへの割り当て」を追加するオプションが含まれます。これにより、プロセスを実行している人がヒューマン タスクを実行できるようになります。

5. [プロセスの開始] 長時間実行プロセスを Queue and Trigger on Center で開始します。

6. [新しいフォーム デザイン コンポーネント] フォーム デザインの新しいコンポーネント: アップロード、ダウンロード、プレビューの 3 つの新しいタイプのコンポーネントをサポートします。
  - アップロード: ユーザーがコラボレーション センターでタスクを実行するときにファイルをアップロードできるようにします。
- ダウンロード: ユーザーがコラボレーション センターでタスクを実行するときにファイルをダウンロードできるようにします。
 - プレビュー: ユーザーがコラボレーション センターでタスクを実行するときにファイルをプレビューできるようにします。

7. [コラボレーション センターで検索] フォーム内の値でタスクを検索する: コラボレーション センターでは、ユーザーはタスク名またはプロセス名でタスクを検索するだけでなく、各タスクの対応するフォーム内に表示される特定の値を検索することもできます。

8. コラボレーション センターのモバイル モード。

9. [保留中のタスクをカウント] 保留中のタスクをカウントし、メニューバーに直接通知バッジを表示する機能を追加します。

10. 【2段階の資産管理】 資産を2段階で管理する機能を追加します。
- レベル 1: 管理者は、1 つのアセットを個人または個人のグループに割り当てることができます。割り当てられた個人またはグループのみが表示および編集できます。
- レベル 2: 管理者は、資産セット内の特定の値を個人または個人のグループに割り当てることができます。割り当てられた個人またはグループのみが値を表示および編集できます。

11. [サポートボット] 自動化中に他のシステムでの認証にユーザーのセンターのユーザー名/パスワードを使用するボットをサポートします。

12. [トリガーの追加] センターのトリガー機能を更新します。
-  実行中の別のワークフローを停止するアクティビティを設定できます。
- 新しいトリガーを作成し、アクティビティとして「電子メールの送信」を選択する場合、タスクの実行時に詳細なエラー情報を表示するために「エラー情報」フィールドを追加することをお勧めします。さらに、電子メール内で HTML フォーマットをサポートすることにより、ユーザーはテキストを簡単にフォーマット (太字、斜体など) できるようになります。
