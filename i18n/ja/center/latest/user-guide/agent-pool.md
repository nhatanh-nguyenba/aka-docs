# センターのエージェントプールの使用方法

**1.前提条件**

エージェント プール機能を使用する前に、次の要件が満たされていることを確認してください。

* akaBot Center がインストールされ、アクセス可能になります。
* Postman または別の HTTP クライアントを使用できます。
* HTTP リクエスト経由でエージェント プールをトリガーする場合は、「マネージド トークン ガイドライン」ドキュメントの手順に従ってマネージド トークンを作成する必要があります。

**2.エージェント プールの作成**

**エージェント プール** は、特定のワークフローを実行するように構成された akaBot エージェントのグループです。一貫性と安定した実行を確保するには、同じプール内のすべてのエージェントに同一の環境が必要です。

akaBot Center で新しいエージェント プールを作成するには、**エージェント > エージェント プール > 新規作成** に移動します。

![1771820288041-812.png](/static/img/72eef5_1771820288041-812.png)

[*エージェント プールの作成*] ダイアログで:

1. エージェント プールの意味のある名前を入力します。
2. エージェントが実行するワークフローを選択します。
3. [**トークン**] ドロップダウン リストで、エージェント プールにメッセージをプッシュするための HTTP リクエストを呼び出すために使用されるマネージド トークンを選択します。
*センターのスケジュール機能のみを介してエージェント プールをトリガーする予定がある場合、この手順はオプションです。*
4. ワークフローの実行を担当する 1 つ以上のエージェントを選択します。

![1771820332495-836.png](/static/img/6e0ce6_1771820332495-836.png)

作成後、新しいプールがエージェント プール リストに表示されます。

![1771820349412-497.png](/static/img/1efb90_1771820349412-497.png)

エージェント プールをクリックすると、その詳細な構成を表示できます。

![1771820364096-384.png](/static/img/9420bd_1771820364096-384.png)

**3.スケジュールを使用してエージェント プールをトリガー**

akaBot Center のスケジュール機能を使用して、エージェント プールを自動的にトリガーできます。

**[オートメーション] > [スケジュール] > [新規作成]** に移動します。

![1771820395129-275.png](/static/img/a4cfc5_1771820395129-275.png)

*新しいスケジュールの作成* ダイアログで:

1. スケジュールにわかりやすい名前を入力します。
2. 対象のワークフローを選択します。このワークフローは、エージェント プールで構成されているものと同じである必要があります。
3. 時間の繰り返しを構成して、スケジュールを実行する頻度を定義します。

![1771820414485-470.png](/static/img/4ba9be_1771820414485-470.png)

次に、[**転送先**] タブに移動します。

* トリガーするエージェント プールを選択します。
* [**保存**] をクリックして構成を完了します。

![1771820434228-395.png](/static/img/05f330_1771820434228-395.png)

スケジュールが作成されると、スケジュール リストに表示されます。

![1771820452793-860.png](/static/img/2e595b_1771820452793-860.png)

**実行動作**

スケジュールがトリガーされると、次のようになります。

* 新しいキューアイテムが作成されます。
* プール内の 1 つの利用可能なエージェントがキュー アイテムを取得し、実行を開始します。
* タスクの実行中、キュー アイテムのステータスは **InProgress** になります。
* 対応可能なエージェントがない場合、キュー アイテムはエージェントが対応可能になるまで **新規** ステータスのままになります。
* 実行後:
  + 正常に完了すると、ステータスは **成功** になります。
  + ビジネス例外が発生すると、ステータスは **失敗** になります。

![1771820474256-428.png](/static/img/559350_1771820474256-428.png)

![1771820485476-954.png](/static/img/cc9915_1771820485476-954.png)

**4.サードパーティクライアントを使用してエージェントプールをトリガー**

![1771820545137-108.png](/static/img/81a915_1771820545137-108.png)

![1771820552772-723.png](/static/img/2502fe_1771820552772-723.png)

リクエストを送信するには、次のものが必要です。

* **エージェント プール キー**
* 有効な **マネージド トークン**

たとえば、Postman を使用して API を呼び出すことができます。

![1771820580767-314.png](/static/img/43fead_1771820580767-314.png)

**入力データをワークフローに渡す**

入力データをワークフローに渡す場合は、エスケープされた JSON 文字列をリクエスト本文の **specificData** プロパティに含めます。

![1771820599994-165.png](/static/img/1f06b2_1771820599994-165.png)

ワークフローは次のことを行う必要があります。

* **inputData** という名前の **String** 型の入力引数を定義します。
* **Deserialize Json\<T>** アクティビティを使用して、入力文字列を JSON オブジェクトに変換してさらに処理します。

![1771820619342-196.png](/static/img/f47122_1771820619342-196.png)

**リクエストステータスのクエリ**

サーバーからの応答には、**requestId** という名前のプロパティが含まれています。この値は、後でリクエスト メッセージのステータスを照会するために使用できます。

![1771820647729-876.png](/static/img/302a99_1771820647729-876.png)

![1771820660862-792.png](/static/img/407911_1771820660862-792.png)
