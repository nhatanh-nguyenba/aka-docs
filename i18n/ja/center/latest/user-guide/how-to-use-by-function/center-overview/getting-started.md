akaBot Center へようこそ — RPA 運用のための集中管理プラットフォームです。このガイドでは、チームが自動化ワークフローの展開と管理を開始できるように、システムにアクセスして構成するために必要な最初の手順を説明します。続行する前に、akaBot Center のインスタンスがインストールされていてアクセス可能であること、およびシステム管理者から有効なログイン資格情報を受け取っていることを確認してください。

まず、サポートされている Web ブラウザを開いて、akaBot Center インスタンスの URL (`http://your-server:8080`など) に移動します。割り当てられたユーザー名とパスワードを使用してログインします。初めてログインすると、デフォルトのパスワードを変更するように求められる場合があります。認証されると、Center Home ダッシュボードが表示され、実行中のジョブ、保留中のスケジュール、エージェントの可用性など、現在の自動化アクティビティの概要が表示されます。

ログイン後、新しい展開で推奨されるセットアップ シーケンスは、最初にエージェントを登録し、次にエージェントをグループまたはプールに編成し、次に自動化パッケージをアップロードし、最後にワークフローとスケジュールを構成することです。この順序により、自動化ジョブのオーケストレーションを開始する前にすべての実行リソースが利用可能になります。

## 初期セットアップのチェックリスト

- [ ] akaBot Center Web インターフェイスにアクセスしてログインします。
- [ ] デフォルトのパスワードを変更します (初回ログインの場合)
- [ ] 少なくとも 1 つの akaBot Agent を登録します
- [ ] エージェント グループまたはエージェント プールを作成します (オプションですが推奨)
- [ ] Studio 経由で自動化パッケージをアップロードする
- [ ] パッケージをエージェントまたはプールにリンクするワークフローを作成します
- [ ] スケジュールまたはトリガーを構成して実行を自動化する
- [ ] ホーム ダッシュボードを確認してジョブの実行を確認します

## センターにアクセスするためのシステム要件

| 要件 | おすすめ |
|---|---|
| ブラウザ | Chrome 90 以降、Edge 90 以降、Firefox 88 以降 |
| ネットワーク | 設定されたポート (デフォルトは 8080) でのセンター サーバーへのアクセス |
| アカウント | 少なくとも閲覧者の役割を持つ有効なユーザー アカウント |
| 画面解像度 | 1280×720以上 |
