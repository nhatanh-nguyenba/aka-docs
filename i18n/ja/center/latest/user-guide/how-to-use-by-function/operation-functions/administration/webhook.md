# Webhook

Webhook を使用すると、akaBot プラットフォームをアプリケーション エコシステム全体とより効果的に統合できます。 Center イベントをサブスクライブし、外部の DCM、BPM、または CRM ソリューションに転送して、新しいキュー アイテムの処理準備ができたとき、トリガーの失敗、プロセスの変更などをさまざまなユーザーに通知できます。

注: アプリケーションは Web フックされており、akaBot と互換性がある必要があります。

[**Webhook**] タブを選択すると、次のページが表示されます。

![image-20221101165929-27.png](/static/img/96aed6_image-20221101165929-27.png)

| いいえ | コラム | 説明 |
| --- | --- | --- |
| 1 | アクション | Webhook を管理するために使用できるアクション。含まれるもの: 編集: ユーザーが特定の Webhook を編集できるようにします。削除: ユーザーが選択した Webhook を削除できるようにします。 |
| 2 | URL | アプリケーションの URL |
| 3 | サブスクライブされたイベント | イベントはアプリケーションに転送されます。 |
| 4 | 有効にする | Webhook のステータス。 **有効**または**無効**の 2 つのステータスが考えられます。 |

## **Webhook の作成/編集**

Webhook を作成するには

![image-20221101165953-28.png](/static/img/5d5b4c_image-20221101165953-28.png)

| いいえ | 列/ラベル | 説明 | タイプ | 最大値 | 必須ですか? | 入力要件 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | URL | akabot センターを購読して聞く権利を与えるシステムの URL を入力してください | 文字列 | 1000文字 | はい |  |
| 2 | 有効 | この Webhook を機能させたい場合は、このボックスにチェックを入れてください | チェックボックス |  | いいえ |  |
| 3 | すべてのイベントを選択 / イベントを選択 | すべてのイベントをアプリケーションに送信するか、選択したイベントのみを送信するかを選択できます。 | 単一選択 |  | はい |  |
| 4 | フィルター | アプリケーションに転送するイベントを選択します。リストから複数のイベントを選択できます。 | 複数選択 |  | はい | **イベントの選択** オプションを選択した場合のみ利用可能 |
