akaBot Center のパッケージは、自動化のデプロイ可能な単位、つまり akaBot Studio で作成された自動化プロジェクトのコンパイルされた出力です。開発者が Studio でオートメーションの構築とテストを完了すると、それを NuGet 形式のパッケージ ファイル (.nupkg) として公開し、akaBot Center にアップロードします。 Center は、公開されたすべてのパッケージをパッケージ ライブラリに保存して管理し、組織全体での自動化ロジックのバージョン管理、ロールバック、および制御された展開を可能にします。

**パッケージ** ページには、akaBot Studio から公開されたすべてのプロジェクトと、手動でアップロードされたプロジェクトが表示されます。プロジェクトは、akaBot Center に公開されるとパッケージになります。

[**パッケージ**] ページにアクセスするには、左側のメニューの [**パッケージ**] タブをクリックします。クリックすると、センターに公開されたすべての**パッケージ**を含む**パッケージ一覧**ページが開きます。

![image-20221028171907-10.png](/static/img/9b7527_image-20221028171907-10.png)

| いいえ | コラム | 説明 |
| --- | --- | --- |
|  | 名前 | akaBot Center に公開されたパッケージの名前 |
|  | バージョン | akaBot Center に公開されたパッケージの最新バージョン。パッケージのバージョンは、akaBot Studio で公開オプションを選択したときに決定されます。 |
|  | ツールのバージョン | パッケージの公開元となった、インストールされている Studio のバージョン |
|  | 作成日 | プロジェクトが公開された日付 |
|  | ステータス | 言及されたプロジェクトの最新バージョンのステータス。パッケージ バージョンには次のステータスがあります: **アクティブ** - 現在プロセスにデプロイされています。 **INACTIVE** - 現在プロセスにデプロイされていません（削除できるのはプロセスのみです）。 |
|  | 説明 | プロジェクト用に作成した説明。 |
|  | リリースノート | このメモは、各パッケージ バージョンの Studio に追加されました。 |
|  | 作成者 | センターでパッケージを作成したユーザー/エージェント |

## **a.パッケージを見る**

**パッケージ**の詳細を表示するには、ユーザーは**目**ボタンをクリックします。詳細ページには、センターにアップロード/公開された、選択したパッケージのすべてのバージョンを含むリストが表示されます。

![image-20221028172050-12.png](/static/img/5c4ee3_image-20221028172050-12.png)

| いいえ | コラム | 説明 |
| --- | --- | --- |
|  | 名前 | akaBot Center に公開されたパッケージの名前 |
|  | ステータス | パッケージのバージョンのステータス。パッケージ バージョンには次のステータスがあります。 **アクティブ** - 現在プロセスにデプロイされています。 **非アクティブ** - 現在プロセスにデプロイされていません（削除できるのはプロセスのみです）。 |
|  | 作成日 | バージョンが公開された日付 |
|  | バージョン | akaBot Center に公開されたパッケージのバージョン。 |
|  | ツールのバージョン | パッケージの公開元となった、インストールされている Studio のバージョン |
|  | 説明 | プロジェクト用に作成した説明。 |
|  | リリースノート | それぞれのパッケージ バージョンの公開時に Studio で追加されたメモ |
|  | 作成者 | センターでパッケージを作成したユーザー/エージェント |

## **b.パッケージの検索**

このオプションを使用すると、パッケージ名を入力してパッケージを検索できます。

![image-20221028172147-13.png](/static/img/42d36d_image-20221028172147-13.png)

## **c.パッケージのアップロード**

通常、ワークフローをセンターに公開すると、パッケージが自動的にセンターにアップロードされます。ただし、[リスト] ページの右上にある [新規作成] ボタンを使用して、**手動でパッケージを Center にアップロード**することもできます。

![image-20221028172417-14.png](/static/img/84d523_image-20221028172417-14.png)

パッケージをアップロードするためのウィンドウが表示されます。

![image-20221028172438-15.png](/static/img/b61202_image-20221028172438-15.png)

「ファイルを選択」をクリックしてパッケージ（拡張子 .nupkg 付き）を選択し、**開く** をクリックします。

ローカルマシンからアップロードする**パッケージ**を選択したら、**保存**を押してアップロードを終了し、**キャンセル**を押して操作を終了します。

## **d.パッケージの削除**

[パッケージ リポジトリ] ページで、パッケージの名前をクリックします。パッケージ情報とすべてのバージョンのリストが表示されます。

![image-20221028172537-16.png](/static/img/e652f6_image-20221028172537-16.png)

削除するパッケージのバージョンに対応する [**削除**] アクションを選択します。確認のポップアップが表示されます。 [**削除**] を押して操作を完了します。

![image-20221028172552-17.png](/static/img/4a905d_image-20221028172552-17.png)

選択したパッケージのバージョンが削除されます。 **[関連項目を削除しますか?**] が選択されている場合、このパッケージにリンクされているタスク、ログ、スケジュール、ワークフローも削除されます。
