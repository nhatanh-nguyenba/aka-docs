# IT開発チーム向け

## **使用例 1: OU の表示/作成/編集**

S1: **組織単位ページ**にアクセスするには、左側のメニューで**管理タブ**をクリックし、次に**組織単位タブ**をクリックします。

![image-20230308181201-1.png](/static/img/5660d8_image-20230308181201-1.png)

S2: OU の詳細をすべて表示するには、**目** ボタンをクリックします。

![image-20230308181201-2.png](/static/img/c4be5a_image-20230308181201-2.png)

すると詳細ページが表示されます

![image-20230308181201-3.png](/static/img/3ddacc_image-20230308181201-3.png)

|  |  |
| --- | --- |
| 列/ラベル | 説明 |
| 名前 | OUの名前 |
| ユーザー | この OU に割り当てられているすべてのユーザーのリスト。このリストには次のものが含まれます: *ユーザー名* 電子メール |

S3: OUを作成/編集するには

**「新規作成」** ボタンをクリックします。 OU を編集する場合は、対応する OU 名の横にある [アクション] から編集オプションを選択します。

![image-20230308181201-4.png](/static/img/882d1e_image-20230308181201-4.png)

S4: クリックすると、情報を入力するためのポップアップ フォームが表示されます。

![image-20230308181201-5.png](/static/img/3e6dae_image-20230308181201-5.png)

|  |  |
| --- | --- |
| 列/ラベル | 説明 |
| 名前 | 作成/編集する OU のカスタム名を入力します |
| 説明 | 作成/編集するOUの説明を入力します。 |
| ユーザー | 作成/編集している OU に追加するユーザーを選択します。 注: 多数のユーザーを追加できますが、追加できるのは 1 人ずつのみです |

## **使用例 2: 新しいロールの作成/編集**

ロールは権限のセットです。つまり、特定のセンター リソースへのアクセスと制御に必要な権限がロールに割り当てられます。

S1: メニューで [**管理**] を選択し、次に [**ロール管理**] タブを選択して、各ロールの分散化を確認します。

![image-20230308181201-6.png](/static/img/b29d3c_image-20230308181201-6.png)

S2: ページの上部にある [**新規作成**] ボタンをクリックして、新しいロールを作成します。

![image-20230308181201-7.png](/static/img/e66009_image-20230308181201-7.png)

S3: ポップアップ フォームが表示されますので、入力してください。

![image-20230308181201-8.png](/static/img/f6f778_image-20230308181201-8.png)

S4: ボックスにチェックを入れて新しいロールに権限を付与し、[**保存**] をクリックします。

|  |  |
| --- | --- |
| **権限** | **カテゴリ** |
| なし | ユーザーにはインターフェースのセンターにリソースが表示されません |
| 見る | ユーザーはリソースを表示できますが、操作することはできません |
| 編集 | ユーザーはリソースを表示および編集できます |
| 作成 | ユーザーは新しいリソースを表示および作成できます |
| 削除 | ユーザーはリソースを削除できます |

S5: 既存のロールを編集するには、[*アクション**] 列のロールに対応する [**編集**] ボタンをクリックします。

![image-20230308181201-9.png](/static/img/ea09db_image-20230308181201-9.png)

## **使用例 3: 新しいアカウントの表示/作成/編集**

S1: 左側のメニュー **>** で **管理** をクリックし、**ユーザー** タブを選択すると、以下のような詳細ページが表示されます。

![image-20230308181201-10.png](/static/img/b4e50b_image-20230308181201-10.png)

S2: ユーザーを表示するには、目のボタンをクリックします

![image-20230308181254-11.png](/static/img/c017de_image-20230308181254-11.png)

すると、以下のような画面が表示されます。

![image-20230308181254-12.png](/static/img/fcbddc_image-20230308181254-12.png)

|  |  |
| --- | --- |
| **コラム** | **説明** |
| ステータス | ユーザーのステータス。アクティブ化または非アクティブ化可能 |
| プロフィール | ユーザーに割り当てられた役割。デフォルトのロールは 3 つあります: ROLE\_ADMIN ROLE\_USER \* ROLE\_ROBOT ユーザーは複数のロールを持つことができます。各役割の権限の詳細については、「役割管理」セクションで説明します。 |

S3: アカウントを作成/編集するには、右側の **新規作成** をクリックします

![image-20230308181254-13.png](/static/img/fb55a2_image-20230308181254-13.png)

[**新規作成/編集**] ボタンをクリックすると、入力するフォームが表示されます。詳細は以下の表のとおりです。

![image-20230308181254-14.png](/static/img/23f3a5_image-20230308181254-14.png)

フォームに入力した後、[**保存**] ボタンをクリックして保存すると、ユーザーが作成されます。 [**キャンセル**] ボタンをクリックして、キューの作成をキャンセルすることもできます。

## **使用例 4: アカウントの削除**

ユーザーを削除するには、対応するユーザーの「削除」ボタンをクリックします。

![image-20230308181254-15.png](/static/img/915d9e_image-20230308181254-15.png)

確認のポップアップが表示されます。 [**削除**] を押して操作を完了します。

## **使用例 5: パッケージの表示/検索/アップロード**

S1: [**パッケージ**] ページにアクセスするには、左側のメニューの [**パッケージ**] タブをクリックします。クリックすると、センターに公開されたすべての**パッケージ**を含む**パッケージ一覧**ページが開きます。

![image-20230308181254-16.png](/static/img/c37388_image-20230308181254-16.png)

S2: パッケージを表示するには、ユーザーは **目** ボタンをクリックします。詳細ページには、センターにアップロード/公開された、選択したパッケージのすべてのバージョンを含むリストが表示されます。

![image-20230308181254-17.png](/static/img/e496af_image-20230308181254-17.png)

S3:パッケージを検索するには、ここにパッケージ名を入力します

![image-20230308181254-18.png](/static/img/fb88a6_image-20230308181254-18.png)

S4: パッケージをアップロードする

ワークフローをセンターに公開すると、パッケージが自動的にセンターにアップロードされます。ただし、[リスト] ページの右上にある [新規作成] ボタンを使用して、**手動でパッケージを Center にアップロード**することもできます。

![image-20230308181254-19.png](/static/img/d0b999_image-20230308181254-19.png)

パッケージをアップロードするためのウィンドウが表示されます。

![image-20230308181254-20.png](/static/img/aa3483_image-20230308181254-20.png)

## **使用例 8: エージェントの表示/作成/編集**

![image-20230308181334-27.png](/static/img/193af0_image-20230308181334-27.png)

## **使用例 9: ワークフローの表示/作成/編集**

S1: 左側のメニューで [自動化] を選択し、[**ワークフロー**] タブをクリックします。

![image-20230308181423-34.png](/static/img/08dc03_image-20230308181423-34.png)

S2: ワークフローを表示するには、目のボタンをクリックします。

![image-20230308181423-35.png](/static/img/2eaa4b_image-20230308181423-35.png)

タスクの**時刻、名前、状態、エージェント** でタスク リストを検索できます。

![image-20230308181423-36.png](/static/img/fc21a5_image-20230308181423-36.png)

S3: ワークフローを作成するには、ページの上部にある**「新規作成」** ボタンをクリックします。

![image-20230308181423-37.png](/static/img/1c7218_image-20230308181423-37.png)

S4: クリックすると、ワークフローを作成できるフォームが表示されます。

![image-20230308181423-38.png](/static/img/48ed6e_image-20230308181423-38.png)

|  |  |
| --- | --- |
| **権限** | **カテゴリ** |
| パッケージ名 | 作成しているワークフローに必要なパッケージを選択します |
| パッケージバージョン | 上記で選択したパッケージによって異なります。選択したパッケージのバージョンを選択します |
| エージェントグループ | 選択したパッケージのバージョンに応じて異なります。作成しているワークフローを接続するエージェント グループを選択します。 |
| 説明 | 作成/編集しているワークフローの説明を入力します |

S5: すべての一般情報を入力したら、さらに **パラメータ** および **マシン環境** 変数をワークフローに追加できます。複数の変数を追加できますが、追加できるのは 1 つずつのみです。

![image-20230308181423-39.png](/static/img/62d47d_image-20230308181423-39.png)

S6: ワークフローを編集するには、[**3 つの点**] ボタンをクリックしてから、[**編集**] をクリックします。

![image-20230308181423-40.png](/static/img/decfbe_image-20230308181423-40.png)

S7: クリックすると、ワークフローの情報を編集できるフォームが表示され、**保存**

![image-20230308181423-41.png](/static/img/74e802_image-20230308181423-41.png)

## **使用例 12: ボットの実行/停止**

1. ボットを手動で実行 (新しいタスクの作成、クローン)

**オプション 1:**

S1: 左側のメニューで [自動化] をクリックし、[**タスク**] タブをクリックします。

![image-20230308181710-49.png](/static/img/b6ae38_image-20230308181710-49.png)

**S2:** 手動で実行する必要があるワークフローを選択し、**保存**

![image-20230308181710-50.png](/static/img/949358_image-20230308181710-50.png)

**オプション 2:** *または*、タスクの横にある [アクション] 列で [**アクション**] をクリックし、ドロップダウン リストが表示されます。 > [**クローン**] を選択し、[**保存**] をクリックします。

![image-20230308181710-51.png](/static/img/1e574d_image-20230308181710-51.png)

2. Bot スケジュールの実行 (スケジュールの設定、スケジュールの無効化)

S1: メニューで **[オートメーション**] を選択し、**[スケジュール] タブをクリックし、** [**新規作成**] をクリックします。

![image-20230308181710-52.png](/static/img/af3a69_image-20230308181710-52.png)

S2: ポップアップフォームに記入します

|  |  |
| --- | --- |
| 
* **名前 (\*):** スケジュールの名前
* **ワークフロー (\*):** スケジュールを作成するワークフローを選択します
* **タイム ゾーン (\*):** スケジュールを適用するタイム ゾーンを選択します
**トリガータブ:**
* **繰り返し:** スケジュールのサイクルを選択します: 分/時間/毎日/毎週/毎月/事前
* **開始日 (\*):** スケジュールを開始する時刻を選択し、
* **終了日:** スケジュールが終了する時刻を選択します
**[実行ターゲット] タブ:** は選択したワークフローに応じて、適切なエージェントにチェックを入れます **[パラメータ] タブ:**
**[休日設定] タブ**: 単一の選択肢
* 継続的に実行 (デフォルト)
* バイパス休日
* 次の勤務日まで延期する

| オン | 月の正確な日を選択してください |
| の | 年の正確な月を選択してください |

**2.2.スケジュールの開始/停止:**

S1:**スケジュール**を強制的に開始/停止するには、**アクション**ボタンをクリックし、**今すぐ開始**ボタンを選択するか、スケジュールが実行中の場合は**停止**ボタンを選択します。

![image-20230308181710-54.png](/static/img/8febf6_image-20230308181710-54.png)

**2.3. スケジュールの無効化/有効化**

S1: スケジュールを無効にするには、対応する **アクション** ボタンをクリックし、**無効にする**を選択します。

![image-20230308181710-55.png](/static/img/7d3f64_image-20230308181710-55.png)

S2: クリックすると、システムはステータスを切り替えるための確認を求めます。

![image-20230308181710-56.png](/static/img/c8e577_image-20230308181710-56.png)

スケジュールを無効にすると、そのスケジュールのステータスが「**無効**」に変わります。

S3: スケジュールを有効にするには、対応する [**アクション**] ボタンをクリックし、[**有効にする**] を選択します。

![image-20230308181710-57.png](/static/img/c47303_image-20230308181710-57.png)

**2.4. スケジュールを削除します**

S1: 削除するには、対応するスケジュールの [アクション] ボタンをクリックし、[**削除**] ボタンを選択する必要があります。

![image-20230308181710-58.png](/static/img/500cf0_image-20230308181710-58.png)

2. ボットを停止する

S1: 実行中のボットを停止するには、[**タスク**] タブ > タスクの横にある [アクション] 列の [**アクション**] をクリック > ドロップダウン リストが表示されます > [**停止**] を選択します

![image-20230308181710-59.png](/static/img/d92652_image-20230308181710-59.png)

S2: システムは確認を求めるメッセージを表示します。 [**確認**] をクリックしてタスクを削除します。

![image-20230308181826-61.png](/static/img/a31523_image-20230308181826-61.png)
