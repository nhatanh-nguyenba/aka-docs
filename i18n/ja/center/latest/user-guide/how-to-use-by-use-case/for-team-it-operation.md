# チームIT運用のために

## **使用例 1: OU の作成/編集**

S1: **組織単位ページ**にアクセスするには、左側のメニューで [管理] タブをクリックし、次に [組織単位] タブをクリックします。

![image-20230425154231-1.png](/static/img/8c5b48_image-20230425154231-1.png)

S2: OU の詳細をすべて表示するには、**目** ボタンをクリックします。

![image-20230425154231-2.png](/static/img/f81b09_image-20230425154231-2.png)

すると詳細ページが表示されます

![image-20230425154231-3.png](/static/img/292f96_image-20230425154231-3.png)

|  |  |
| --- | --- |
| 列/ラベル | 説明 |
| 名前 | OUの名前 |
| ユーザー | この OU に割り当てられているすべてのユーザーのリスト。このリストには次のものが含まれます: *ユーザー名* 電子メール |

S3: OUを作成/編集するには

**「新規作成」** ボタンをクリックします。 OU を編集する場合は、対応する OU 名の横にある [アクション] から編集オプションを選択します。

![image-20230425154231-4.png](/static/img/e2261a_image-20230425154231-4.png)

S4: クリックすると、情報を入力するためのポップアップ フォームが表示されます。

![image-20230425154231-5.png](/static/img/f9b39f_image-20230425154231-5.png)

|  |  |
| --- | --- |
| 列/ラベル | 説明 |
| 名前 | 作成/編集する OU のカスタム名を入力します |
| 説明 | 作成/編集するOUの説明を入力します。 |
| ユーザー | 作成/編集している OU に追加するユーザーを選択します。 注: 多数のユーザーを追加できますが、追加できるのは 1 人ずつのみです |

## **使用例 2: OU を削除する**

**組織単位**メニューで、ページ上のOU名の横にある**削除**ボタンをクリックしてOUを削除します

***注意: OU を削除すると、ユニット内のすべてのリソースが削除されます***

![image-20230425154231-6.png](/static/img/030999_image-20230425154231-6.png)

## **使用例 3: 新しいロールの作成/編集**

ロールは権限のセットです。つまり、特定のセンター リソースへのアクセスと制御に必要な権限がロールに割り当てられます。

S1: メニューで [**管理**] を選択し、次に [**ロール管理**] タブを選択して、各ロールの分散化を確認します。

![image-20230425154231-7.png](/static/img/d44c49_image-20230425154231-7.png)

S2: ページの上部にある [**新規作成**] ボタンをクリックして、新しいロールを作成します。

![image-20230425154231-8.png](/static/img/44618b_image-20230425154231-8.png)

S3: ポップアップ フォームが表示されますので、入力してください。

![image-20230425154231-9.png](/static/img/71cfa0_image-20230425154231-9.png)

S4: ボックスにチェックを入れて新しいロールに権限を付与し、[**保存**] をクリックします。

|  |  |
| --- | --- |
| **権限** | **カテゴリ** |
| なし | ユーザーにはインターフェースのセンターにリソースが表示されません |
| 見る | ユーザーはリソースを表示できますが、操作することはできません |
| 編集 | ユーザーはリソースを表示および編集できます |
| 作成 | ユーザーは新しいリソースを表示および作成できます |
| 削除 | ユーザーはリソースを削除できます |

S5: 既存のロールを編集するには、[アクション] 列のロールに対応する [**編集**] ボタンをクリックします。

![image-20230425154231-10.png](/static/img/5d3552_image-20230425154231-10.png)

## **使用例 5: 新しいアカウントの表示/編集/作成**

S1: 左側のメニュー **>** で **管理** をクリックし、**ユーザー** タブを選択すると、以下のような詳細ページが表示されます。

![image-20230425154231-13.png](/static/img/c775ef_image-20230425154231-13.png)

S2: ユーザーを表示するには、目のボタンをクリックします

![image-20230425154231-14.png](/static/img/fe1abe_image-20230425154231-14.png)

すると、以下のような画面が表示されます。

![image-20230425154231-15.png](/static/img/39d7f9_image-20230425154231-15.png)

|  |  |
| --- | --- |
| **コラム** | **説明** |
| ステータス | ユーザーのステータス。アクティブ化または非アクティブ化することができます |
| プロフィール | ユーザーに割り当てられた役割。デフォルトのロールは 3 つあります: ROLE\_ADMIN ROLE\_USER \* ROLE\_ROBOT ユーザーは複数のロールを持つことができます。各役割の権限の詳細については、「役割管理」セクションで説明します。 |

S3: アカウントを作成/編集するには、右側の「新規作成」をクリックします

![image-20230425154231-16.png](/static/img/1ee1eb_image-20230425154231-16.png)

[**新規作成/編集**] ボタンをクリックすると、入力するフォームが表示されます。詳細は以下の表にあります。

![image-20230425154231-17.png](/static/img/6a800f_image-20230425154231-17.png)

フォームに入力した後、[**保存**] ボタンをクリックして保存すると、ユーザーが作成されます。 [**キャンセル**] ボタンをクリックして、キューの作成をキャンセルすることもできます。

ユーザーを削除するには、対応するユーザーの [**削除**] ボタンをクリックします。

![image-20230425154231-18.png](/static/img/7e5f3e_image-20230425154231-18.png)

確認のポップアップが表示されます。 [**削除**] を押して操作を完了します。

## **ユースケース 7: ホームページでセンターの概要を参照**

S1: 左側のメニューで、[ホームページ] を選択します。このページには、以下を含むすべての情報が表示されます。

* エージェントの数 (対応可能、ビジー、切断、無応答)
* タスクの数 (成功、実行中、停止、失敗)
* スケジュール（時間、日、週、月単位で計算）
* パッケージ (アクティブ、非アクティブ)
* タスク統計 (日次、週次、月次、年次ビューが利用可能)
* 今後のタスク
* 最近のタスク

S2: **フィルター** ボタンを使用して、エージェント グループ、エージェント名に基づいてタスクを表示できます。

![image-20230425154231-19.png](/static/img/3e7b8e_image-20230425154231-19.png)

S3: タスクをすばやく作成するには、この **「プラス」** ボタンをクリックします

![image-20230425154231-20.png](/static/img/455256_image-20230425154231-20.png)

## **使用例 9: エージェント グループの削除**

S1:

**オプション 1**: エージェント グループを削除するには、エージェント グループ リストから削除するエージェント グループの [削除] ボタンをクリックします。

![image-20230425154754-25.png](/static/img/84f09d_image-20230425154754-25.png)

**オプション 2**: 各エージェントの前にチェックボックスをオンにすると、上部のフィルター オプションの横に削除オプションが表示されます。 [アクション] の横にあるチェックボックスをクリックすると、削除するすべてのエージェントを選択できるようになります。

S2: アクションを確認するには > ポップアップ ウィンドウで **削除** を押します

![image-20230425154754-26.png](/static/img/a1d748_image-20230425154754-26.png)

## **使用例 10: エージェントのステータスを監視する**

S1: 左側のメニューで [リソース] をクリックし、[エージェント] タブを選択すると、エージェントの詳細が表示されます。

![image-20230425154754-27.png](/static/img/f53bee_image-20230425154754-27.png)

|  |  |
| --- | --- |
| **列/ラベル** | **説明** |
| **エージェントグループ** | エージェントが属するエージェント グループ。 |
| **名前** | エージェントの名前。エージェントの名前をクリックすると、その詳細が表示されます。 |
| **マシン名** | エージェントが登録されているマシンの名前。 |
| **マシンのユーザー名** | エージェントマシンへのログインに使用されるユーザー名 |
| **タイプ** | エージェントのタイプ。これは、開発、アシスタント、ステージング、実稼働のみです。エージェントの種類に応じてページをフィルタリングすることができます。 |
| **ステータス** | エージェントの現在のステータス: * **利用可能** - エージェントはセンターに接続されていますが、プロセスは実行されておらず、自由に使用できます。 * **ビジー** - エージェントはプロセスを実行しています。 * **切断** - エージェントとセンターは過去 10 秒間通信していません。 |
| **説明** | エージェントの説明 |
| **バージョン** | ローカル マシンにインストールされている akaBot Agent ツールのバージョン |
| **作成者** | エージェントを作成したユーザー。 |

## **使用例 12: エージェントの削除**

![image-20230425155109-35.png](/static/img/8c962d_image-20230425155109-35.png)

![image-20230425155109-37.png](/static/img/bd2dfd_image-20230425155109-37.png)

## **使用例 15: ボットの実行/停止**

1. ボットを手動で実行 (新しいタスクの作成、クローン)

**オプション 1:**

S1: 左側のメニューで [自動化] をクリックし、[**タスク**] タブをクリックします。

![image-20230425155109-46.png](/static/img/2e05ac_image-20230425155109-46.png)

S2: 手動で実行する必要があるワークフローを選択し、**保存**

![image-20230425155109-47.png](/static/img/d5c239_image-20230425155109-47.png)

**オプション 2:** または、タスクの横にある [アクション] 列で [**アクション**] をクリックし、ドロップダウン リストが表示されます。 > [**クローン**] を選択し、[**保存**] をクリックします。

![image-20230425155109-48.png](/static/img/1f0ac2_image-20230425155109-48.png) 

2. Bot スケジュールの実行 (スケジュールの設定、スケジュールの無効化)

**2.1.スケジュールの作成/編集:**

S1: メニューで **[オートメーション]** を選択し、**[スケジュール] タブ**をクリックし、[**新規作成**] をクリックします。

![image-20230425155109-49.png](/static/img/72b022_image-20230425155109-49.png)

S2: ポップアップフォームに記入します

|  |  |
| --- | --- |
| 
* **名前 (\*):** スケジュールの名前
* **ワークフロー (\*):** スケジュールを作成するワークフローを選択します
* **タイム ゾーン (\*):** スケジュールを適用するタイム ゾーンを選択します
**トリガータブ:**
* **繰り返し:** スケジュールのサイクルを選択します: 分/時間/毎日/毎週/毎月/事前
* **開始日 (\*):** スケジュールを開始する時刻を選択し、
* **終了日:** スケジュールが終了する時刻を選択します
**[実行ターゲット] タブ:** 選択したワークフローに応じて、適切なエージェントにチェックを入れます
**「パラメータ」タブ:**
**[休日設定] タブ**: 単一の選択肢
* 継続的に実行 (デフォルト)
* バイパス休日
* 次の勤務日まで延期する
***注: (\*) は必須です***

![image-20230425155109-50.png](/static/img/image-20230425155109-50.png)

2.2. **スケジュールの開始/停止**

B1:**スケジュール**を強制的に開始/停止するには、**アクション**ボタンをクリックし、**今すぐ開始**ボタンを選択するか、スケジュールが実行中の場合は**停止**ボタンを選択します。

![image-20230425155109-51.png](/static/img/acd650_image-20230425155109-51.png)

2.3.スケジュールを無効にする/有効にする:

S1: スケジュールを無効にするには、対応する **アクション** ボタンをクリックし、**無効にする**を選択します。

![image-20230425155109-52.png](/static/img/a28347_image-20230425155109-52.png)

S2: クリックすると、システムはステータスを切り替えるための確認を求めます。

![image-20230425155109-53.png](/static/img/2ad1e3_image-20230425155109-53.png)

スケジュールを無効にすると、そのスケジュールのステータスが「**無効**」に変わります。

S3: スケジュールを有効にするには、対応する [**アクション**] ボタンをクリックし、[**有効にする**] を選択します。

![image-20230425155109-54.png](/static/img/81bca4_image-20230425155109-54.png)

2.4. **スケジュールを削除する:**

B1: 削除するには、対応するスケジュールの [アクション] ボタンをクリックし、[**削除**] ボタンを選択する必要があります。

![image-20230425155109-55.png](/static/img/4110bb_image-20230425155109-55.png)

3. ボットを停止する

S1: 実行中のボットを停止するには、[**タスク**] タブ > タスクの横にある [アクション] 列の [**アクション**] をクリック > ドロップダウン リストが表示されます > [**停止**] を選択します

![image-20230425155109-56.png](/static/img/0b6352_image-20230425155109-56.png)

S2: システムは確認を求めるメッセージを表示します。 [**確認**] をクリックしてタスクを削除します。

![image-20230425155109-57.png](/static/img/2a65ce_image-20230425155109-57.png)
