# 担当者プール

担当者プールは、プロセス内でヒューマン タスクを実行する資格のあるユーザーの名前付きグループです。ヒューマン タスクを 1 人の個人に割り当てる代わりに、それをプールに割り当てることができ、そのプールのメンバーであれば誰でもタスクを要求して完了できます。このアプローチにより、特定の個人が不在、休暇中、または仕事が過負荷になったときに発生するボトルネックが防止され、プロセスの回復力が向上します。

担当者プールはシステム レベルで定義され、複数のプロセスおよびヒューマン タスクにわたって再利用されます。各プールには名前、オプションの説明、およびユーザー メンバーのリストがあります。メンバーシップは、管理者が手動で管理することも、組織のディレクトリ サービス (LDAP/Active Directory) と自動的に同期することもできます。ヒューマン タスクがプールに割り当てられると、すべてのプール メンバーが通知を受け取り、タスクが各メンバーの受信箱に表示されます。タスクを要求した最初のメンバーがそのタスクをロックし、重複した処理を防ぎます。

**負荷分散**戦略を使用してプールを構成することもできます。この場合、システムはアクティブな割り当てが最も少ないプール メンバーにタスクを自動的にルーティングします。これにより、作業が公平に分散され、全体的なタスクの完了時間が短縮されます。スーパーバイザーは、コラボレーション センター ダッシュボードから、平均処理時間や保留中のタスク数などのプール レベルの統計を表示できます。

## プール割り当て戦略

| 戦略 | 説明 | 最適な用途 |
|---|---|---|
| 最初の請求 | プールメンバーは誰でも自分の受信箱からタスクを要求できます | 一般的な承認タスク |
| ラウンドロビン | タスクはローテーションでプールメンバーに順番に割り当てられます | ワークロードの均等な分散 |
| 負荷ベース | タスクはアクティブなタスクが最も少ないメンバーにルーティングされます | 大量のキュー |
| スキルベース | タスクはメンバーのスキルタグに基づいてルーティングされます | 特殊な処理タスク |
| 手動オーバーライド | スーパーバイザーがタスクを特定のメンバーに手動で割り当てます | エスカレーションされたタスクまたは機密性の高いタスク |

## 担当者プールの作成

1. **ヒューマン ボット コラボレーション** > **プロセス設計** > **担当者プール** に移動します。
2. [**+新しいプール**] をクリックします。
3. **プール名**とオプションの**説明**を入力します。
4. ドロップダウンから **割り当て戦略** を選択します。
5. [**メンバー**] で [**メンバーの追加**] をクリックし、名前またはメールアドレスでユーザーを検索します。
6. 必要に応じて、**LDAP 同期** を切り替え、グループ識別名 (DN) を入力して、メンバーシップを自動的に同期します。
7. [**保存**] をクリックしてプールを作成します。

## ヒューマンタスクへのプールの割り当て

1. **プロセス デザイナー**でプロセスを開きます。
2. 構成する **ヒューマン タスク** ノードをクリックします。
3. プロパティ パネルで、**担当者** フィールドを見つけます。
4. 割り当てタイプとして **プール** を選択します。
5. [**プール名**] ドロップダウンから目的のプールを選択します。
6. [**保存**] をクリックして構成を適用します。
