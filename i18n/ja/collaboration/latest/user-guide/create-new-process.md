# 新しいプロセスを作成する

akaBot Center で新しいプロセスを作成することは、ボットと人間の参加者の両方が関与するビジネス ワークフローを自動化するための最初のステップです。プロセス デザイナーは、プロセスを構成する一連のタスク、意思決定ポイント、およびイベントを定義できるビジュアル キャンバスを提供します。フローを構築する前に、名前、説明、バージョン、ヒューマン タスクに関与する参加者などのプロセス メタデータを設定する必要があります。

明確に定義されたプロセスは、どのステップがボットによって完全に処理できるか、どのステップが人間の判断や承認を必要とするか、タスク間でどのようなデータが流れる必要があるかなど、ビジネス要件を理解することから始まります。 akaBot Center プロセス モデルは、ボット タスクとヒューマン タスクを自由に組み合わせ、論理ゲートウェイを介して接続して現実世界のビジネス ルールをモデル化できる混合ワークフローをサポートします。

プロセスが作成され、ドラフト状態で保存されると、運用用に公開する前に反復できます。公開されたプロセスは、オペレーターによって手動でトリガーしたり、特定の時間に実行するようにスケジュールしたり、API またはシステム イベントを介して自動的に呼び出したりすることができます。バージョン管理により、プロセスの更新によって、以前のバージョンで実行されている実行中のインスタンスが中断されないことが保証されます。

## プロセス作成手順

1. akaBot Center で **ヒューマン ボット コラボレーション** > **プロセス設計** に移動します。
2. プロセス リストの上部にある [**+ 新しいプロセス**] ボタンをクリックします。
3. **プロセスの作成** ダイアログで、次のフィールドに入力します。

   | フィールド | 説明 | 必須 |
   |---|---|---|
   | プロセス名 | プロセスの一意でわかりやすい名前 | はい |
   | 説明 | プロセスの内容の簡単な説明 | いいえ |
   | カテゴリ | プロセスの組織的なグループ化 | いいえ |
   | バージョンラベル | バージョン識別子 (例:`1.0.0`) | はい |
   | デフォルトの担当者プール | ヒューマンタスク用のデフォルトのユーザープール | いいえ |
   | 開始条件 | プロセスのトリガー方法 (手動、スケジュール、API) | はい |

4. [**作成**] をクリックして、デフォルトの開始イベントがすでに配置されたプロセス デザイナー キャンバスを開きます。
5. 左側の **タスク パレット** を使用して、ボット タスク、ヒューマン タスク、およびゲートウェイをキャンバスにドラッグします。
6. 接続矢印が表示されるまでタスク ノードの上にマウスを移動し、次のノードにドラッグしてタスクを接続します。
7. 各タスクをクリックして、右側のパネルでプロパティを編集して、各タスクを構成します。
8. 最後のタスクを **End Event** ノードに接続して、終了条件を定義します。
9. **ドラフトとして保存** をクリックして作業内容を保存するか、**公開** をクリックしてプロセスを実行できるようにします。

## ベストプラクティス

- プロセス名は簡潔ですが、ビジネス機能を識別できるように十分に説明的なものにしてください。
- 意味のある説明を追加すると、他のチーム メンバーがデザイナーを開かなくてもプロセスの目的を理解できるようになります。
- 後で再構成を避けるために、タスクを設計する前に参加者の役割と担当者プールを定義します。
- メイン キャンバスをクリーンな状態に保つために、タスクの繰り返しシーケンスにはサブプロセスを使用します。
- 組織全体に公開する前に、少人数のユーザー グループでプロセスをテストしてください。
