# デザインフォーム

akaBot Center のデザイン フォーム機能を使用すると、プロセス設計者は、ワークフローの実行中に人間の参加者に提示される対話型のデータ入力フォームを作成できます。これらのフォームは、プロセスが次のステップに進む前に、人間が提供する必要がある構造化された入力 (承認、データ修正、意思決定など) を収集します。フォームはドラッグ アンド ドロップ インターフェイスを使用して構築され、実際のビジネス要件に合わせてさまざまなフィールド タイプを含めることができます。

各フォームは、プロセス内の特定のヒューマン タスクに関連付けられます。ヒューマン タスクがアクティブになると、割り当てられたユーザーは通知を受け取り、入力を完了するためのフォームに誘導されます。フォームには、以前のプロセス ステップまたは統合されたビジネス システムからのデータを事前に入力できるため、必要な手動入力の量が減り、エラーのリスクが最小限に抑えられます。設計者は、各フィールドに検証ルールを適用して、送信前にデータの品質を確認できます。

フォームは条件付き表示をサポートしています。つまり、他のフィールドに入力された値に基づいて特定のフィールドを表示または非表示にできます。これにより、インターフェイスがクリーンで各ユーザーにとって適切な状態に保たれます。完成したフォーム データはプロセス変数として保存され、後続のタスク (人間が提供したデータを使用して下流で自動アクションを実行するボット タスクなど) から参照できます。

## サポートされているフィールドの種類

| フィールドタイプ | 説明 | 使用例の例 |
|---|---|---|
| テキスト入力 | 単一行の自由テキスト入力 | お客様のお名前、整理番号 |
| テキストエリア | 複数行の自由テキスト入力 | コメント、拒否理由 |
| 番号 | オプションの最小/最大検証を備えた数値入力 | 請求金額、数量 |
| ドロップダウン | 事前定義されたリストからオプションを 1 つ選択します | 部門、承認状況 |
| 複数選択 | リストから複数のオプションを選択します | 影響を受けるシステム、通知受信者 |
| 日付ピッカー | カレンダーに基づいた日付の選択 | 期限、発効日 |
| 日時ピッカー | 日付と時刻の選択 | 予約時間、締め切り |
| チェックボックス | ブール値の true/false 切り替え | 確認の確認 |
| ラジオボタン | 相互に排他的な選択 | 優先度（高/中/低） |
| ファイルのアップロード | 1 つ以上の文書を添付してください | 裏付けとなる証拠、スキャンされた請求書 |
| 隠しフィールド | ユーザーには見えない値を保存する | プロセスインスタンスID、システムコード |

## フォームを作成する手順

1. **プロセス デザイナー**でプロセスを開き、フォームを作成するヒューマン タスクを選択します。
2. タスクのプロパティ パネルで、[**デザイン フォーム**] をクリックしてフォーム ビルダーを開きます。
3. 左側の **フィールド パレット** から必要なフィールド タイプをフォーム キャンバスにドラッグします。
4. 各フィールドをクリックして [**プロパティ**] パネルを開き、以下を設定します。
   - **ラベル**: ユーザーに表示される表示名。
   - **フィールド名**: 値を保存するために使用される変数キー。
   - **必須**: 送信前にフィールドに入力する必要があるかどうかを切り替えます。
   - **デフォルト値**: フィールドに静的な値またはプロセス変数を事前に入力します。
   - **検証ルール**: 最小長、最大値、正規表現パターンなどの制約を設定します。
5. [**条件ロジック**] タブを使用して、他のフィールド値に基づいて表示/非表示ルールを追加します。
6. [**プレビュー**] をクリックして、フォームがエンド ユーザーにどのように見えるかを確認します。
7. [**フォームの保存**] をクリックして、フォームをヒューマン タスクに添付します。
