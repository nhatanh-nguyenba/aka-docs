# プロセス内でボット タスクを作成する方法

ボット タスクは、人間とボットのコラボレーション プロセス内の自動化されたステップであり、akaBot が人間の介入なしで事前定義されたロボット ワークフロー パッケージを実行します。ボット タスクは、ソース システムからのデータの読み取り、計算の実行、データベース内のレコードの更新、ドキュメントの生成など、高速で反復的な処理作業を担当します。ボット タスクをプロセス フローに直接埋め込むことで、組織は、大規模なエンドツーエンド ワークフロー内で、必要なときに必要な場所で自動化を正確に実行できるようになります。

プロセスがボット タスクに到達すると、akaBot Center は、割り当てられたエージェント グループ内の利用可能なエージェントにジョブをディスパッチします。エージェントは、指定されたワークフロー パッケージをダウンロードし、構成された入力パラメーターを使用して実行し、結果 (成功、失敗、または返された出力値) をプロセス エンジンに報告します。その後、プロセスは結果に基づいて次のタスクに進み、ボットの結果に基づいた条件分岐が可能になります。

ボット タスクは、一時的なエラーを適切に処理するために、構成可能なタイムアウトと再試行ポリシーをサポートしています。ボット タスクが失敗し、再試行が限界に達した場合、例外処理パスにルーティングするか、スーパーバイザに警告するか、インスタンスを終了するようにプロセスを構成できます。開始時刻、終了時刻、使用されたエージェント、および出力変数を含むすべての実行の詳細は、インスタンス監査ログに記録されます。

## ボットタスク構成パネル

| プロパティ | 説明 | 必須 |
|---|---|---|
| タスク名 | デザイナーでのこのタスクの表示名 | はい |
| エージェントグループ | このタスクを実行できるエージェントのグループ | はい |
| ワークフローパッケージ | 実行する akaBot ワークフロー パッケージ (.nupkg) | はい |
| パッケージバージョン | 特定のバージョンまたは「最新」 | はい |
| 入力変数 | 入力引数としてパッケージに渡すプロセス変数 | いいえ |
| 出力変数 | パッケージ出力からキャプチャする変数 | いいえ |
| タイムアウト (分) | タスクがタイムアウトとしてマークされるまでの最大実行時間 | はい |
| リトライ回数 | 失敗時の自動再試行回数 | いいえ |
| 再試行遅延 (秒) | 再試行間の待ち時間 | いいえ |
| 失敗時 | エラーパスにルーティングするか、インスタンスを終了します | はい |

## ボットタスクを追加する手順

1. **プロセス デザイナー**でプロセスを開きます。
2. 左側の **タスク パレット**で、**Bot Task** 要素をキャンバスにドラッグします。
3. フロー矢印を使用して、ボット タスクを前の要素に接続します。
4. Bot Task ノードをクリックして、右側の [**プロパティ**] パネルを開きます。
5. わかりやすい **タスク名** を入力します。
6. ドロップダウンから **エージェント グループ** を選択します。少なくとも 1 つのアクティブなエージェントを持つグループのみがリストされます。
7. [**ワークフロー パッケージ**] フィールドをクリックし、実行するパッケージを参照または検索します。
8. 目的の **パッケージ バージョン**を選択します。
9. **入力変数** セクションで、プロセス変数をパッケージの予期される入力パラメーターにマップします。
10. **出力変数** セクションで、パッケージの戻り値を格納する変数名を定義します。
11. 予想される実行時間に適切な **タイムアウト** 値を設定します。
12. 一時的なエラーが予想される場合は、**再試行回数**と**再試行遅延**を構成します。
13. [**保存**] をクリックして構成を適用します。
