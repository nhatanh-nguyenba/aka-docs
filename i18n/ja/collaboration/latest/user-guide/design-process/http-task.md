# プロセス内で HTTP タスクを作成する方法

akaBot Center の HTTP タスクを使用すると、ボット エージェントが統合を処理する必要がなく、プロセスがプロセス エンジンから外部 REST API エンドポイントを直接呼び出すことができます。これは、Webhook 通知の送信、外部システムからの参照データの取得、サードパーティ サービスへのフォームの送信、別のプラットフォームでのアクションのトリガーなどの軽量の統合に役立ちます。 HTTP タスクは、すべての標準 HTTP メソッドをサポートし、JSON とフォームエンコードされたリクエスト本文の両方を処理できます。

HTTP タスクはプロセス フロー内で同期的に実行されます。プロセスは HTTP タスクで一時停止し、構成された要求を送信し、応答を待ってから続行します。ステータス コード、応答ヘッダー、本文を含む応答データはプロセス変数に取り込まれ、後続のタスクで使用できます。 HTTP 呼び出しが失敗した場合 (ネットワーク エラー、2xx 以外のステータス コード)、タスクを再試行するか、エラー パスにルーティングするように設定できます。

安全な API の場合、HTTP タスクは、API キーの挿入 (ヘッダーまたはクエリ パラメーター経由)、ベアラー トークン、および基本認証を含むいくつかの認証メカニズムをサポートします。トークンやパスワードなどの機密の資格情報は、ログやプロセス デザイナー UI での公開を防ぐために、タスク構成にハードコーディングするのではなく、安全なプロセス定数として保存する必要があります。

## HTTP タスク構成パネル

| プロパティ | 説明 | 必須 |
|---|---|---|
| タスク名 | このタスクの表示名 | はい |
| URL | 完全なエンドポイント URL (変数補間をサポート) | はい |
| HTTPメソッド | 取得、投稿、配置、パッチ、削除 | はい |
| ヘッダー | リクエストヘッダーのキーと値のペア | いいえ |
| クエリパラメータ | URL に追加されるキーと値のペア | いいえ |
| 認証タイプ | なし、API キー、ベアラー トークン、基本認証 | いいえ |
| リクエストボディタイプ | なし、JSON、フォームデータ、生 | いいえ |
| リクエストボディ | 本文のコンテンツまたはテンプレート | いいえ |
| 応答状態変数 | HTTPレスポンスのステータスコードを格納する変数 | いいえ |
| レスポンスボディ変数 | 解析されたレスポンスボディを格納する変数 | いいえ |
| タイムアウト (秒) | 応答までの最大待ち時間 | はい |
| リトライ回数 | 失敗時の再試行回数 | いいえ |
| 失敗時 | エラーパスまたは終了 | はい |

## HTTP タスクを追加する手順

1. **プロセス デザイナー**でプロセスを開きます。
2. **HTTP タスク** 要素をタスク パレットからキャンバスにドラッグします。
3. プロセス フロー内で接続します。
4. HTTP タスク ノードをクリックして、**プロパティ** パネルを開きます。
5. **タスク名**を入力します。
6. **URL** フィールドにエンドポイントを入力します。プロセス変数を埋め込むには、`${variableName}`構文を使用します (`https://api.example.com/invoices/${invoiceId}`など)。
7. **HTTP メソッド** を選択します。
8. 必要な **ヘッダー** (例:`Content-Type: application/json`) を追加します。
9. API に認証が必要な場合は、**認証タイプ** を選択し、資格情報またはトークン変数の参照を指定します。
10. POST/PUT/PATCH メソッドの場合は、**リクエスト本文タイプ**を選択し、リクエスト本文テンプレートを入力します。
11. **レスポンス ステータス変数** と **レスポンス本文変数** をプロセス変数名にマップします。
12. 適切な **タイムアウト** 値を設定します。
13. **再試行回数**と**失敗時**ルーティングを構成します。
14. [**保存**] をクリックします。

## 例: Webhook の呼び出し

```json
URL: https://hooks.example.com/services/notify
Method: POST
Headers:
  Content-Type: application/json
Body:
{
  "processId": "${instanceId}",
  "status": "${approvalStatus}",
  "message": "Process step completed"
}
```
