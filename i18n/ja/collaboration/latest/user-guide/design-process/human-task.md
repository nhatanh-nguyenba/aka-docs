# プロセス内でヒューマン タスクを作成する方法

ヒューマン タスクは、ワークフローを続行する前に人がアクションを実行する必要があるプロセス ステップです。一般的な人間のタスクには、ドキュメントのレビューと承認、欠落データの入力、ビジネス コンテキストに基づく意思決定、または通知の確認が含まれます。ヒューマン タスクは、完全に自動化されたボット処理と、依然として人間の関与が必要な判断ベースのアクティビティとの間のギャップを橋渡しし、akaBot Center におけるヒューマン ボット コラボレーションの中核を形成します。

プロセスがヒューマン タスクに到達すると、プロセスは一時停止し、割り当てられたユーザー、ロール、または担当者プールに通知を送信します。担当者は受信トレイからタスクを開き、前のボット タスクによって提供されたコンテキスト データまたはドキュメントを確認し、必要なフォーム フィールドに記入して応答を送信します。送信後にのみ、プロセスは次のステップに進みます。担当者が設定された期限内に応答しない場合、エスカレーションをトリガーしてスーパーバイザーに通知するか、タスクを再割り当てすることができます。

ヒューマン タスクは、以前のボット タスクによって抽出されたデータなど、担当者に表示されるリッチ コンテキスト データを使用して構成できます。フォーム デザイナを使用して作成したカスタム フォームをタスクに添付できるため、担当者は構造化された入力を行うことができます。ヒューマン タスクの結果 (「承認」、「拒否」、カスタム ステータスなど) はプロセス変数として保存され、後続の分岐の決定を行うことができます。

## ヒューマンタスク構成パネル

| プロパティ | 説明 | 必須 |
|---|---|---|
| タスク名 | 担当者の受信箱に表示される表示名 | はい |
| 割り当ての種類 | ユーザー、ロール、またはプール | はい |
| 譲受人 | 特定のユーザー、ロール名、またはプール名 | はい |
| 優先順位 | 低、中、高、または重大 | いいえ |
| 期限 | 固定日付、またはプロセスの開始から計算 (例: +2 日) | いいえ |
| フォーム | 担当者が記入する必要があるフォーム | いいえ |
| コンテキストデータ | 読み取り専用コンテキストとして表示するプロセス変数 | いいえ |
| エスカレーションルール | 期限を過ぎた場合の対処法 | いいえ |
| 完了結果 | 考えられる結果を定義します (承認、拒否など)。 | はい |

## ヒューマンタスクを追加する手順

1. **プロセス デザイナー**でプロセスを開きます。
2. **ヒューマン タスク** 要素を **タスク パレット** からキャンバスにドラッグします。
3. ヒューマン タスクを前の要素に接続します。
4. ヒューマン タスク ノードをクリックして、**プロパティ** パネルを開きます。
5. 必要なアクションを伝えるわかりやすい **タスク名** を入力します (例: 「請求書の確認」)。
6. **割り当てタイプ**を**ユーザー**、**ロール**、または**プール**に設定します。
7. [**担当者**] フィールドで、ターゲット ユーザー、ロール、またはプールを選択します。
8. **優先度** レベルを設定して、担当者が緊急性を判断できるようにします。
9. **期限**を構成します — 固定日付または動的オフセット (例: プロセス開始 + 48 時間) のいずれかです。
10. [**デザイン フォーム**] をクリックして、担当者が入力するフォームを添付または作成します。
11. **コンテキスト データ** に、担当者が参照するプロセス変数を追加します (読み取り専用)。
12. **エスカレーション ルール**を定義します。エスカレーション アクション (再割り当て、マネージャーに通知) を選択し、トリガーの遅延を設定します。
13. **完了結果** で、有効な送信結果をすべて追加し、それぞれをプロセス変数値にマップします。
14. [**保存**] をクリックして構成を適用します。
