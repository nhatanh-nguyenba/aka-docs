# プロセス内の最終結果タスクを作成する方法

akaBot Center の最終結果タスクは、ボット タスク、HTTP タスク、ヒューマン タスクのいずれであっても、最後に完了したタスクの出力をキャプチャして名前付きプロセス変数に保存するユーティリティ タスクです。これにより、すべての出力フィールドを手動でマッピングしなくても、タスク間で結果を簡単に受け渡すことができます。これは、最小限の構成で前のタスクの出力を下流に転送する必要がある動的プロセスやデータ駆動型のプロセスで特に役立ちます。

最終結果タスクは、各タスクの出力が完了時に一時的に保存される内部プロセス実行コンテキストから読み取ることによって機能します。この出力を名前付き変数に明示的に永続化することにより、設計者は、後続の多くのタスクが実行された後でも、残りのプロセス存続期間中、値にアクセスできる状態を保つことができます。最終結果タスクがないと、生の最終結果コンテキストが次のタスクの実行によって上書きされる可能性があります。

最終結果タスクは、複雑な JSON 応答を返す HTTP タスクの後に使用すると特に強力です。このタスクは、JSON 本文から特定のフィールドを抽出し、厳密に型指定されたプロセス変数に割り当てることができ、ゲートウェイの条件式で使用したり、後続のボット タスクへの入力として使用したりできます。

## 最終結果タスク構成パネル

| プロパティ | 説明 | 必須 |
|---|---|---|
| タスク名 | このタスクの表示名 | はい |
| ソース | 出力ソース: 前回のタスクの結果、特定のタスクの出力 | はい |
| ソースタスク | Sourceが「Specific Task Output」の場合、タスクを選択します | 条件付き |
| 出力パス | 結果から抽出する JSONPath またはフィールド名 (例:`$.data.id`) | いいえ |
| ターゲット変数 | 抽出された値を保存するプロセス変数名 | はい |
| データ型 | 抽出された値の予期されるデータ型 | はい |
| デフォルト値 | ソース出力が null または欠落している場合に使用する値 | いいえ |

## JSONPath の抽出例

| ソース出力(JSON) | JSONパス式 | 抽出された値 |
|---|---|---|
| `{"status": "approved", "id": 42}` | `$.id` | `42` |
| `{"items": [{"name": "A"}, {"name": "B"}]}` | `$.items[0].name` | `"A"` |
| `{"result": {"code": "OK"}}` | `$.result.code` | `"OK"` |
| プレーンテキスト応答 | _(空白のまま)_ | 文字列としての応答全体 |

## 最終結果タスクを追加する手順

1. **プロセス デザイナー**でプロセスを開きます。
2. **最後の結果タスク** 要素をタスク パレットからキャンバスにドラッグします。
3. 出力をキャプチャしたいタスクの直後に配置します。
4. フロー内の最後の結果タスクを接続します。
5. [最後の結果タスク] ノードをクリックして、[**プロパティ**] パネルを開きます。
6. **タスク名** (例: 「キャプチャ API 応答 ID」) を入力します。
7. **ソース**を**最後のタスク結果**に設定して直前のタスクからキャプチャするか、**特定のタスク出力**を設定して名前付きの以前のタスクをターゲットにします。
8. 該当する場合は、JSONPath 構文を使用して **出力パス** を入力し、特定のフィールドを抽出します。
9. 値が保存される **ターゲット変数** の名前を入力します。
10. 正しい**データタイプ**を選択してください。
11. 必要に応じて、ソースが空の場合の **デフォルト値** を設定します。
12. [**保存**] をクリックします。
