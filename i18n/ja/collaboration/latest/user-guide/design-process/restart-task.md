# プロセス内で再起動タスクを使用する方法

再起動タスクは、プロセス インスタンスが以前に実行されたタスクまたはシーケンスの先頭にループバックできるようにする akaBot Center の特別な制御要素です。これは、特定の条件が満たされるまで一連のタスクを繰り返す必要がある反復的なビジネス シナリオをモデル化するために使用されます。たとえば、レビュー担当者が修正を要求し、最終承認のために再提出する前に改訂のためにドキュメントを返送できるドキュメント レビュー サイクルなどです。

単純なゲートウェイ ループとは異なり、再起動タスクはインスタンスがプロセス フローのどのポイントに戻るかを明示的に制御するため、プロセス図の理解と保守が容易になります。また、構成可能なループ制限を適用して、無限ループによるシステム リソースの消費を防ぎます。終了条件が満たされないままループ制限に到達すると、プロセスは例外パスに誘導されるか、エラーで終了する可能性があります。

再起動タスクは、品質チェック、反復的なデータ強化、または複数ラウンドのネゴシエーションを含むプロセスで特に価値があります。これらをヒューマン タスクと組み合わせて、人間の決定 (「再作業のリクエスト」など) によって再起動をトリガーしたり、ボット タスクと組み合わせて、プロセスを進める前に出力が必要な基準を満たしているかどうかを検証ボットが判断したりできます。

## 再起動タスクの構成オプション

| プロパティ | 説明 | 必須 |
|---|---|---|
| タスク名 | このリスタートポイントの表示名 | はい |
| 対象タスク | プロセスがループバックするタスクまたはゲートウェイ | はい |
| ループ条件 | ループを実行するには true でなければならない式 | はい |
| 最大ループ数 | ループが許可される最大回数 | はい |
| 制限を超えています | 最大数に達した場合のアクション: エラー パスまたは終了 | はい |
| ループカウンター変数 | 現在のループ数を追跡するプロセス変数 | いいえ |

## 一般的な使用例

| シナリオ | ループ条件 | 終了条件 |
|---|---|---|
| 文書の改訂 | 人間のレビュー担当者が「再作業のリクエスト」を選択 | 人間のレビュー担当者が「承認」を選択 |
| 部分データに対するボットの再試行 | ボット出力の検証が失敗する | ボットの出力が検証に合格する |
| 複数ラウンドの承認 | 上級承認者が修正のために送り返す | 上級承認者が最終承認を与える |
| データの強化 | 必須フィールドがまだ空です | すべての必須フィールドが入力されています |

## 再起動タスクを追加する手順

1. **プロセス デザイナー**でプロセスを開きます。
2. プロセスをループバックさせたいタスク (**ターゲット タスク**) を特定します。
3. **タスクの再開** 要素をタスク パレットからキャンバスにドラッグし、ループをトリガーするタスクの後に配置します。
4. トリガータスク (またはゲートウェイ) からタスク再起動ノードまでのシーケンス フローを描画します。
5. [タスクの再起動] ノードをクリックして、[**プロパティ**] パネルを開きます。
6. 再起動アクションを説明する **タスク名** を入力します (例: 「レビューに戻る」)。
7. **ターゲット タスク** で、プロセスが戻るタスクをドロップダウンから選択します。
8. **ループ条件** 式を入力します。ループを実行する必要がある場合、この式は`true`と評価されます。
9. **最大ループ数** を設定します (推奨: ほとんどのシナリオでは 5 ～ 10)。
10. **制限を超えた場合**の動作を構成します。
11. 必要に応じて、**ループ カウンター変数** 名を指定して、反復回数を後続のタスクに公開します。
12. [**保存**] をクリックします。
