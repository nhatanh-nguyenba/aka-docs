# プロセスの設計後に保存する方法

作業内容を akaBot Center Process Designer に正しく保存することは、設計の整合性を維持し、チーム メンバー間のコラボレーションを可能にし、操作の実行に使用できるプロセスのバージョンを制御するために不可欠です。デザイナーは 2 つの保存モードを提供します。進行中の作業を保存するための **ドラフト保存** と、完了し検証されたプロセスを実行できるようにするための **公開** です。これらのモードの違いを理解し、適切に使用することで、不完全なプロセスが誤って展開されることを防ぎます。

akaBot Center は、編集中に数分ごとに作業のドラフト スナップショットを定期的に保存する **自動保存** 機能も提供します。自動保存は、ブラウザのクラッシュや誤ったページ ナビゲーションによるデータの損失を防ぎます。ただし、自動保存されたドラフトは他のユーザーには表示されず、新しいバージョン エントリは作成されません。明示的に保存して公開した場合にのみ、監査可能なバージョン レコードが作成されます。

akaBot Center でのバージョン管理とは、明示的に保存するたびに、タイムスタンプと保存したユーザーの名前を持つ新しいバージョン レコードが作成されることを意味します。古いバージョンは保持されており、いつでも表示または復元できます。これにより、問題が発見された場合に公開プロセスへの変更をいつでもロールバックでき、実行中のプロセス インスタンスはその後の変更の影響を受けることなく、開始時のバージョンで実行を継続できるようになります。

## 保存状態の説明

| 州 | 説明 | オペレータに表示される | 新しいインスタンスをトリガーする |
|---|---|---|---|
| ドラフトの自動保存 | 定期的なバックグラウンド保存。現在の編集者にのみ表示されます | いいえ | いいえ |
| ドラフト保存 | ユーザーによる明示的な保存。バージョンレコードを作成します | いいえ (プレビューのみ) | いいえ |
| 発行済み | プロセスが完了し、運用用にリリースされる | はい | はい |
| アーカイブ済み | プロセスはリタイアされ、アクティブなプロセスのリストから非表示になります | いいえ | いいえ |

## プロセスをドラフトとして保存する手順

1. **プロセス デザイナー**での作業中に、ツールバーの**ドラフトとして保存** をクリックします。
2. **バージョンの保存** ダイアログが表示されます。
3. 加えられた変更を説明するオプションの **バージョンメモ** を入力します (例: 「請求書承認タスクにエスカレーション ルールを追加しました」)。
4. [**保存**] をクリックします。
5. バージョンは記録され、デザイナーはさらに編集できるように開いたままになります。

## プロセスを公開する手順

1. プロセス設計を完了し、すべてのタスクが検証警告なしで適切に接続されていることを確認します。
2. ツールバーの **検証** ボタンをクリックして、公開前チェックを実行します。報告されたエラーをすべて解決します。
3. ツールバーの**公開**をクリックします。
4. **プロセスの公開** ダイアログで:
   - **バージョン ラベル** (例:`2.1.0`) を確認します。
   - このバージョンでの変更点を説明する **リリース ノート** を入力します。
   - 必要に応じて、[**関係者に通知**] を選択して、プロセス所有者グループに電子メール通知を送信します。
5. [**公開の確認**] をクリックします。
6. プロセスのステータスが **公開済み** に変わり、オペレーターとスケジューラが使用できるようになります。

## バージョン履歴

- バージョン履歴にアクセスするには、プロセス デザイナー ツールバーの **バージョン履歴** をクリックします。
- 各行には、バージョン番号、保存日、保存者、およびバージョンに関するメモが表示されます。
- [**表示**] をクリックして、過去のバージョンの読み取り専用スナップショットを開きます。
- [**復元**] をクリックして、以前のバージョンに基づいて新しいドラフトを作成します。
