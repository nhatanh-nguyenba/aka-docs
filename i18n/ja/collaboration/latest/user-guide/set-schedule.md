# スケジュールを設定する

akaBot Center のスケジュールの設定機能を使用すると、手動介入なしで、特定の時間に自動的に開始するか、定期的にプロセスを開始するように構成できます。スケジュール設定は、バッチ プロセス、一日の終わりの調整、定期的なレポートの生成、またはビジネス サイクルに合わせて一定の間隔で実行する必要があるワークフローに特に役立ちます。

スケジュールは、一般的なパターン (毎日、毎週、毎月) の単純なタイムピッカー インターフェイス、または高度なスケジュール要件の cron 式のいずれかを使用して定義されます。単一のプロセスに複数のスケジュールをアタッチできるため、プロセスを平日の午前 8 時に実行し、さらに毎月末日にプロセスを実行するなど、複雑なトリガー パターンが可能になります。スケジュールされた実行ごとに独立したプロセス インスタンスが作成されるため、同時実行が完全にサポートされます。

スケジュール エンジンはタイム ゾーンを尊重するため、グローバル組織は各地域の正しい現地営業時間にプロセスをトリガーできます。スケジュール ステータスは、プロセス自体を変更することなくいつでも有効または無効にできるため、メンテナンス期間や休日中に自動実行を一時的に簡単に停止できます。

## スケジュール設定オプション

| オプション | 説明 |
|---|---|
| スケジュール名 | このスケジュールトリガーのわかりやすい名前 |
| プロセス | トリガーされる公開プロセス |
| 周波数 | 1 回限り、1 分ごと、1 時間ごと、毎日、毎週、毎月、またはカスタム cron |
| 開始日 | スケジュールがアクティブになる日付 |
| 終了日 | スケジュールのトリガーが停止する日付 (オプション) |
| タイムゾーン | スケジュール時刻の解釈に使用されるタイムゾーン |
| 最大同時実行数 | このスケジュールで許可される同時インスタンスの最大数 |
| 入力変数 | 開始時に渡されるプロセス入力変数のデフォルト値 |

## スケジュールを設定する

1. **ヒューマンボット コラボレーション** > **プロセス設計** を開き、**スケジュール** タブに移動します。
2. [**+新しいスケジュール**] をクリックします。
3. **スケジュール名**を入力し、ドロップダウンからターゲットの**プロセス**を選択します。
4. **頻度** オプションを選択します。
   - 単純なパターンの場合は、**ビジュアル ピッカー** (例: 毎日 09:00) を使用します。
   - 高度なパターンの場合は、**Cron 式の使用** を切り替えて、有効な cron 文字列 (例: 平日午前 9 時の`0 9 * * 1-5`) を入力します。
5. **開始日**を設定し、必要に応じて**終了日**を設定します。
6. 適切な**タイムゾーン**を選択します。
7. オプションで、プロセスがスケジュールされた実行ごとに受け取る **入力変数** を指定します。
8. スケジュールを **有効** に切り替えます。
9. [**保存**] をクリックします。

## 一般的な cron 式の例

```
# Every day at 8:00 AM
0 8 * * *

# Every Monday at 9:00 AM
0 9 * * 1

# Every weekday (Mon-Fri) at 6:00 PM
0 18 * * 1-5

# First day of every month at midnight
0 0 1 * *

# Every 30 minutes
*/30 * * * *
```
