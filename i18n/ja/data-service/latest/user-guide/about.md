# データサービスについて

Data Service は、ロボティック プロセス オートメーション (RPA) プロジェクトにコード不要の強力なデータ モデリングとストレージ機能をもたらす永続的なデータ ストレージ サービスです。

Data Service を使用すると、akaBot 内でビジネス データを安全に保存および管理できます。これでできることは次のとおりです。

1. データの収集: 自動プロセスを実行して、さまざまなシステムからデータを収集し、他のシステムでさらに処理できるように準備します。
2. データの活用: 収集したデータは、自動タスクと手動タスクの間でデータを転送する複雑なビジネス プロセスに使用します。
3. Studio および Agent との統合: 次のような機能を使用して、Data Service を自動化ワークフローにシームレスに組み込みます。

* 高度なデータ型: リレーショナル形式で構造化データを操作し、データをより効果的に整理および操作できるようにします。
* セキュリティの強化: 組み込みのセキュリティ機能を活用して、データの安全性とプライバシーを確保します。
* 迅速なプロビジョニングと展開: データ サービスを迅速にセットアップして展開し、自動化プロジェクトですぐに使用できるようにします。

## **II.使用例**

以下は、データ サービスの一般的な使用例のリストです。

**1.従業員のニーズに関するデータの収集**

説明: 従業員のニーズ (机、椅子、ウェブカメラなど) に関するデータを収集するための新しいプロセスを作成する必要があります。

問題: データは膨大になる可能性があるため、調達システムでリクエストを作成する前に保存する必要があります。

解決策: データベース、ファイル共有、またはクラウド ストレージを使用する代わりに、それらを Data Service に保存できます。

**2.月次財務報告**

説明: 月次財務報告のプロセスは、予約、請求書、請求、支払いなどの複数のシステムからデータを取得します。

問題: 有人ワークフローと無人ワークフローの両方が、実行されるたびにこのデータを取得します。これには、すべてのユーザーがすべてのバックエンド システムにアクセスして接続できる必要があります。これにより、すべてのプロセスが完了する前に基礎となるデータが変更される可能性があるため、レポートの遅延や不一致が発生する可能性があります。

解決策: Data Service で必要なすべてのデータを収集する無人ワークフローを 1 つ作成できます。そうすれば、他のすべての有人ワークフローまたは無人ワークフローを使用できるようになります。これにより、集約された重要なビジネス データをユーザーまたはロボット マシン上のファイルに保存する必要もなくなる可能性があります。
