# アクセスの管理

## **概要**

アクセス管理モジュールを使用すると、Data Service 内の特定のエンティティに対するアクセス許可を監視できます。この包括的かつ詳細な権限フレームワークにより、すべてのビジネス ユーザーをサービスにシームレスに組み込むことができ、各ユーザーの個々の習熟度レベルを考慮し、特定のビジネス ニーズに合わせることができます。さらに、組織からユーザーまたはエージェントを厳選し、それらに関連する権限を割り当てることができる柔軟性もあります。

**注:** デフォルトでは、組織ユーザーには Data Service 上の何も表示する権限がありません。

akaBot Center 管理者または**管理者権限**を持つユーザーのみが、アクセス権限を付与し、データ サービス上のデータを表示する権限を持っています。

次の手順により、Data Service へのアクセスを管理できます。

1. メインメニューの**データサービス**にアクセスします
2. [**アクセスの管理**] タブを選択します。 [**アクセスの管理**] ページが表示されます。

次のタブが使用可能です。

* [Assign User](/bin/view/Data%20Service/Getting%20Started/Manage%20Access/Assign%20Users/)
* [Assign Agents](/bin/view/Data%20Service/Getting%20Started/Manage%20Access/Assign%20Agents/)
* [Assign Tokens](/bin/view/Data%20Service/Getting%20Started/Manage%20Access/Assign%20Tokens/)

## **標準の権限**

### 管理者の権限

以下は、**管理者権限** の説明です。

| 許可 | 詳細な説明 |
| --- | --- |
| ユーザーと権限の管理 | * データ サービスにアクセスするために 1 人以上のユーザー、エージェント、トークンを割り当てることができます * 他のユーザーの権限を編集できます |

### エンティティの権限

以下は、**エンティティ権限** の説明です。

| 許可 | 詳細な説明 |
| --- | --- |
| エンティティの表示 | エンティティとそのフィールドのリストを表示できます |
| エンティティのカスタマイズ | エンティティとそのフィールドを表示、編集、削除できます |

### データ権限

以下は、**データ権限** の説明です。

| 許可 | 詳細な説明 |
| --- | --- |
| 作成 | すべてまたは特定のエンティティにデータを作成できる |
| 読む | すべてまたは特定のエンティティのデータを表示できます |
| 編集 | すべてまたは特定のエンティティのデータを編集および表示できます |
| 削除 | すべてまたは特定のエンティティのデータを削除できます |
