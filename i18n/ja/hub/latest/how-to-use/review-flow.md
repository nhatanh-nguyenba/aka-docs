## レビューの流れ
共有方法に応じて、アイデアはさまざまなレビュー段階を経ます
## 1. アイデアを検討する
**a.アイデア承認者がアイデアをレビューします**

アイデアが akaBot Hub と共有された後、アイデア承認者は **ワークスペース -> 自動化パイプライン** にアクセスしてアイデアをレビューし、アイデアを承認、重複としてマーク、または拒否することができます。承認後のアイデアは、「レビュー待ち」ステータスで評価フェーズに移行し、**ワークスペース -> 自動化パイプライン** でアクセスできます。

![image-20230223092408-1.png](/static/img/image-20230223092408-1.png)

> **注意:** アイデアを承認する前に、そのアイデアがプロセス所有者に割り当てられているかどうかを確認してください。そうでない場合は、詳細アイデアにアクセスし、「編集」に移動してアイデアの一番下までスクロールしてプロセス所有者を割り当ててください。

**b.プロセス所有者はアイデアの詳細情報を入力します**

アイデアが承認されると、そのアイデアに割り当てられたプロセス所有者は、詳細な評価の完了を要求する評価招待状を受け取ります。詳細な評価タスクは、プロセス所有者の未完了 - 私のタスクに送信されます。一方、プロセス所有者は、ワークスペースの自動化パイプラインでレビューが必要なアイデアを見つけることができます。プロセス所有者が [評価の送信] をクリックすると、アイデアのフェーズが認定フェーズに更新され、ステータスがレビュー待ちに変わります。

![image-20230223093939-2.gif](/static/img/image-20230223093939-2.gif)

**c.プログラム マネージャーはアイデアを検討し、さらに実装フェーズに進むかどうかを決定します**

アイデアの実装に必要な作業の複雑さに基づいて、プログラム マネージャーは次の 2 つのアクションから選択できます。

- **CoE の承認**: アイデアは CoE 実装チームによって開発される必要があります
- **Citizen Developer の承認**: 低リスク、単純または中程度のタスクを含むアイデアは、Citizen Developer によって開発できます。

![image-20230223094550-3.png](/static/img/image-20230223094550-3.png)

**d.プログラム マネージャーが選択したフローに基づいて、次の 2 つのケースが考えられます。**

**CoE の承認**

アイデアは認定フェーズに更新され、ステータスは承認済みになります。プログラム マネージャーは、プロジェクト マネージャーをコラボレーターとして追加する必要があります。

![image-20230223095132-4.png](/static/img/image-20230223095132-4.png)

割り当てられたプロジェクト マネージャーは、その特定のアイデアに関連するアクションを希望の順序で実行できます。

- 「ビジネス アナリスト」や「RPA 開発者」の役割を持つユーザーなどのコラボレータを追加する

![image-20230223095313-5.png](/static/img/image-20230223095313-5.png)

- 自動化プロファイルの編集ビューで利用可能なドロップダウンからアイデアのフェーズとステータスを更新する - 「概要」セクション

![image-20230223095932-8.png](/static/img/image-20230223095932-8.png)

- プロジェクト計画を作成し、費用対効果分析セクションでコスト見積もりを追跡する

![image-20230223101602-13.png](/static/img/image-20230223101602-13.png)

- アイデアに関連するドキュメントを持参するよう協力者に奨励します: 現状、将来のドキュメント (PDD、SDD、DSD)

![image-20230223101938-14.gif](/static/img/image-20230223101938-14.gif)

- RPA 開発者に、再利用可能なコンポーネントのギャラリーを探索し、独自のコンポーネントをアップロードすることを奨励します。
  - コンポーネントを再利用する

![image-20230223102414-15.gif](/static/img/image-20230223102414-15.gif)

    - [ワークスペース] -> [マイ コンポーネント] -> [コンポーネントのアップロード] にアクセスして、独自のコンポーネントをアップロードします。
- 導入が完了したら、自動化の可能性 %、コスト、およびプロジェクト計画を更新して、プロジェクトの現実を反映します。これは、費用便益分析を編集し、そこに最新の情報を取り込むことによって行われます。
- 必要な実装フェーズが完了すると、プロジェクト マネージャーはアイデア フェーズをライブに更新し、自動化の実装が完了したことを反映しました。

![image-20230223103332-16.gif](/static/img/image-20230223103332-16.gif)

## 2. Citizen Developer の承認
- アイデアのフェーズが「実装」ビューで使用できる「開発ステータス未開始」に更新されます。

![image-20230223103639-17.png](/static/img/image-20230223103639-17.png)
- シチズン開発者は実装ビューにアクセスし、[開発の開始] を選択してさらに開発するためのアイデアを選択します。

![image-20230223104347-19.gif](/static/img/image-20230223104347-19.gif)
- アイデアの開発が完了したらすぐに、[ドキュメント] タブにアップロードし、クリックして [テクニカル レビューに移動] をクリックします。
![image-20230223105848-20.gif](/static/img/image-20230223105848-20.gif)
- テクニカルレビュー担当者はコードレビューを実行し、更新が必要ない場合は自動化を承認し、フェーズをライブに変更します。

![image-20230223110009-22.gif](/static/img/image-20230223110009-22.gif)

## 4. CoE 主導のアイデアを検討する
**a.プロセス所有者はアイデアの詳細情報を確認します**

CoE 主導のアイデアが提出されるとすぐに、そのアイデアに割り当てられたプロセス所有者に評価の招待状が送信され、詳細な評価の情報の修正を確認するよう求められます。 **私のタスク – 未完了セクション** のプロセス所有者アクセス アイデア。プロセス所有者が [評価の送信] をクリックするとすぐに、アイデアのフェーズが資格認定に更新され、そのステータスはレビュー待ちに更新されます。

**b.プログラム マネージャーはアイデアを検討し、さらに実装フェーズに進むかどうかを決定します**

アイデアの実装に必要な作業の複雑さに基づいて、プログラム マネージャーは次の 2 つのアクションから選択できます。

- CoE の承認: アイデアは CoE 実装チームによって開発される必要があります
- Citizen Developer の承認: 低リスク、単純または中程度のタスクを含むアイデアは、Citizen Developer が開発できます。

**c.プログラム マネージャーが選択したフローに基づいて、次の 2 つのケースが考えられます。**

**1. CoE の承認**

アイデアは認定フェーズに更新され、ステータスは承認済みになります。プログラム マネージャーは、プロジェクト マネージャーをコラボレーターとして追加する必要があります。

割り当てられたプロジェクト マネージャーは、その特定のアイデアに関連するアクションを希望の順序で実行できます。

- 「ビジネス アナリスト」や「RPA 開発者」の役割を持つユーザーなどのコラボレータを追加する
- 自動化プロファイルの編集ビューで利用可能なドロップダウンからアイデアのフェーズとステータスを更新する - 「概要」セクション
- プロジェクト計画を作成し、費用対効果分析セクションでコスト見積もりを追跡する
- アイデアに関連するドキュメントを持参するよう協力者に奨励します: AS IS、TO BE ドキュメント (PDD、SDD、DSD)。
- RPA 開発者に、再利用可能なコンポーネントのギャラリーを探索し、独自のコンポーネントをアップロードすることを奨励します。
- 導入が完了したら、自動化の可能性 %、コスト、およびプロジェクト計画を更新して、プロジェクトの現実を反映します。これは、費用便益分析を編集し、そこに最新の情報を取り込むことによって行われます。
- 必要な実装フェーズが完了すると、プロジェクト マネージャーはアイデア フェーズをライブに更新し、自動化の実装が完了したことを反映しました。

**2. Citizen Developer を承認**

- アイデアのフェーズが「実装」ビューで使用できる「開発ステータス未開始」に更新されます。
- Citizen Developer は実装ビューにアクセスし、さらに開発するためにアイデアを選択します。
- アイデアの開発が完了するとすぐにテクニカルレビューに移されます。
- テクニカル レビューはコード レビューを実行し、更新が必要ない場合は自動化を承認し、フェーズをライブに変更します。

## 5. 自動化をレビューする

**a. Business Reviewer によるビジネス観点からのレビューの自動化**

自動化が提出された後は、ビジネス レビュー フェーズに移行し、自動化が現在のビジネスに適応できるかどうかをビジネス レビュー担当者によってレビューされます。 **ワークスペース -> オートメーション パイプライン** にアクセスすることで、ビジネス レビュー担当者は、承認、保留、拒否、または技術レビューに移動することができます。ビジネスレビュー担当者が承認して技術レビューに移行すると、自動化はレビュー待ちステータスで技術レビューフェーズに移行します。

**b.技術レビュー担当者が技術の観点から自動化をレビューします**

テクニカル レビューでは、テクニカル レビュアーは **ワークスペース -> 自動化パイプライン** にアクセスして、技術の観点から自動化をレビューする責任があります。

- 技術レビュー担当者が承認すると、自動化は運用段階のステータスでライブ段階に入り、その自動化は誰でも使用できるようになります。
- 技術レビュー担当者が自動化の再作業が必要な場合は、プログラム マネージャーがその自動化を再作業する方法を再検討して決定し、自動化は認定フェーズに移行します。さらに、自動化の開発フローは、プログラム マネージャーの開発方法によって異なります。
