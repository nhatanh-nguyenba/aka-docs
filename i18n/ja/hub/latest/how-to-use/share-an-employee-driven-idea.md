# 従業員主導のアイデアを共有する

[**共有**] -> [**CoE 主導のアイデアを共有**] を選択して、アイデアの詳細を追加します

![image-20230116140932-1.png](/static/img/image-20230116140932-1.png)

CoE 主導のアイデアを共有するためのフォームが表示されたら、すぐに必要な情報を順番に入力します。

1. **概要**: 名前、説明、問題点、マイナスの影響などのアイデアの概要を説明し、アイデアが適用できるビジネス領域を選択します。
2. **詳細な評価**: 質問に対する回答を記入します
3. **評価 KPI**: アイデアの提出プロセス中、アルゴリズムがサイレントに実行され、次の出力スコアが計算されます。
   - **プロセス実装の容易さ**: % で表します (0 ～ 100%、高いほど優れています)。この KPI は、アイデアの実装の容易さを予測します。
   - **FTE (人) の節約**: この KPI は、アイデアが自動化された場合に節約される人の数を予測します。
   - **自動化の価値**: この KPI は、アイデアがもたらす価値を計算します。節約されたFTE数/合計FTE数に基づいて計算できます
     - 値が 0.3 未満の場合、アイデアは組織に貢献する価値が低いことになります。
     - 値が 0.3 ～ 0.6 の場合、アイデアは組織に中程度の価値をもたらします。
     - 値が 0.6 以上であれば、そのアイデアは組織に高い価値をもたらします。
   4. **Assign Process Owner**: アイデアの詳細情報の修正を確認する責任を負うプロセス オーナーを割り当てます。

共有後、プロセス所有者がアイデア内の情報の修正を確認します。アーカイブしたり、評価を提出したりできます。詳細については、アイデアのレビュー フローを確認してください。
