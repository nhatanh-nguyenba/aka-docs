# アイデアを共有する

**[共有] -> [アイデアを共有]** をクリックして、アイデアの詳細を追加します。

![image-20230116140830-1.png](/static/img/image-20230116140830-1.png)

[アイデアの共有] フォームが開き、アイデアに関する詳細を順番に入力できます。

1. **概要**: 名前、説明、問題点、マイナスの影響などのアイデアの概要を説明し、アイデアが適用できるビジネス領域を選択します。
2. **高レベルの評価**: 質問に対する回答を記入します
3. **評価 KPI**: アイデアの提出プロセス中、アルゴリズムがサイレントに実行され、次の出力スコアが計算されます。
   - **アイデア スコア** - % で表します (0 ～ 100% - 高いほど優れています)。これは、提出者によってリッカート スケール (1 から 5) でランク付けされた次の 5 つの要素間の単純な算術平均、または適合性 % と準備性 % の加重平均として計算されたアイデアの全体スコアです。
     - プロセスにルールがある度合い。
     - 入力のデジタル化の程度。
     - 入力データがどの程度構造化されているか。
     - 今後のプロセスが変わります。
   - **適合性**: このスコアは、アイデアが自動化にどの程度適合しているかを判断するのに役立ち、次の 3 つの要素間の単純な算術平均として計算されます。
     - プロセスにルールがある程度。
     - 入力のデジタル化の程度。
     - 入力データが構造化されている度合い。
   - **準備スコア**: このスコアは、プロセスのライフサイクルに基づいてアイデアを自動化する準備がどの程度整っているかを判断するのに役立ちます。これは、次の 2 つの要素間の単純な算術平均として計算されます。
     - 今後のプロセスが変わります。
     - プロセス/タスクを説明するドキュメントの存在。
4. **プロセス所有者の割り当て**: アイデアの詳細情報を入力する責任を負うプロセス所有者を割り当てます。

共有後、アイデア承認者がアイデアをレビューします。承認済み、重複、または拒否としてマークを付けることができます。詳細については、アイデアのレビュー フローを確認してください。
