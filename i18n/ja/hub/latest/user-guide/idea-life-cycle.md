# アイデアのライフサイクル

akaBot Hub は、**アイデア ライフ サイクル** と **オートメーション ライフ サイクル** という 2 つのライフ サイクル フローをユーザーに提供します。この記事では、これら 2 つのフローの詳細を詳しく説明し、ユーザーがそれらを業務にどのように適用できるかを説明します。

特定のビジネス プロセスの自動化に関するアイデアはあるものの、それを実装する能力が不足しているユーザーの場合、最初のステップは akaBot Hub とアイデアを共有することです。その後、アイデアは、ユーザーが生成したアイデアの実装をサポートするように設計されたアイデア ライフ サイクル プロセスに従います。

一方、ユーザーが特定のビジネス プロセスを自動化するためのアイデアを実装できる場合は、アイデアの実装に進み、結果の自動化を akaBot Hub と共有できます。これにより、ユーザー生成の自動化ソリューションの実装を管理するために特別に設計された自動化ライフ サイクル プロセスが開始されます。

## 1. アイデアのライフサイクル

![image-20230222173039-7.png](/static/img/image-20230222173039-7.png)

## 1.1.アイデアを共有する
akaBot Hub でライフ サイクル プロセスを開始するには、ユーザーはまず特定のプロセスまたはタスクを自動化するためのアイデアを持っている必要があります。アイデアが生成されたら、ユーザーは akaBot Hub でのアイデアの共有に進むことができます。ユーザーが akaBot Hub でアイデアを共有するには、**アイデアの共有** と **CoE 主導のアイデアの共有** の 2 つの方法があります。これら 2 つのオプションのどちらを選択するかは、自動化したいプロセスまたはタスクについてユーザーが持つ詳細レベルによって異なります。

ユーザーがプロセスまたはタスクに関して限られた量の情報しか持っていない場合は、**アイデアの共有** オプションを選択できます。このオプションは、まだアイデア開発の初期段階にあり、専門家からの追加情報が必要なユーザーに最適です。

一方、ユーザーが自動化したいプロセスやタスクを完全かつ詳細に理解している場合は、**CoE 主導のアイデアを共有** オプションを選択できます。このオプションは、自動化プロセスに精通している経験豊富なユーザー向けに設計されており、自動化したいプロセスやタスクに関する包括的な情報を提供できます。

![image-20230220160149-1.png](/static/img/image-20230220160149-1.png)

ユーザーがアイデアを akaBot Hub に送信すると、プラットフォームはアイデア評価で概説されている式を使用して各アイデアの可能性を評価します。各アイデアの評価に使用される具体的な主要業績評価指標 (KPI) は、提出されたアイデアの種類によって異なります。

- **「アイデアの共有**」カテゴリの場合、akaBot Hub はアイデア スコア、準備スコア、適合性スコアに基づいてアイデアを評価します。これらのスコアは、アイデアの可能性と、それがカテゴリの目標と一致するかどうかを判断するために使用されます。

![image-20230220160422-2.png](/static/img/image-20230220160422-2.png)

- **「CoE 主導のアイデアの共有」** カテゴリで提出されたアイデアの場合、akaBot Hub は 3 つの重要な要素に基づいてアイデアを評価します。それは、既存のプロセスへのアイデアの実装の容易さ、アイデアがフルタイム従業員 (FTE) の時間を節約できる可能性、アイデアが企業にもたらす可能性のある全体的な価値です。これらの要素は、アイデアの可能性と、それがカテゴリの目標と一致するかどうかを判断するために使用されます。

![image-20230220160439-3.png](/static/img/image-20230220160439-3.png)

このアイデアは、「アイデアのレビュー」に続いてレビューされます。

## 1.3.アイデアの実装
アイデアがプログラム マネージャーによって承認され、コストと利点を分析するためにプロジェクト マネージャーに割り当てられると、プロジェクト マネージャーは、プロジェクト計画、必要なロボット、利点の詳細、およびアイデアを実装するためのコストの見積もりを見積もります。

- プロジェクト計画: アイデアを実装するためのタイムラインを見積もる

![image-20230220160533-4.png](/static/img/image-20230220160533-4.png)

- 必要なロボット: プロジェクト マネージャーが、このビジネス プロセスにどの自動化タイプを使用するかを決定します。

![image-20230220160953-5.png](/static/img/image-20230220160953-5.png)

- コストの見積もり: プロジェクト マネージャーは、アイデアの実装にかかるコストを見積もります。

![image-20230220161238-6.png](/static/img/image-20230220161238-6.png)

- 利点を絞り込む: akaBot Hub は、いくつかの基準に関連するアイデアの自動化後の節約額を見積もります。合計 FTE/FTR (ビジネス プロセスが有人自動化タイプを使用する場合)。年間合計時間。年間ランニングコスト。ランニングコスト/月とランニングコスト/トランザクション

![image-20230220162203-9.png](/static/img/image-20230220162203-9.png)

コストと利点の分析が完了したら、プロジェクト マネージャーはタスクを細分化し、チーム メンバーにタスクを割り当てます。また、プロジェクトマネージャーはプロジェクトの進捗に応じてアイデアのフェーズやステータスも変更します。  チームメンバーはタスクを実行し、ドキュメントとソースコードをそのアイデアの「ドキュメント」タブにアップロードします。

![image-20230220161435-7.png](/static/img/image-20230220161435-7.png)

## 1.4.アイデアの投資効果を評価する
開発チームがアイデアの実装を完了すると、プロジェクト マネージャーは、アイデアの実装にかかる実際のコストを「コストと利点」セクションで報告する責任があります。見積コストと実際のコストの間に差異がある場合、プロジェクト マネージャーはそれに応じてコストを編集する必要があります。

実際のコストが記録されると、akaBot Hub は、推定コストと実際のコストの差を考慮して、節約額を正確に再計算します。これにより、節約の計算が可能な限り正確になり、アイデアの実装の実際の価値が反映されるようになります。

## 1.5.アイデアはユーザーの準備が整っています
プロジェクト マネージャーがアイデアを「運用中」ステータスの「ライブ」フェーズに変更すると、対応するビジネス プロセスに関与する従業員がそのアイデアを利用して、ボットをダウンロードしてタスクを実行できるようになったということになります。

さらに、従業員は、akaBot Hub の「自動化の探索」セクションで利用可能なすべてのアイデアと自動化を探索して、特定のビジネス プロセスに適したボットを見つけたり、独自のプロセスを自動化するための新しいアイデアを探したりすることができます。これにより、従業員は akaBot Hub の可能性を最大限に活用し、企業全体でプロセスの改善と自動化の機会を特定できるようになります。

## 2. 自動化のライフサイクル

![image-20230222172953-6.png](/static/img/image-20230222172953-6.png)

## 2.1.アイデアの実装
ライフサイクルを開始するには、ユーザーは特定のプロセスを自動化するためのアイデアを持ち、まずそのアイデアを実装できる必要があります。その後、ユーザーは RPA プラットフォームを使用してアイデアを実装できます。

## 2.2.自動化の共有
アイデアの実装が完了したら、ユーザーは akaBot Hub と自動化を共有できます

![image-20230220174514-12.png](/static/img/image-20230220174514-12.png)

次に、akaBot Hub は、従業員あたりの潜在的な利益 (時間/年) と会社全体の潜在的な利益 (時間/年) に関連して、自動化が会社にもたらすメリットを評価します。

akaBot Hub は、従業員ごとの潜在的な利益 (時間/年) と会社全体の潜在的な利益 (時間/年) という 2 つの重要な要素に基づいて自動化を評価します。これらの要素は、アイデアの可能性と、それがカテゴリの目標と一致するかどうかを判断するために使用されます。

![image-20230220174644-13.png](/static/img/image-20230220174644-13.png)

## 2.3 レビューの自動化
自動化が送信されると、包括的な審査プロセスが行われます。このレビューの最初の段階では、ビジネス プロセスに対する自動化の適合性に焦点を当てます。ビジネスレビュー担当者は、提出者から提供されたドキュメントとソースコードを検査して、自動化が提出物に概要が記載されているビジネスプロセスと一致しているかどうかを判断します。自動化がビジネス レビューに合格すると、次の段階に進み、技術レビュー担当者が必要な技術基準を満たしているかどうかを評価します。詳細なレビュー フローの詳細については、「自動化のレビュー フロー」を参照してください。

## 2.4.ユーザーは自動化を利用できるようになります
技術レビュー担当者が自動化を承認すると、「運用中」ステータスのライブ フェーズに移行します。これは、関連するビジネス プロセスに関与している従業員がボットをダウンロードし、それを使用してタスクの完了を支援するための自動化が利用可能になったことを示しています。従業員は、akaBot Hub の Explore Automation セクションで利用可能なすべてのアイデアと自動化を探索し、自分たちのビジネス プロセスに適したボットを見つけることができます。
