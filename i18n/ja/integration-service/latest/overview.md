# 統合サービス

Akabot Integration Service は、Akabot をエンタープライズ アプリケーション、クラウド プラットフォーム、サードパーティ API と橋渡しするローコード接続レイヤーを提供します。すべてのワークフロー内にカスタム HTTP 呼び出しを記述する代わりに、チームは再利用可能な接続とトリガーを一度設定すれば、どこでも使用できます。

## 主な特徴

- **構築済みコネクタ** — SAP、Salesforce、ServiceNow、Microsoft 365、Jira、主要なクラウド サービスを含む 200 以上のアプリケーションへの既成の接続。
- **REST および SOAP のサポート** — HTTP エンドポイントを名前付き接続として構成します。 Integration Service は、認証、再試行、エラー マッピングを処理します。
- **イベント トリガー** — 新しいメール、フォームの送信、SaaS アプリからの Webhook などの外部イベントが発生したときに、ワークフローを自動的に開始します。
- **データ変換** — ビジュアル フィールド マッパーまたは JSONata 式を使用して、システム間でペイロードをマッピングおよび変換します。
- **接続ボールト** — 資格情報と OAuth トークンは安全に保管され、ワークフロー開発者に公開されることはありません。
- **ヘルスモニタリング** — 設定されているすべての接続のリアルタイムの接続ステータスと、障害に関するアラート。

## 一般的な統合パターン

| パターン | 説明 |
|---------|-------------|
| **イベントでトリガー** | 外部システムからの Webhook により、Akabot ワークフローが自動的に起動されます。 |
| **読み取りと同期** | ソース システムからデータを取得し、変換して、ターゲット システムに書き込みます。 |
| **完了時のコールバック** | ワークフローが終了したら、結果を元のシステムにポストして戻します。 |
| **投票中** | 外部 API で新しいレコードを定期的にチェックし、それぞれをキュー アイテムとして処理します。 |

## サポートされている認証方法

- APIキー/ベアラートークン
- OAuth 2.0 (認証コード、クライアント認証情報)
- 基本認証
- 証明書ベース (mTLS)

## はじめに

1. Akabot Center を開き、**Integration Service** に移動します。
2. コネクタ カタログを参照し、ターゲット アプリケーションを選択します。
3. 認証を構成し、接続をテストします。
4. Studio で **Integration Service** アクティビティ グループを介してコネクタを使用します。
