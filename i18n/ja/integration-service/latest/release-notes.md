# 統合サービス — リリースノート

## v3.2.0 — 2026 年 4 月

### 新機能
- **イベント メッシュ** — 直接結合せずに、あるワークフローからイベントを発行し、別のワークフローでそれらをサブスクライブします。複数のサブスクライバへのファンアウトをサポートします。
- **GraphQL コネクタ** — 完全なスキーマ イントロスペクションと変数サポートを使用して、GraphQL API のクエリと変更を行います。
- **接続の健全性チェック** - Center の自動アラートと健全性ステータス バッジを備えた、スケジュールされた接続テスト。

### 新しいコネクタ
- Workday (人事および財務)
- ハブスポット (CRM)
- Twilio (SMS および音声)
- Azure Service Bus (メッセージング)

### 改善点
- OAuth 2.0 トークンの更新は、ワークフローを中断することなく透過的に処理されるようになりました。
- ペイロード変換エディターは、単体テスト モードで JSONata v2 式をサポートするようになりました。
- コネクタごとに再試行ポリシーを構成できるようになりました: 最大試行数、遅延戦略 (固定/指数)、および再試行可能なエラー コード。

### バグ修正
- 有効期限の計算における 1 つずつのずれが原因で、ちょうど 2 時間後に Salesforce コネクタがセッション トークンを失う問題を修正しました。
- 2 つのワークフローが同じ共有 OAuth トークンを同時に更新した場合のデッドロックが解決されました。
- 受信ワークフローがすでに最大同時実行数に達している場合に Webhook イベントがドロップされる問題を修正しました。

## v3.1.0 — 2026 年 1 月

### 新機能
- **双方向同期テンプレート** — ウィザードを使用して構成可能な一般的なペア (SAP ↔ Salesforce、Jira ↔ ServiceNow) の事前に構築された同期パターン。
- **署名の要求** — 送信 Webhook 呼び出しに対する HMAC-SHA256 署名により、受信側システムが信頼性を検証できます。
- **接続グループ** — 複数の接続を名前付きグループにバンドルし、ワークフロー内でそのグループを参照します。 1 つの設定を変更するだけで環境を交換できます。

### 新しいコネクタ
- Microsoft ダイナミクス 365
- Google Workspace (スプレッドシート、ドライブ、Gmail)
- たるみ

### 改善点
- コネクタ カタログは 150 から 200 以上のアプリケーションに増加しました。
- 統合呼び出しのログの詳細度が構成可能になりました (エラーのみ/概要/完全なペイロード)。

### バグ修正
- WS-Security ヘッダーを含む SOAP コネクタが .NET 8 ランタイムで失敗する問題を修正しました。
- タイムゾーン オフセットを含む ISO 8601 日付形式を使用するコネクタのフィールド マッピングを修正しました。

## v3.0.0 — 2025 年 9 月

### 新機能
- **Integration Service v3 のリリース** — ビジュアル コネクタ ビルダーと集中接続ボルトを備えた新しいローコード接続レイヤー。
- **200 以上の構築済みコネクタ** — 主要な ERP、CRM、ITSM、クラウド プラットフォーム用のすぐに使えるコネクタ。
- **イベント トリガー** — 接続されたシステムからの受信 Webhook またはポーリング イベントに応答してワークフローを開始します。

### 重大な変更
- Integration Service v2 のカスタム コネクタ定義は、新しい YAML コネクタ仕様に移行する必要があります。
- v2`InvokeConnector`アクティビティは、コネクタ スキーマから生成されたアプリ固有のアクティビティに置き換えられます。

### バグ修正
- 大量の API 呼び出しワークロードが継続した場合に発生する接続プールの枯渇の問題を解決しました。
- RFC 関数モジュールがゼロ以外の戻りコードを返したときに、SAP RFC コネクタがサイレントに失敗する問題を修正しました。
