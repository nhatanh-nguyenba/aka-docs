# アナリティクスの使用法

分析は、カスタマー サポート業務で何が起こっているかを理解するのに役立ちます。メッセージ、会話、チケット、連絡先のアクティビティを時系列で表示します。

次のような質問に答えたい場合は、Analytics を使用します。

- 今週は顧客からのメッセージが増えていますか?
- どのチャンネルが最も混雑していますか?
- 作成または解決されたチケットは何件ありますか?
- 多くの顧客がサポートを待っていますか?
- 時間の経過とともに連絡先は増えていますか?

## Analytics が追跡するもの

分析は、次のような運用アクティビティを長期にわたって追跡するのに役立ちます。

- 新しい連絡先
- 受信/送信されたメッセージ
- 作成/更新された会話
- チケットアクティビティ (作成、解決、再オープン、割り当て、優先度の変更)
- 連絡先のライフサイクル アクティビティ

## オープンアナリティクス

1. 左側のサイドバーで **AI** を開きます。
2. **分析**を選択します。
3. 5 つのセクションから 1 つを選択します。
   - **概要**
   - **会話**
   - **メッセージ**
   - **チケット**
   - **接触**

Analytics が表示されない場合は、管理者にロールのアクセス権を確認するよう依頼してください。

## ツールバーを使用する (ページの上部)

### 1) 日付範囲

以下を選択できます。

- **今日**
- **今月**
- **月を選択** (月ピッカーを表示)
- **昨日**
- **過去 7 日間**
- **過去 14 日間**
- **過去 30 日間**

### 2) リセット

- [**リセット**] をクリックしてフィルタをデフォルトに戻します。
  - **概要**:`Today`にリセット
  - 他のタブ:`Current month`にリセット

### 3) エクスポート(Excel)

- **会話 / メッセージ / チケット / 連絡先** タブで、**エクスポート** をクリックします。
- Excel ファイルが自動的にダウンロードされます。
- エクスポートは **概要** には表示されません。

## [分析] の各タブを理解する

### 1. 概要

概要には、高レベルのカードと傾向グラフが表示されます。

- **新しい連絡先**
- **新しいメッセージ** (トップチャンネルを含む)
- **新しい会話** (トップチャンネルを含む)
- **チケット** (解決済み + 再開/新規)

![Overview analytics tab](/static/img/overview.png)

カードの下には、4 つの傾向グラフが表示されます。

- 新しい連絡先
- 新しいメッセージ
- 新しい会話
- チケット (新規、再開、解決済み)

### 2. 会話

このタブを使用して、会話のボリュームと所有権を把握します。

チャートには次のものが含まれます。

- **会話が作成されました**
- **会話ステータスが変更されました**
- **会話が割り当てられました**

![Conversation analytics tab](/static/img/conversation.png)

各グラフにはフィルター コントロールが含まれる場合があります。

- **ディメンション** (例: チャネル タイプ、チャネル、新しいステータス、担当者)
- **値** (値を 1 つ選択するか、**すべて**)

enum タイプのディメンションで **すべて** を選択すると、チャートの表示を切り替えることができます。

- **グループ化**
- **積み重ね**

### 3. メッセージ

このタブを使用して、受信および送信メッセージのアクティビティを理解します。

チャートには次のものが含まれます。

- **メッセージを受信しました**
- **メッセージを送信しました**

![Message analytics tab](/static/img/message.png)

次のようなディメンションでフィルタリングできます。

- チャネルの種類
- チャネル

### 4. チケット

このタブを使用して、サポートのワークロードと問題の解決策を理解します。

チャートには次のものが含まれます。

- **チケットが作成されました**
- **チケットは解決されました**
- **チケットの再開**
- **チケットが割り当てられました**
- **チケットの優先順位が変更されました**

![Ticket analytics tab](/static/img/ticket.png)

一般的な寸法は次のとおりです。

- チケットの種類
- 接触
- 譲受人
- 新たな優先事項

### 5.連絡先

このタブを使用して、顧客の増加と連絡先の変更を理解します。

チャートには次のものが含まれます。

- **連絡先が作成されました**
- **連絡先 ID がリンクされました**

![Contact analytics tab](/static/img/contact.png)

共通寸法:

- ライフサイクル段階

## チャートを素早く読む方法

- 横軸＝時間
- 縦軸 = イベント数
- 棒/線の上にマウスを置くと正確な値が表示されます
- **ディメンション + 値** を使用して、特定の質問に回答します (例: 「1 つのチケット タイプに対して何枚のチケットが再オープンされましたか?」)

## 実際の例

### 毎週のサポートレビュー

1. **概要**を開きます。
2. **過去 7 日間**を選択します。
3. メッセージやチケットが増加したかどうかを確認します。
4. **チケット**を開いて、解決済みおよび再オープンされたケースを確認します。
5. 毎週のチーム会議用にレポートをエクスポートします。

### チャンネルパフォーマンスチェック

1. **メッセージ**を開きます。
2. **過去 30 日間**を選択します。
3. チャネルタイプでフィルタリングします。
4. Zalo、Facebook、WhatsApp、または Telegram のアクティビティを比較します。
5. スタッフの対応が最も必要な場所を決定します。

## 推奨されるワークフロー

1. 異常な変更を検出するには、**概要**から始めてください。
2. 詳細タブ（会話、メッセージ、チケット、連絡先）に移動します。
3. パターンをより早く見つけるために、最初は日付範囲を小さくしてください (今日 / 過去 7 日間)。
4. ディメンション フィルターを使用して、1 つのチャネル/チーム/ステータスを分離します。
5. マネージャーや週次レポートと共有する必要がある場合は、Excel をエクスポートします。

## Analytics が他の機能と接続する方法

- [Inbox](./inbox-usage)は、会話とメッセージ アクティビティを作成します。
- [Tickets](./ticket-usage)チケット アクティビティを作成します。
- [Contacts](./contact-management)連絡先アクティビティを作成します。
- [AI Assistant](../scaleflow-ai/ai-assistant)は手動の作業負荷を軽減し、Analytics は運用への影響を監視するのに役立ちます。

## 素早いトラブルシューティング

### 分析メニューが表示されない

- 管理者に Analytics へのアクセスを依頼してください。

### 「エクスポート」ボタンが表示されない

- エクスポートは、**会話 / メッセージ / チケット / 連絡先** タブにのみ表示されます。
- 仕様により、エクスポートは **概要** に表示されません。

### 「エクスポート」をクリックしたが、ファイルがダウンロードされなかった

- ブラウザのダウンロード許可/ポップアップ制限を確認してください。
- 日付範囲を狭めて再度お試しください。

### チャートが空っぽに見える

- 日付範囲を既知のアクティビティのある期間に変更します。
- **値 = すべて** を設定してフィルターを削除します。
