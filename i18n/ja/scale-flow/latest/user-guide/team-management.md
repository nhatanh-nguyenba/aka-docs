# チームマネジメント

チームは、ユーザーをサポート、セールス、アドミッション、請求、テクニカル サポートなどの作業グループに編成するのに役立ちます。

会話やチケットを 1 人ではなくグループに割り当てる必要がある場合は、チームを使用します。

## チーム管理の用途

**組織 -> チーム管理**では、次のことができます。

- ワークスペース内のすべてのチームを表示する
- チームを名前で検索する
- 新しいチームを作成する
- チーム情報を編集する
- チームメンバーの追加または削除
- 不要になったチームを削除する

## アクセスと権限

ボタンが見つからない場合は、ワークスペース管理者に連絡して権限を更新してください。

## オープンなチーム管理
![Open team](/static/img/open-team.png)
1. 左側のサイドバーで、**組織**を開きます。
2. **チーム管理**を選択します。
3. 次のような重要な情報を含むチーム リストが表示されます。
   - チーム名
   - 説明
   - 会員数
   - 状態
   - 作成日
   - アクション (manage がある場合)

![Team list overview](/static/img/list-team.png)

## 新しいチームを作成する

複数のユーザーが同じ責任を共有する場合は、チームを作成します。

 ![Add team button](/static/img/add-team-1.png)

1. [**+チームを追加**] をクリックします。
2. 必須フィールドに入力します。
   - チーム名
3. 必要に応じて以下を追加します。
   - 説明
   - 初期メンバー

   ![Add team dialog](/static/img/add-team.png)

4. [**チームの作成**] をクリックします。

ヒント:

- 明確な命名規則を使用してください (例:`Support - APAC`、`Sales - SMB`)。
- ワークフローに基づいて小規模なチームを作成し、オーナーシップを向上させます。

## チームメンバーの追加または削除

1. チーム行で、**編集** アイコンをクリックします。
2. **メンバーの管理** で、メンバー ピッカーを開きます。
3. ユーザーをメンバーリストに追加するか、チームに属さなくなったユーザーを削除します。
4. [**変更を保存**] をクリックします。

注:

- ユーザーは、組織のポリシーに基づいてチーム間で移動できます。
- 責任が変わるたびにメンバーを更新します。

例: 1 人のサポート スタッフ メンバーが請求チームに移動した場合、そのメンバーを`Customer Support`から削除し、`Billing Support`に追加します。

## チームの詳細を編集する

  ![Edit team button](/static/img/edit-team-1.png)

1. チーム行で、**編集** アイコンをクリックします。
2. 次のようなフィールドを更新します。
   - チーム名
   - 説明
   - メンバー

   ![Edit team confirmation dialog](/static/img/edit-team.png)

3. [**変更を保存**] をクリックします。

## チームを削除する

![Delete team confirmation dialog](/static/img/delete-team-1.png)

1. チーム行で、**削除** アイコンをクリックします。
2. ダイアログでアクションを確認します。

![Delete team confirmation dialog](/static/img/delete-team.png)

3. 確認が成功すると、チームは削除されます。

このアクションは慎重に使用してください。ワークフローで必要な場合は、削除する前にメンバーが別のチームに再割り当てされていることを確認してください。

## チームが受信トレイとチケットに接続する方法

- [Inbox](./inbox-usage)では、グループ内の誰かが支援できる場合に、会話をそのチームに割り当てることができます。
- [Tickets](./ticket-usage)では、次のステップがそのグループに属している場合に、チケットをチームに割り当てることができます。
- [User Management](./user-management)では、ユーザーをチームに追加するには、ユーザーが存在している必要があります。

## 現実世界のワークフロー

1. 管理者は`Refund Support`というチームを作成します。
2. 管理者は、払い戻しリクエストを確認できるスタッフを追加します。
3. 顧客が払い戻しを要求すると、スマート アシスタントがチケットを作成します。
4. チケットは`Refund Support`に割り当てられます。
5. チームメンバーなら誰でも問題を見つけて解決できます。

## 現在の UI の動作に関する注意事項

- チームのステータス（**アクティブ/非アクティブ**）がリストに表示されます。
- チームのステータスは現在、このページからは読み取り専用です。
- チームの編集はモーダル ダイアログで行われます (個別のチーム詳細ページはありません)。
- リストはページネーション (ページとページ サイズの制御) をサポートしています。

## トラブルシューティング

### チーム管理が表示されない

- チーム表示権限がない可能性があります。
- 管理者に組織チームのビューへのアクセスを許可するよう依頼してください。

### チームを表示できますが、作成または編集できません

- 表示権限のみを持っている可能性があります。
- 管理者にチーム管理の許可を求めてください。

### メンバーの割り当てができない

- メンバー管理は管理の一部です。
- 管理者にチーム管理アクセスを許可するよう依頼してください。

### 更新がすぐに表示されない

- 数秒待ってページを更新します。
- チームを再度開いて、保存された変更を確認します。
- 必要に応じて、サインアウトして再度サインインします。
