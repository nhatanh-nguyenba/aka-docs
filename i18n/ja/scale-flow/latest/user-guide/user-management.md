# ユーザー管理

ユーザー管理では、管理者がユーザーを ScaleFlow に追加し、そのアクセスを管理します。

新しいチームメイトが参加した場合、誰かがチームを変更した場合、またはユーザーがワークスペースにアクセスできなくなった場合に使用します。

## ユーザー管理の用途

**[組織] -> [ユーザー管理]** では、次のことができます。

- 組織内のすべてのユーザーを表示する
- ユーザーを検索する
- ページ分割されたユーザー リスト間を移動する
- 1 人または複数のチーム メンバーをメールで招待する
- ユーザーを手動で追加する
- ユーザー基本情報の編集
- 役割の割り当てまたは更新
- ユーザー詳細ページを開く
- ユーザーを削除する

## アクセスと権限

ボタンが見つからない場合は、ワークスペース管理者に連絡して権限を更新してください。

## オープンユーザー管理

![Open Organization and User Management from the sidebar](/static/img/open-user-manage.png)

1. 左側のサイドバーで、**組織**を開きます。
2. [**ユーザー管理**] を選択します。
3. 列を含むテーブルが表示されます。
   - 電子メール
   - 名前
   - 役割
   - ステータス
   - 作成日
   - アクション

   ![User management list page](/static/img/list-user.png)

## ユーザーを素早く見つける

ページの上部にある検索ボックスを使用します。

![Find user](/static/img/search-user.png)

- キーワードを入力します (例: メールアドレスまたは名前)
- 結果は少し遅れて自動的に更新されます
- [**フィルターをクリア**] をクリックして検索をリセットします

表ではページネーションが使用されています。テーブル コントロールからページとページ サイズを変更できます。

## 電子メールでユーザーを招待する

チームメイトを追加する場合、招待は最も簡単なオプションです。

   ![Open invite user](/static/img/open-invite-user.png)

1. [**ユーザーを招待**] をクリックします。
2. 1 つ以上の電子メール アドレスを入力します。
3. 次の方法でメールを追加できます。
   - **Enter** を押す
   - カンマを入力する
   - スペースを入力する
   - 複数のメールを一度に貼り付ける

   ![Invite team users dialog](/static/img/invite-team-user.png)

4. [**招待**] をクリックします。

重要:

- 無効な電子メールまたは重複した電子メールはダイアログでブロックされます。
- 招待リンクは **7 日間**有効です。
- 招待を受け入れると、ユーザーはサインインできます。会社で SSO を使用している場合は、[Account Registration](./account-registration)で説明されている SSO オプションを使用する必要があります。

## ユーザーを手動で追加する

手動作成は、組織が管理者にアカウントを直接作成してもらいたい場合にのみ使用してください。

![Open Add User](/static/img/open-add-user.png)

1. [**+ユーザーの追加**] をクリックします。
2. 必須フィールドに入力します。
   - 電子メール
   - 名
   - 姓
   - パスワード

   ![Add User dialog](/static/img/create-user.png)

3. [**ユーザーの作成**] をクリックします。

検証ルール:

- 電子メールは有効である必要があります
- 姓名は必須です
- パスワードは6文字以上である必要があります

## ユーザーを編集する
![Edit delete assign role user](/static/img/edit-delete-user.png)
1. ユーザー行で、**編集** アイコンをクリックします。
2. ユーザー情報を更新します。

![Edit user dialog](/static/img/edit-user.png)

3. [**変更を保存**] をクリックします。

注:

- 編集モードでは電子メールは読み取り専用です。
- このアクションにより、プロファイル フィールドが更新されます (役割の割り当てではありません)。

## ユーザーに役割を割り当てる

1. ユーザー行で、**役割の割り当て** アイコン (盾) をクリックします。
2. 1 つ以上の役割を選択します。
3. [**保存**] をクリックします。

注:

- 役割の割り当ては専用のダイアログで処理されます。
- 所有者アカウントの場合、ロールの割り当てはリストから無効になります。
- どのロールを選択すればよいかわからない場合は、[Roles & Permissions](./roles-permissions)を確認してください。

## ユーザー詳細ページを開く

「電子メール」列でユーザーの電子メールをクリックして詳細ページを開きます。

![Detail user](/static/img/detail-user.png)

以下を確認できます。

- 身元情報
- 連絡先情報
- 日付以降のステータスとメンバー
- ワークスペース関連のプロファイルの詳細

## ユーザーを削除する

1. ユーザー行で、**削除** アイコンをクリックします。
2. ダイアログで確認します。

![Delete user confirmation dialog](/static/img/delete-user.png)

3. 確認が成功すると、ユーザーは削除されます。

アクセスとワークフローの所有権に影響を与える可能性があるため、このアクションは慎重に使用してください。

## 実際のオンボーディングの例

1. 新しいサポートスタッフが入社しました。
2. 管理者は仕事用メールでユーザーを招待します。
3. ユーザーは[Account Registration](./account-registration)から SSO を使用してサインインします。
4. 管理者は`Support Agent`ロールを割り当てます。
5. 管理者は、[Team Management](./team-management)の適切なチームにユーザーを追加します。
6. ユーザーは[Inbox](./inbox-usage)で会話の処理を開始します。

## オフボーディングの例

1. スタッフが会社を辞めました。
2. 管理者がアクセスを削除または無効にします。
3. 管理者は、そのユーザーに割り当てられているオープンチケットを確認します。
4. チケットと会話は別のユーザーまたはチームに再割り当てされます。

## トラブルシューティング

### ユーザー管理が表示されない

- ユーザー表示権限がない可能性があります。
- 組織ユーザーに表示アクセスを許可するように管理者に依頼してください。

### テーブルを表示できますが、招待/追加/編集/削除はできません

- 表示権限のみを持っている可能性があります。
- 管理者にユーザー管理権限を依頼してください。

### 役割を割り当てることができません

- 役割の割り当て権限が必要です。
- 役割の割り当てへのアクセスを許可するように管理者に依頼してください。

### 変更はすぐには表示されません

- 通常、リストはアクションが成功した後に自動的に更新されます。
- 必要に応じて、ページを更新して再度検索してください。
