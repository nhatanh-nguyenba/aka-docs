# 概要

**Ask AI**は、**akaBot Studio**に直接統合された、インテリジェントで対話型かつコンテキストを認識するアシスタントです。RPA開発者のコパイロットとして機能するAsk AIを使用すると、自然言語で対話して、ワークフローのロジック説明、アクティビティの構成、エラーのトラブルシューティング、式の作成を行うことができ、自動化設計プロセスをより迅速かつスムーズに進めることができます。

## 主な機能

構築している特定のワークフローやアクティビティに関係なく、Ask AIは開発者向けの主要な機能を提供します。

* **コンテキスト認識アシスタンス**: Ask AIは単に一般的な質問に答えるだけではありません。現在のプロジェクト、変数、引数、選択したアクティビティ、またはエラー出力のコンテキストを理解し、カスタマイズされた解決策を提供します。
* **ワークフロー設計のガイダンス**: 特定のタスクに対してどのアクティビティを使用し、どのように連結させるかなど、自動化プロセスの構築方法について即座に推奨事項を得ることができます。
* **アクティビティの説明**: akaBotのアクティビティ、そのプロパティ、入力/出力要件、および構成方法について学ぶことができます。
* **エラーのトラブルシューティングとデバッグ**: ログやエラーメッセージを提供すると、Ask AIが根本原因を分析し、問題を解決するための正確な手順を案内します。
* **式の生成**: 変数やデータ型に合わせて調整されたVB.NetまたはC#の式（例：文字列操作、LINQ、日付形式の変換など）を生成します。

## UIの概要

**Ask AI**パネルはakaBot Studioの右側にドッキングされており、**Properties**、**Outline**、**Locals**とペイン領域を共有しています。

![Ask AI Pane](/static/img/ask-ai.png)

### UIコンポーネントのリファレンス

1. **Ask AI タブ**: このタブをクリックして、Ask AIの対話パネルを開くか、切り替えます。
2. **ヘッダーとクォータインジケーター**: 機能名と1日の使用状況ステータス（例：`Daily usage: 1/15 prompts used`）を表示します。Ask AIのデフォルトの1日クォータは15プロンプトです。
3. **チャットエリア / 会話履歴**: 初期挨拶、推奨されるプロンプトテンプレート、進行中の対話スレッドを表示します。
4. **入力フィールド**: 質問、指示、またはプロンプトを入力するテキストボックスです。
5. **コンテキストタグメニュー**: 入力フィールドに`@`を入力すると自動的にトリガーされます。アクティブなプロジェクトから特定のコンテキスト要素をリンクさせることができます。
6. **送信ボタン**: プロンプトとアクティブなコンテキストをAsk AIサービスに送信します。

### コンテキストタグのリファレンス (`@`)

コンテキストタグを使用することで、AIに正確なデータを渡すことができます。以下の要素を参照できます。

| タグ | 参照ターゲット | 説明 |
| :--- | :--- | :--- |
| `@variables` | アクティブなワークフロー変数 | 現在開いているワークフローで定義されている変数を参照します。 |
| `@arguments` | アクティブなワークフロー引数 | 現在開いているワークフローで定義されている引数を参照します。 |
| `@activity` | 選択されたアクティビティ | デザイナー（Designer）で現在選択されているアクティビティのプロパティ構成を参照します。 |
| `@output` | 出力 / 実行ログ | 最新のログ、実行メッセージ、およびエラートレースを参照します。 |
| `@[filename].xaml` | 特定のワークフローファイル | 特定のワークフローファイル（例：`@Main.xaml`）の基礎となるコード/構造を参照します。 |

## 操作手順

自動化開発中にAsk AIと対話するには、次の手順に従います。

### ステップ 1: パネルを開く
akaBot Studioの右下パネルにある **Ask AI** タブをクリックして、チャットインターフェイスを開きます。

### ステップ 2: クエリを作成する
入力フィールドにプロンプトを入力します。リクエストでコンテキストを認識させるには、次の手順を実行します。
1. 入力フィールドに `@` と入力します。
2. ポップアップメニューから適切なタグを選択します（例：構成について質問する場合は `@activity` を、エラーについて質問する場合は `@output` を選択します）。
3. 続けて、自然言語で指示を入力します。

### ステップ 3: プロンプトを送信する
**送信**ボタンをクリックするか、**Enter**キーを押します。Ask AIが質問を処理し、添付されたコンテキストを検査して、回答を生成します。

### ステップ 4: 確認と適用
AIによって生成された提案、手順、またはコードブロックを確認します。
* **指示の場合**: 提案されたステップバイステップのUI調整に従います。
* **式/コードの場合**: チャットペイン内のコードスニペットにある**コピー（Copy）**アイコンをクリックし、アクティビティプロパティまたはExpression Editorに直接貼り付けます。

## 具体的な使用例

RPAタスクを効率化するためにAsk AIを使用する代表的なシナリオを以下に示します。

### エラーの修正サポート

ワークフローの実行が失敗した場合、Outputパネルのログを活用してエラーを特定し、解決することができます。

* **シナリオ**: 実行中にファイルがすでに「保護されたビュー（Protected View）」で開かれているため、*Read Range* アクティビティが失敗します。
* **ステップ 1**: デザイナー（Designer）パネルで失敗している *Read Range* アクティビティをクリックして選択します。
* **ステップ 2**: **Ask AI** パネルを開きます。
* **ステップ 3**: 次のプロンプトを入力します。
  > "`@output @activity` How do I fix the error shown in for my selected activity?"
* **ステップ 4**: **Send**（送信）をクリックします。Ask AIが `@output` タグからアクティブなエラーメッセージ（例：`IOException: The process cannot access the file because it is being used by another process`）を読み取り、選択された `@activity` のプロパティを識別します。
* **ステップ 5**: AIから提供された解決策を確認して適用します。
```markdown
選択したアクティビティに表示されているエラーを修正するには、Excelファイルがすでに保護されたビュー（Protected View）で開かれているときに発生するエラーを処理する必要があります。

Excel Application Scopeアクティビティにエラーハンドラーを追加して、この例外をキャッチして処理できます。

これを行うには、アクティビティを右クリックし、「エラー処理（Error Handling）」>「エラーバウンダリの追加（Add Error Boundary）」を選択して、アクティビティにエラーハンドラーを追加します。

その後、「ログメッセージ（Log Message）」アクティビティを追加してエラーメッセージを記録するか、「スロー（Throw）」アクティビティを追加してカスタムエラーをスローするなど、エラーを処理するアクティビティを追加できます。

あるいは、「トライキャッチ（Try Catch）」アクティビティを使用して例外をキャッチし、処理することもできます。
```

![Fix an error example](/static/img/fix-an-error-ask-ai.png)


### アクティビティの使用方法

Ask AIは、特定のアクティビティのプロパティ構成や必要な式の記述をサポートします。

* **シナリオ**: *Deserialize JSON* アクティビティを使用して、JSONレスポンス文字列をJSONオブジェクトにデシリアライズします。
* **ステップ 1**: デザイナー（Designer）で *Deserialize JSON* アクティビティを追加し、選択します。
* **ステップ 2**: Variables パネルで、生の入力用の文字列変数（例：`str_jsonInput`）と、結果を格納する変数（例：`jo_parsedData`）が作成されていることを確認します。
* **ステップ 3**: **Ask AI** パネルを開き、以下を入力します。
  > "@activity @variable Explain how to configure my selected  to parse `str_jsonInput` and store the result in `jo_parsedData`"
* **ステップ 4**: **Send**（送信）をクリックします。
* **ステップ 5**: AIの回答を確認し、ガイダンスに従ってプロパティを設定します。
```markdown
選択した Deserialize Json <T> アクティビティを構成して str_jsonInput をパースし、結果を jo_parsedData に格納するには、次の手順に従います。

1. JSON String プロパティに、変数 str_jsonInput の値を入力します。JSONはインデントなしでダブルクォーテーションで囲み、配列内の各文字列はシングルクォーテーションで記述されていることを確認してください。たとえば、str_jsonInput に文字列 {'Name': 'Maria'} が含まれている場合は、{"Name": 'Maria'} として入力します。

2. Misc セクションで、このアクティビティのデータをログに記録するために Public チェックボックスをオンにします。

3. 今回はジェネリック型の System.String を使用しているため、TypeArgument ドロップダウンボックスは関係ありません。

4. Deserialize Json <T> アクティビティの出力は変数 jo_parsedData に格納されます。

構成例は以下のようになります。

- JSON String (String): {"Name": 'Maria'}
- Public (Checkbox): オン (Checked)
- Display Name (String): Deserialize Json <T>
- TypeArgument (Dropdown box): System.String (デフォルトで選択)

これらのプロパティを構成すると、Deserialize Json <T> activity は str_jsonInput 内のJSON文字列をパースし、結果を jo_parsedData に格納します。
```
