# Recorderの使い方

## アプリ/Web Recorderについて

Recorderを使用すると、タスクの自動化にかかる時間を大幅に短縮できます。Recorderは画面上で実行した操作を記録し、その操作に基づいて一連のアクティビティを含むApplication/Browserアクティビティを生成します。

記録中はすべてのユーザーインターフェイス要素が強調表示されるため、正しいボタン、フィールド、またはメニューが選択されていることを確認できます。RecorderはUI要素の種類ごとに既定の操作を記録します。また、操作を実行する前に、目的の操作を手動で選択することもできます。

Recorderには、プロジェクトに追加されるアクティビティのリアルタイムプレビューが表示されます。プレビューには説明用のスクリーンショットが含まれており、これらは自動化プロジェクト内のアクティビティにも追加されます。**Recorded Steps** ヘッダーをクリックすると、プレビューセクションを折りたたむことができます。

記録した各操作から1つのアクティビティが生成されます。次の表は、記録できる操作と、それらに対して生成されるアクティビティを示しています。一部の操作は自動的に記録されますが、その他の操作は記録前に手動で選択する必要があります。

| 自動的に記録される操作 | 記録前に選択する必要がある操作 |
| --- | --- |
| - ボタン、リンク、アイコン、画像などのクリック可能な要素をクリックする（**Click** アクティビティ）。<br />- テキストボックスまたはテキストエリアにテキストを入力する（**Type Into** アクティビティ）。<br />- チェックボックスを選択または解除する（**Check/Uncheck** アクティビティ）。<br />- ドロップダウンリストから項目を選択する（**Select Item** アクティビティ）。 | - **Get Text** アクティビティを使用してテキストをコピーする。<br />- **Hover** アクティビティを使用して要素上にマウスポインターを移動する。<br />- キーボードショートカットを送信する（**Send Hot Key** アクティビティ）。<br />- **Select Multiple Items** アクティビティを使用して複数の項目を選択する。 |

## 操作を記録する

1. 自動化するアプリケーションまたはWebブラウザーのページを開きます。

2. Studioのリボンで **Recorder** をクリックします。

   ![リボンのRecorder](/static/img/record-ribbon.png)

   Recorderウィンドウが開きます。

   ![Recorderウィンドウ](/static/img/record-window.png)

3. **Record Window** で **Record** をクリックします。Recorderが操作を記録できる状態になります。記録を一時停止するには **Recording** をクリックします。記録を再開するには、もう一度 **Record** をクリックします。

   既定では、操作対象のUI要素を指定すると、ターゲットの選択範囲が自動的に設定されます。

4. 操作を実行する要素をクリックして指定します。ウィンドウ内の各要素は、マウスを移動すると強調表示されます。

   - UI要素の既定の操作を記録するには、ターゲット要素をクリックします。
   - **Get Text** や **Hover** など、Recorderが自動的に取得しない操作を実行するには、記録を停止し、**Record** ボタンの横にある **Element** メニューから適切なオプションを選択します。目的の操作を選択した後、アプリケーションまたはブラウザーでターゲット要素をクリックします。

   ![選択した記録操作](/static/img/record-action-selected.png)

5. アクティビティを生成する前に入力が必要な操作があります。プロンプトが表示されたら必要な情報を入力し、キーボードの **Enter** を押すか、**Confirm** をクリックします。たとえば、テキストボックスを選択すると **Type Into** 操作が検出され、入力するテキストの入力を求められます。

6. 操作の記録が完了したら、**Save & Exit** をクリックしてRecorderを閉じ、アクティビティをプロジェクトに追加します。

   ![操作の記録](/static/img/recording.gif)

### 記録したステップの管理

記録中は、**Recording** をクリックしていつでも記録を一時停止できます。記録を再開するには、もう一度 **Record** をクリックします。

**Recorded Steps** の下にあるプレビューを使用して、追加済みのアクティビティを管理できます。アクティビティを削除するには、操作の右上にある **Recycle Bin** アイコンをクリックします。

既存の記録済み操作の後に操作を挿入するには、記録済み操作の一覧から対象の操作をクリックします。選択した操作が強調表示されます。次に、自動記録を使用して新しい操作を追加するか、Recorderメニューから操作を手動で選択するかを指定します。新しい操作は、強調表示された操作の後に挿入されます。

Recorderウィンドウでは、記録中とワークフロー実行時の両方で、記録した操作の実行に使用する **Input Mode** を選択できます。これにより、特定の要素に対する入力方法をリアルタイムでテストできます。記録した操作が実行されないように見える場合は、入力方法を **UI Automation** に変更してみてください。要素が他の入力方法をサポートしていない可能性があります。

入力モードメニューをクリックすると、**Default**、**UI Automation**、または **Active Accessibility** を選択できます。既定では入力方法に **Default** が設定され、最適な入力方法が自動的に検出されます。

## 自動生成されるアクティビティ

対応する操作が検出されると、次のアクティビティが自動的に生成されます。必要に応じて、記録の完了後にアクティビティのプロパティを更新できます。

- **Click** — 記録中にボタンまたはリンクを選択すると生成されます。クリックの種類とマウスボタンも記録されます。

- **Type Into** — 記録中にテキストボックスまたはその他の編集可能なUI要素を選択すると生成されます。

  ![Type Into操作](/static/img/action-type-into.png)

- **Select Item** — 記録中にドロップダウンリストまたはコンボボックスを選択すると生成されます。ドロップダウンリストを選択した後、操作を完了するためにリストから項目を選択するよう求められます。

  ![Select Item操作](/static/img/action-select-item.png)

- **Check/Uncheck** — ラジオボタンまたはチェックボックスを選択すると生成されます。チェックボックスに対する既定の操作は **Toggle** です。

  ![Check/Uncheck操作](/static/img/action-check.png)

## 手動生成されるアクティビティ

状況によっては、操作を記録してアクティビティを生成する前に、実行する操作を手動で選択する必要があります。

次の操作は自動的に記録できません。

- **Get Text** アクティビティを使用してアプリケーションまたはWebブラウザーからテキストをコピーする。操作を選択してターゲット要素を指定した後、テキストを格納する変数名を入力します。
- **Hover** アクティビティを使用してUI要素上にマウスポインターを移動する。

自動的に記録される操作が、目的の操作とは異なる場合があります。たとえば、メモ帳のテキストエリアに入力するためにクリックした場合でも、既定では **Click** 操作が記録されることがあります。この場合は、既定で追加された操作を削除し、目的のアクティビティを手動で追加してください。

記録中にUI要素の横に表示されるメニューから、操作を手動で選択できます。
