# APIリファレンス
## はじめに

akaBot Vision API を使用すると、組織のデータとアカウント情報にプログラムでアクセスして管理できます。

このドキュメントでは、開発者の観点から API の使用法を紹介し、すべての API オブジェクトとメソッドへのリファレンスを示します。

## akaBot Vision の API はどのように機能しますか?

![idp-sequence-drawio.png](/static/img/idp-sequence-drawio.png)

### ａ．ドキュメントのインポート

ユーザーは akaBot Vision にログインして、最初にユーザーのプロフィール セクションの下にあるアカウント セクションの API キーのトークンを取得する必要があります。

![image-20230221102727-1.png](/static/img/image-20230221102727-1.png)

その後、ユーザーは[API Import Document](#)を呼び出し、リクエスト情報 (トークンとファイルをアップロードする必要がある) を IDP サーバーに送信することでドキュメントを手動でアップロードできます。その後、IDP サーバーは documentId とドキュメントのステータス「インポート中」をユーザーに応答します。ドキュメントのサイズによっては、IDP サーバーがドキュメントを処理するのに 2 ～ 3 分かかる場合があります。

### b.ドキュメントステータスの取得

処理時間が経過した後、ユーザーは API[Get Document Status](#)を呼び出してドキュメントのステータスを再確認します。リクエスト情報は、API[Import Document](#)を介して応答された documentId です。

* ドキュメントのステータスが「確認済み」の場合、このドキュメントは正常に処理され、ユーザーは入力パラメータ (Id およびexportType) を指定して API[Export Document](#)を呼び出すことでデータをエクスポートできることを意味します。

![confirmed.png](/static/img/confirmed.png)

* ドキュメントのステータスが「拒否」になっている場合は、IDP がこのドキュメントが請求書ではないことを検出し、拒否したことを意味します。

![rejected.png](/static/img/rejected.png)

* ドキュメントのステータスが「分割」で、分割されたドキュメントが空でない場合は、このドキュメントに複数のページがあり、IDP が自動的に子ドキュメントに分割されていることを意味します。ユーザーは親 ID を使用して分割ドキュメントをマッピングできます

![split.png](/static/img/split.png)

  + 現時点では、ユーザーは、splittedDocs で応答された documentId の API[Get Document Status](#)を呼び出すことで子ドキュメントのステータスを確認し、API[Export Document](#)を呼び出すことによって、splittedDocs の「確認済み」ドキュメントをエクスポートできます。
  
![split-docs.png](/static/img/split-docs.png)

* ドキュメントのステータスが「レビュー中」の場合、ユーザーはドキュメントをレビューし、データをエクスポートする前に API[Update Document Status](#)によってドキュメントを「確認済み」ステータスに変更する必要があることを意味します。

![to-review.png](/static/img/to-review.png)

### c.ドキュメントの取得

ユーザーが特定のステータスを持つすべてのドキュメントのリストを取得したい場合は、API[Get Documents](#)を呼び出すことができます。

### d.ドキュメントのエクスポート

ユーザーは、ドキュメントが「確認済み」ステータスに変更されるのを待ち、入力パラメーター (トークン、抽出する必要がある documentId、およびユーザーがエクスポートしたいファイル形式) を使用して IDP サーバーに[API Export Document](#)を呼び出して、抽出されたデータ情報を取得する必要があります。その後、[API Export Document](#)は、選択した形式で抽出されたデータをユーザーに応答します。

API サンプルは、以下の API 詳細に従います。

## APIの詳細

### 1. ドキュメントのインポート

* 目的: REST API を使用してドキュメントを akaBot Vision にインポートできます。サポートされているファイル形式は PDF、PNG、JPEG です。
* URL：[`{serverEndpoint}/api/uploadFile/{pipelineId}`](http://idp.akabot.io/api/uploadFile/%7bpipelineId%7d)
* Content-Type: multipart/form-data
* メソッド: POST
* リクエストヘッダー：`Authorization: Bearer {apiKey}`
* 入力本文:

|  |  |  |  |
| --- | --- | --- | --- |
| **属性** | **タイプ** | **説明** | **必須** |
| パイプラインID | 文字列 | パイプラインのID | はい |
| ファイル | フォームデータ | アップロードするドキュメントをリストする | はい |

* 応答:

成功するとコード 200 が返され、応答はドキュメント モデル配列になります。

失敗した場合は、以下のエラーコードとメッセージコードを返します。

|  |  |
| --- | --- |
| **応答コード** | **説明** |
| 200 | OK |
| 201 | 作成されました |
| 401 | 無許可 |
| 403 | 禁止 |
| 404 | 見つかりません |

応答例:

![image-20221128175926-2.png](/static/img/image-20221128175926-2.png)

### 2. ドキュメントのエクスポート

* 用途：確認後のドキュメントはAPIエクスポートドキュメント経由でcsv,json,xlxs,xmlファイルにエクスポート可能
* URL：`{serverEndpoint}/api/export-documents/{id}?exportType={exportType}`
* コンテンツタイプ: application/json
* メソッド: GET
* リクエストヘッダー：`Authorization: Bearer {apiKey}`
* 入力パラメータ:

|  |  |  |  |
| --- | --- | --- | --- |
| **属性** | **タイプ** | **説明** | **必須** |
| ID | 文字列 | ドキュメント ID がエクスポート データを必要としています | はい |
| エクスポートタイプ | 文字列 | ユーザーがエクスポートしたいファイル形式 * csv * xlxs * xml * json | はい |

* 出力
  + 成功すると、コード 200 と入力タイプのファイルが返されます。
  + 失敗した場合は、以下のエラーコードとメッセージコードを返します。

|  |  |
| --- | --- |
| **応答コード** | **説明** |
| 200 | OK |
| 401 | 無許可 |
| 403 | 禁止 |
| 404 | 見つかりません |

### 3. 書類を入手する

* 目的: 特定のステータス (IMPORTING、TOREVIEW、CONFIRMED、EXPORTED、DELETED、REJECTED) を持つ特定のパイプラインのリスト ドキュメントを取得できます。ユーザーがステータスを API に送信しない場合、API はすべてのステータスを含むドキュメントのリストを応答します。
* URL：`{serverEndpoint}/api/documents?page={page}&size={size}&status={status}&pipelineId={pipelineId}&documentName={documentName}&dateFrom={dateFrom}&dateTo={dateTo}&sort=createdDate,desc`
* コンテンツタイプ: application/json
* メソッド: GET
* リクエストヘッダー：`Authorization: Bearer {apiKey}`
* 入力パラメータ:

|  |  |  |  |
| --- | --- | --- | --- |
| **属性** | **タイプ** | **説明** | **必須** |
| ページ | 整数 | 要求されたページのページ番号 | いいえ |
| サイズ | 整数 | ページのサイズ | いいえ |
| 並べ替える | 文字列 | 形式の並べ替え基準: property(,asc|説明）。デフォルトの並べ替え順序は昇順です。 | いいえ |
| ステータス | 文字列 | 文書のステータス (インポート中、レビュー中、確認済み、エクスポート済み、削除済み、拒否済み) | いいえ |
| パイプラインID | 文字列 | パイプラインの ID | はい |
| 文書名 | 文字列 | 入力値を含むドキュメント名の検索 | いいえ |
| 日付から | 整数 | 文書を日付から並べ替える | いいえ |
| 日付まで | 整数 | 文書を日付までに並べ替える | いいえ |

* 出力

成功するとコード 200 が返され、応答はドキュメント モデル配列になります。

失敗した場合は、以下のエラーコードとメッセージコードを返します。

|  |  |
| --- | --- |
| **応答コード** | **説明** |
| 200 | OK |
| 401 | 無許可 |
| 403 | 禁止 |
| 404 | 見つかりません |
