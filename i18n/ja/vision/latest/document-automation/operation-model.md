# 運用モデル
## 自動化処理
akaBot Vision (IDP) は、ドキュメント処理の自動化を念頭に置いて設計されています。自動化処理における IDP の運用モデルを以下に説明します。

![1694170289882-347.png](/static/img/1694170289882-347.png)

**ステップ 1 および 2:** ドキュメントは、RPA ボット、API、または人間の作業を介して複数のドキュメント ソースから収集され、IDP システムにアップロードされます。

**ステップ 3:** IDP システムは、修正または確認するために人間によるサポートが必要な信頼性の低いフィールドがある場合にユーザーに通知します。

**ステップ 4 および 5:** IDP システムはドキュメントを処理し、出力を準備完了キューに入れます。 RPA ボット、API、または人間は IDP 出力をダウンロードし、その構造化データを使用して他のエンタープライズ システムに入力します。

## 運用モデル
akaBot Vision システムから出力を取得するには 3 つのモードがあります。

### 1. 単一請求書処理モデル:
ユーザーは 1 つの特定のドキュメントをアップロードし、そのドキュメントの出力のみを必要とします。このモデルの場合、ユーザーは次の 3 つの手順を実行できます。

- **ステップ 1:** ユーザーは 4 つの方法のうち 1 つで 1 つの特定のドキュメントを akaBot Vision にインポートする必要があります: **ドキュメントを手動でインポート**、**電子メール経由でドキュメントをインポート**、**API インポート ドキュメント**、または **アクティビティ インポート ドキュメント**、
- **ステップ 2:** **API Get Document Status** または **Activity Get Document Status** によりドキュメント ステータスを取得します。
- **ステップ 3:** **API エクスポート ドキュメント** または **アクティビティ エクスポート ドキュメント** により、「確認済み」ステータスのドキュメントをエクスポートします。

### 2. 手動一括請求書処理モデル:
ユーザーは GetDocument を使用して準備ができているすべてのドキュメントを取得し、定期的にスキャンして完成したドキュメントを取得するバッチ ジョブを設定できるすべてのドキュメントを取得します。このモデルは、多数のファイルを入力し、定期的にチェックして準備ができているすべてのファイルを取得する大量処理用です。このモードでは、ユーザーは次の 4 つの手順を実行できます。

- **ステップ 1:** ユーザーは、次の 4 つの方法のうち 1 つでドキュメントのリストを akaBot Vision にインポートする必要があります: **ドキュメントを手動でインポート**、**電子メール経由でドキュメントをインポート**、**API ドキュメントをインポート**、または **アクティビティ インポート ドキュメント**、

- **ステップ 2:** **API Get Document Status** または **Activity Get Document Status** によりドキュメント ステータスを取得します。
- **ステップ 3:** **API Get Document** または **Activity Get Document** により、「確認済み」ステータスを持つすべてのドキュメントのリストを取得します。
- **ステップ 4:** **API エクスポート ドキュメント** または **アクティビティ エクスポート ドキュメント** により、「確認済み」ステータスのドキュメントをエクスポートします。

### 3. 自動一括請求書処理モデル:
akaBot Vision は、ドキュメントが正常に処理されるとすぐに出力をエンドポイントにプッシュします。このモデルも大量処理向けですが、ファイルを定期的にチェックして取得するジョブを設定する必要がないため、よりスムーズです。このモードでは、ユーザーは次の 2 つの手順を実行できます。

- **ステップ 1:** ユーザーは、次の 4 つの方法のうち 1 つでドキュメントのリストを akaBot Vision にインポートする必要があります: **ドキュメントを手動でインポート**、**電子メール経由でドキュメントをインポート**、**API ドキュメントをインポート**、または **アクティビティ インポート ドキュメント**、

- **ステップ 2:** ドキュメントが「確認済み」ステータスに変更されるとすぐに、akaBot Vision は **API 出力**によって documentId 情報、ドキュメント パス、および抽出されたデータを含む応答をユーザーのサーバーに自動的に送信します。

注: akaBot Vision の API のほとんどは同期 API です。 APIインポートドキュメントとAPI出力のみが非同期APIです
