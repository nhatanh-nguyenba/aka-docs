# RPA リファレンス
## IDPの範囲

IDP アクティビティは、akaBot エコシステムのコンポーネントである akaBot Studio の IDP パッケージに属します。

これらのアクティビティを使用する場合、ファイルを akaBot Vision にインポートしたりエクスポートしたり、ステータスを手動で変更したりする必要はありません。アクティビティは 6 つあります。

akaBot StudioとakaBot VisionをAPIキーで接続します。

* akaBot Vision: [アカウント/API キー] に移動し、API キーを生成してコピーします。
* akaBot Studio: IDP スコープの API キー フィールドにキーを貼り付けます。

![image-20220420200751-1.png](/static/img/image-20220420200751-1.png)

### ドキュメントのインポート

指定されたパイプラインにファイルをインポートします。重点的に取り組む必要がある分野は 3 つあります。

* ファイルパス: インポートするファイル。
* パイプライン キー: ファイルのインポート先のパイプライン。キーはパイプラインの ID です。
* ドキュメント キー: インポートされたファイルの ID。このフィールドは他のアクティビティで使用されます。

![image-20220420200751-2.png](/static/img/image-20220420200751-2.png)

### ドキュメントのエクスポート

必要に応じてファイルをエクスポートし、ステータスを変更します。重点的に取り組む必要がある分野は 3 つあります。

* ドキュメント キー: エクスポートするファイルの ID。このキーは、ドキュメントのインポート アクティビティから使用できます。
* 抽出タイプ: DataTable/Json/XML 形式を選択できます。
* ステータスをエクスポート済みに更新: 値が true の場合はステータスが更新され、それ以外の場合はステータスのままです。 akaBot Visionでステータスを確認できます。

### ドキュメントの取得

特定のパイプライン内の特定のドキュメントを取得します。重点的に取り組む必要がある分野が 2 つあります。

* パイプライン キー: ドキュメントを取得するパイプラインの ID
* ステータス：取得したいステータス

例: キー 1 を持つパイプラインですべての確認済みドキュメントを取得できます。

### ドキュメントステータスの取得

特定のドキュメントのステータスを取得します。注目すべきフィールドが 2 つあります。

* ドキュメントキー: ステータスを取得したいドキュメントのID

### ドキュメントステータスの更新

ステータスを別のステータスに変更すること。重点的に取り組む必要がある分野は 3 つあります。

* ドキュメント キー: 更新するファイルの ID。このキーは、ドキュメントのインポート アクティビティから使用できます。
* ステータスを更新: 更新するステータスを選択します (インポート中、レビューあり、確認済み、エクスポート済み、削除済み、拒否済み)。

### 例

以下は IDP アクティビティの使用方法の例です。

* フォルダー内のすべてのファイルをオートメーション タイプが [常に] のパイプラインにインポートし、エクスポートしてステータスを変更します。

![image-20220420201839-1.png](/static/img/image-20220420201839-1.png)

* フォルダー内のすべてのファイルを、オートメーション タイプが「信頼」であるパイプラインにインポートし、エクスポートしてステータスを変更します。

![image-20220420201909-2.png](/static/img/image-20220420201909-2.png)
