# ダッシュボード
マネージャーまたは管理者として、ユーザーは、たとえば、特定のキューにインポートされたすべてのドキュメントや、最も多くの修正が必要なデータ フィールドについて知りたい場合があります。ダッシュボードのレポートは、従業員がドキュメントをより効果的かつ効率的に処理するのにも役立ちます。

この記事では、ダッシュボードの使い方を紹介します。

まず、ユーザーは上部のメニュー バーで [ダッシュボード] を選択して、ダッシュボードにアクセスできます。

![1704972886297-956.png](/static/img/1704972886297-956.png)

では、akaBot Visionのチャートとその意味を紹介します。

## 1. 総輸入量

この統計は、期間内に akaBot Vision にアップロードされたドキュメントの合計数を示します。

## 2. 正常に処理されました

この統計は、「確認済み」ステータスを持つドキュメントの総数を示します。

注: この数には、ユーザーが「日付範囲」で設定した指定期間より前にインポートされたが、この期間内に確認されたドキュメントが含まれます。

## 4. 自動ドキュメント

この統計は、akaBot Vision にインポートされ、[確認済み] タブに直接移動されたドキュメントの総数を示します。

## 5. 合計レビュー保留中

この統計は、ステータスが「レビュー中」でユーザーによるレビューが必要なドキュメントの総数を示します。

## 6. 平均処理時間

この統計は、ドキュメントが「レビュー中」ステータスから「確認済み」ステータスに移行するまでの平均時間を示します。

次の式で計算されます。

**平均処理時間 = ドキュメントが「レビュー中」ステータスから「確認済み」ステータスに移行する時間 / ドキュメント総数**

## 7. 平均ライフサイクル時間

この統計は、ドキュメントが「レビュー中」ステータスから「エクスポート済み」ステータスに移行するまでの平均時間を示します。

次の式で計算されます。

**平均ライフサイクル時間 = ドキュメントが「レビュー中」ステータスから「エクスポート済み」ステータスに移行する時間 / ドキュメント合計**

## 8. 文書の使用

スループットの追跡は、一定期間内にインポート、エクスポート、および削除されたドキュメントの量を分析する場合に重要です。上部の数値は、選択した期間中に蓄積された合計です。これらの数値の下にある時系列は、ユーザーの日付範囲内でインポート、エクスポート、および削除されたドキュメントの量を表します。
