# Akabot Vision — リリースノート

## v3.2.0 — 2026 年 4 月

**新機能**
- **生成抽出** — 大規模な言語モデルを使用して、事前定義されたテンプレートを使用せずにドキュメントからフィールドを抽出します。フィールドをプレーン テキストで記述するだけです。
- **手書きテキストのサポート** — OCR エンジンは、新しい HTR (手書きテキスト認識) モデルを使用して、筆記体および印刷/手書きの混合コンテンツを処理できるようになりました。
- **バッチ処理キュー** - 1 回の API 呼び出しで数百のドキュメントを送信します。 Vision はそれらを非同期的に処理し、結果を Webhook に送信します。

**改善点**
- ネストされたヘッダーを持つ複雑な請求書のテーブル検出精度が 87% から 94% に向上しました。
- 検証ステーションの UI が再設計されました: キーボードのみのナビゲーション、信頼性の高いページの一括受け入れ。
- モデルのトレーニングで実稼働可能な精度を達成するために必要なラベル付きサンプルは 30% 削減されました。

**バグ修正**
- 複数ページの PDF で 10 ページ以降の信頼スコアが 0% として表示される問題を修正しました。
- 前処理でドキュメントを 180 度回転するとフィールド座標マッピングが壊れる問題が解決されました。
- 明示的に保存せずにドキュメント タイプを切り替えると、抽出テンプレート エディターが変更を保存しない問題を修正しました。

## v3.1.0 — 2026 年 1 月

**新機能**
- **ドキュメントの分割** — 分類前に複数ドキュメントの PDF バンドルを自動的に検出し、個々のファイルに分割します。
- **署名検出** — 定義された領域内の署名の有無を検出する新しいフィールド タイプ。
- **REST API v2** — 5 MB 未満のドキュメントの同期抽出エンドポイント。ポーリングは必要ありません。

**改善点**
- 拡張されたコーパスで再トレーニングした後、ベトナム語モデルの精度が 18% 向上しました。
- 処理スループットは、テナントあたり 500 ページ/分 (200 ページ/分) に増加しました。
- 抽出結果には、抽出された各値の境界ボックスの座標が含まれるようになりました。

**バグ修正**
- CCITT グループ 4 圧縮を使用して TIFF ファイルを処理するときの断続的なエラーが修正されました。
- 右から左へ書く (RTL) テキストと LTR コンテンツが混在するドキュメントでのフィールド抽出を修正しました。

## v3.0.0 — 2025 年 9 月

**新機能**
- **Akabot Vision v3 の発売** — トランスフォーマーベースの OCR および抽出パイプライン上に再構築されました。
- **コード不要のテンプレート エディタ** — ドキュメント画像上にフィールド ゾーンを描画し、ラベルを付けます。設定ファイルは必要ありません。
- **ヒューマンインザループステーション** — レビュー担当者が下流に流れる前に不確実な抽出を検証および修正するための組み込み Web UI。

**重大な変更**
- Vision v2 テンプレート形式 (`.vtemplate`) は v3 と互換性がありません。新しいエディタを使用してテンプレートを再作成します。
- Vision 用 v2 Python SDK は非推奨になりました。 REST API または新しい Studio アクティビティ パックを使用します。

**バグ修正**
- 100 MB を超える PDF ファイルを処理する際のメモリ枯渇の問題を解決しました。
