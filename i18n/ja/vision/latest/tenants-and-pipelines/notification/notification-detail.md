# 通知の詳細
通知の詳細には自動化プロセスのエラーが表示され、通知の詳細は [ヘルプ] ボタンの隣にあるリング アイコンで示されます。エラーには、エラー ドキュメントとエラー モデルの 2 種類があります。

## 1. エラー文書

エラードキュメントとして定義されたドキュメントには、以下の条件があります。

* **インポートされたドキュメントのエラー**: ドキュメントが akaBot Vision にインポートされるまでに 30 分以上かかると、ドキュメントは「インポートに失敗しました」ステータスに移行し、システムはそれらをエラー ドキュメントにリストします。
* **エラー処理されたドキュメント**: ドキュメントが請求書形式として識別されない場合、それらはエラー処理されたドキュメントとして定義されます。
* **エクスポートされたドキュメントのエラー**: ユーザーがドキュメントのエクスポートに失敗すると、ドキュメントはエクスポートされたドキュメントのエラーとしてリストされます。また、パイプラインが API Out を使用している場合、akaBot Vision は API に接続してリクエストを送信できません。これらのドキュメントは、エラー エクスポートされたドキュメントとして定義されます。

エラー ドキュメントの場合、ユーザーはエラー ドキュメントとそのエラーのリストを表示できます。さらに、ユーザーはパイプラインでドキュメントをフィルタリングすることもできます

![image-20221116102925-2.png](/static/img/image-20221116102925-2.png)

エラーがインポートされたドキュメントについては、akaBot Vision はユーザーが「再試行」機能によってこれらのドキュメントを再インポートすることをサポートします。ユーザーはドキュメントを選択して「再試行」をクリックするだけです。

![image-20221116104102-3.png](/static/img/image-20221116104102-3.png)

## 2. エラーモデル

トレーニングされたモデルが失敗した場合、それをエラー モデルとして定義できます。
